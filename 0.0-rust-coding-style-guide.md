# Rust 编码规范 —— 命名、格式化、组织

> 在学 Rust 的每个特性之前，先知道**怎么写才是"地道"的 Rust**。本规范参考 [Rust API Guidelines](https://rust-lang.github.io/api-guidelines/) 和 `rustfmt` 默认配置。

---

## 1. 命名规范 —— 大小写决定"你是谁"

Rust 用**大小写区分**不同种类的标识符。编译器不会强制所有规则（有些只是 convention），但整个生态遵循它们。

| 类别 | 命名风格 | 示例 | 编译器强制？ |
|:---|:---|:---|:---:|
| **类型**（struct, enum, trait, type alias, enum variant） | `UpperCamelCase`（大驼峰） | `HashMap`, `Read`, `IpAddr::V4` | ✅ 是（warning） |
| **变量、函数、方法** | `snake_case`（蛇形） | `user_name`, `get_value()`, `as_bytes()` | ✅ 是（warning） |
| **常量、静态变量** | `SCREAMING_SNAKE_CASE`（全大写蛇形） | `MAX_SIZE`, `DEFAULT_TIMEOUT` | ❌ 否（约定） |
| **模块、crate** | `snake_case` | `std::collections`, `serde_json` | ✅ 是 |
| **宏** | `snake_case`（与函数一致） | `println!`, `vec!` | ❌ 否（约定） |
| **生命周期参数** | 短小的小写字母（`'a`, `'b`, `'de`, `'ctx`） | `'a`, `'static`, `'de` | ❌ 否（约定） |
| **泛型参数** | 单大写字母，或 `UpperCamelCase` | `T`, `K`, `V`, `Error`, `Output` | ❌ 否（约定） |

### 1.1 详细说明

```rust
// ===== 类型：大驼峰 =====
struct UserProfile { ... }       // ✅
enum ConnectionState { ... }     // ✅
trait IntoIterator { ... }       // ✅
type UserId = u64;              // ✅

struct user_profile { ... }     // ❌ warning: should be UpperCamelCase
trait intoIterator { ... }      // ❌ warning: should be UpperCamelCase

// ===== 变量和函数：蛇形 =====
let user_name = "Alice";        // ✅
fn get_user_by_id(id: u64) {}   // ✅
fn GetUserById(id: u64) {}      // ❌ warning: should be snake_case

// ===== 常量：大写蛇形 =====
const MAX_RETRY_COUNT: u32 = 3;         // ✅ 推荐
static DEFAULT_CONFIG: &str = "...";    // ✅ 推荐
const maxRetryCount: u32 = 3;           // ⚠️ 能编译，但不合约定

// ===== 枚举变体：大驼峰 =====
enum IpAddr {
    V4(u8, u8, u8, u8),   // ✅
    V6(String),            // ✅
}
// 注意：V4 和 V6 是类型级别的构造器，所以用大驼峰

// ===== 宏：蛇形 =====
macro_rules! my_macro { ... }   // ✅
// myMacro!();                   // ❌ 不合约定，虽然能编译
// 历史原因：早期 Rust 有 `println!` 而非 `printLn!`
// 现在的共识是宏和函数用同样的 snake_case

// ===== 生命周期：短小 =====
fn longest<'a>(x: &'a str, y: &'a str) -> &'a str { ... }  // ✅ 'a 够用
// 当有多个生命周期需要语义区分时：
fn copy_from<'src, 'dst>(src: &'src [u8], dst: &'dst mut [u8]) { ... }
//                                     ^^^^   ^^^^ 有意义的名字

// ===== 泛型参数 =====
fn first<T>(list: &[T]) -> &T { ... }               // ✅ T 是最常用的
fn get<K, V>(map: &HashMap<K, V>, key: &K) -> &V    // ✅ K, V 有意义
// 当泛型参数有 trait bound 且有语义时，用有意义的名称：
fn parse<Input, Error>(data: Input) -> Result<..., Error> { ... }
```

## 2. 代码格式化 —— `rustfmt` 就是"标准答案"

### 2.1 缩进与空格

```rust
// ✅ 4 个空格缩进（永远不要用 Tab）
fn example() {
    if condition {
        do_something();     // 每层 4 个空格
    }
}

// ✅ 运算符两侧加空格
let x = 1 + 2 * 3;          // ✅
let x=1+2*3;                // ❌ 读起来费劲

// ✅ 逗号后面加空格，前面不加
let tuple = (1, 2, 3);      // ✅
let tuple = (1 ,2 ,3);      // ❌

// ✅ 冒号：类型注解处，冒号后空格、冒号前不空格
let x: i32 = 42;            // ✅
let x : i32 = 42;           // ❌

// ✅ 分号：后面通常换行
let x = 1;
let y = 2;
```

### 2.2 行宽与换行 —— 默认 100 字符

`rustfmt` 默认最大行宽是 **100 字符**。超过时，需要换行。

```rust
// ===== 长参数列表：每个参数独占一行 =====
fn complex_function(
    first_param: Type1,
    second_param: Type2,
    third_param: Type3,      // ← 末尾逗号！便于 git diff
) -> ReturnType {
    // 函数体
}

// ===== 长函数调用 =====
let result = some_function(
    arg1,
    arg2,
    very_long_argument_name, // 末尾逗号
);

// ===== 长链式调用：每个方法调用独占一行 =====
let result = data
    .iter()
    .filter(|x| x.is_valid())
    .map(|x| x.transform())
    .collect::<Vec<_>>();

// ===== where 子句：当泛型约束太长时用 where =====
// ❌ 太长了：
fn foo<T: std::fmt::Debug + Clone + PartialEq + Send>(x: T) {}

// ✅ 用 where 子句：
fn foo<T>(x: T)
where
    T: std::fmt::Debug + Clone + PartialEq + Send,
{
    // ...
}

// ===== 模式匹配：match 分支 =====
match value {
    Pattern1 => short_action,
    Pattern2 => {
        // 多行块用花括号
        longer_action();
        multi_line();
    }
}

// ===== 闭包：短的就一行，长的就加花括号 =====
let short = items.iter().map(|x| x + 1);
let long = items.iter().map(|x| {
    let y = expensive_op(x);
    y + 1
});
```

### 2.3 花括号风格

```rust
// ✅ 左花括号和声明在同一行，前面留一个空格
fn foo() {
    if condition {
        loop {
            break;
        }
    }
}

// ❌ 不要把左花括号放在下一行（Rust 不这样做）
// fn foo()
// {
// }

// ✅ struct/enum 定义也一样
struct User {
    name: String,
    age: u32,
}

enum State {
    Running,
    Stopped,
}
```

### 2.4 逗号 —— 善用尾随逗号

```rust
// ✅ 尾随逗号——推荐！（尤其是多行时）
let values = vec![
    1,
    2,
    3,  // ← 这个逗号让后续添加元素时 git diff 只显示一行
];

enum Color {
    Red,
    Green,
    Blue,  // ← 尾随逗号
}

// match 分支也用尾随逗号：
match x {
    Some(v) => v,
    None => 0,    // ← 尾随逗号
};
```

### 2.5 空行 —— 用空行分隔逻辑段落

```rust
// ✅ 在 impl 块之间、函数之间、结构体之间留空行
struct Config {
    name: String,
    timeout: Duration,
}

impl Config {
    fn new(name: String) -> Self {
        Config {
            name,
            timeout: Duration::from_secs(30),
        }
    }

    fn with_timeout(mut self, timeout: Duration) -> Self {
        self.timeout = timeout;
        self
    }
}

// ❌ 不要在同一函数内部随意加空行——只在逻辑段落变化时加
fn process() {
    // 步骤 1：读取
    let data = read_file();

    // 步骤 2：解析
    let parsed = parse(&data);

    // 步骤 3：验证
    validate(&parsed);
}
```

## 3. 导入顺序 —— `use` 语句的组织

标准顺序：**std → 第三方 crate → crate 内部 → super → self**

```rust
// ✅ 正确的导入顺序（每组之间用空行分隔）
use std::collections::HashMap;
use std::path::PathBuf;

use serde::{Deserialize, Serialize};
use tokio::net::TcpListener;

use crate::config::AppConfig;
use crate::models::User;

use super::parent_module_item;
use self::child_module_item;
```

### 3.1 合并导入

```rust
// ✅ 同一个模块的多个导入合并（5 个以上）
use std::collections::{HashMap, HashSet, BTreeMap, BTreeSet, VecDeque};

// ✅ 也可以合并层级
use std::{
    collections::HashMap,
    path::PathBuf,
    sync::Arc,
};

// ✅ 少量导入不合并也可以
use std::io::Read;
use std::io::Write;
```

### 3.2 重导出

```rust
// pub use 让外部可以直接从你的模块导入
pub use self::inner::PublicType;

// 常见的重导出模式：
pub mod error {
    pub use crate::error::{AppError, Result};
}
```

## 4. 文档注释 —— `///` 和 `//!`

### 4.1 行内文档 vs 模块文档

```rust
// ===== /// 用于"下一个条目"（函数、结构体、方法等） =====
/// 返回两个字符串中较长的那个。
///
/// # Examples
///
/// ```
/// let result = longest("hello", "world");
/// assert_eq!(result, "world");
/// ```
fn longest<'a>(x: &'a str, y: &'a str) -> &'a str {
    if x.len() > y.len() { x } else { y }
}

// ===== //! 用于"包裹着它的条目"（模块、crate 级别） =====
//! 这个模块提供了字符串比较工具。
//!
//! # 使用方式
//!
//! ```
//! use my_crate::string_utils::longest;
//! ```

// ===== 文档注释的第一行是"摘要"——给 `cargo doc` 的索引页看的 =====
/// 创建一个新的空 `HashMap`。
///
/// 这个 map 初始不分配内存。
/// # 后面的内容只在展开后显示
pub fn new() -> HashMap<K, V> { ... }

// ===== 文档中引用其他条目 =====
/// 返回 [`User`] 的姓名。
/// 详见 [`User::name`]。

// ===== 文档中的代码示例应该是可运行的 =====
/// ```
/// # // 以 # 开头的是隐藏行——在文档中不显示，但会运行
/// # use my_crate::add;
/// assert_eq!(add(2, 3), 5);
/// ```
```

### 4.2 什么时候写文档

- **所有 pub 函数、pub 类型、pub 模块** → 应该写文档
- **私有函数** → 复杂逻辑时写 `//` 普通注释
- **复杂实现细节** → 用 `//` 在函数体内注释

## 5. 代码组织 —— 模块、文件、结构

### 5.1 文件与模块结构

```
my_crate/
├── Cargo.toml
├── src/
│   ├── main.rs          # 或 lib.rs —— crate 根
│   ├── config.rs        # 独立模块
│   ├── models/
│   │   ├── mod.rs       # 目录模块的入口（推荐风格）
│   │   └── user.rs      # 子模块
│   └── utils/
│       └── mod.rs
```

```rust
// ===== 声明模块（在 main.rs 或 lib.rs 中） =====
mod config;
mod models;
mod utils;

// 公共 API 使用 pub mod
pub mod api;
```

### 5.2 impl 块的组织

```rust
struct User {
    name: String,
    age: u32,
}

// 1. 构造方法放在最前面
impl User {
    pub fn new(name: String, age: u32) -> Self {
        User { name, age }
    }
}

// 2. 公共方法按逻辑分组
impl User {
    pub fn name(&self) -> &str {
        &self.name
    }

    pub fn age(&self) -> u32 {
        self.age
    }
}

// 3. Trait 实现分开（每个 trait 一个 impl 块）
impl std::fmt::Display for User {
    fn fmt(&self, f: &mut std::fmt::Formatter<'_>) -> std::fmt::Result {
        write!(f, "{} ({} years old)", self.name, self.age)
    }
}

impl From<(&str, u32)> for User {
    fn from((name, age): (&str, u32)) -> Self {
        User { name: name.to_string(), age }
    }
}
```

### 5.3 结构体字段顺序

```rust
// 把最重要的、最常用的字段放在前面
struct HttpResponse {
    status: StatusCode,   // 最关键：状态码
    headers: HeaderMap,   // 其次：头部
    body: Vec<u8>,        // 最后：正文
}

// 对于新类型包装（newtype），遵守：
struct UserId(u64);     // ✅ 外层是类型名，内层是值

// 对于构建器模式，必填字段在可选字段之前
struct RequestBuilder {
    url: String,           // 必填
    method: Method,        // 必填
    timeout: Option<Duration>,  // 可选
    headers: HeaderMap,         // 可选
}
```

## 6. 常见模式的命名约定

### 6.1 构造与转换

```rust
// ===== 构造函数 =====
impl User {
    /// 创建默认实例 → new
    pub fn new(name: String) -> Self { ... }
    
    /// 从已有实例派生 → from_xxx
    pub fn from_row(row: &DatabaseRow) -> Self { ... }
    
    /// 有额外参数 → with_xxx
    pub fn with_timeout(mut self, timeout: Duration) -> Self {
        self.timeout = timeout;
        self
    }
}

// ===== 转换方法 =====
// as_xxx: 廉价的借用→借用转换（不消耗所有权）
fn as_bytes(&self) -> &[u8] { ... }
fn as_str(&self) -> &str { ... }

// to_xxx: 昂贵的转换（通常有分配）
fn to_string(&self) -> String { ... }
fn to_owned(&self) -> T { ... }

// into_xxx: 消耗所有权的转换
fn into_bytes(self) -> Vec<u8> { ... }
fn into_iter(self) -> IntoIter<T> { ... }

// 优先实现标准 trait 而不是自定义方法：
impl From<User> for UserDto { ... }   // ✅ 用 From trait
// 而不是：
// fn to_dto(&self) -> UserDto { ... }  // ❌ 不合约定
```

### 6.2 访问器（Getter/Setter）

```rust
impl User {
    // ===== Getter：不加 get_ 前缀（Rust 约定） =====
    pub fn name(&self) -> &str { &self.name }       // ✅
    pub fn age(&self) -> u32 { self.age }           // ✅

    // 只有当 get 有"非平凡"的含义时才加 get_：
    // 比如需要计算、或者返回 Option
    pub fn get(&self, key: &str) -> Option<&Value>  // ✅ 类似于 HashMap::get

    // ===== Setter：加 set_ 前缀，通常消耗 self =====
    pub fn set_name(mut self, name: String) -> Self {  // ✅ 构建器风格
        self.name = name;
        self
    }
    // 或者 &mut self 风格：
    pub fn set_name_mut(&mut self, name: String) {     // ✅
        self.name = name;
    }
}
```

### 6.3 迭代器

```rust
impl<T> Vec<T> {
    // iter: 不可变引用迭代
    pub fn iter(&self) -> Iter<'_, T> { ... }

    // iter_mut: 可变引用迭代
    pub fn iter_mut(&mut self) -> IterMut<'_, T> { ... }

    // into_iter: 消耗所有权的迭代
    pub fn into_iter(self) -> IntoIter<T> { ... }
}
// 这是 Rust 的"约定三重奏"——标准库所有的集合都遵循
```

### 6.4 错误类型与 Result

```rust
// ===== 错误类型以 Error 结尾 =====
pub enum AppError {
    Io(std::io::Error),
    Parse(String),
    NotFound,
}

// ===== 类型别名 Result<T> = Result<T, AppError> =====
pub type Result<T> = std::result::Result<T, AppError>;

// 模块内直接用 Result<T>：
pub fn load_config(path: &str) -> Result<Config> {
    let content = std::fs::read_to_string(path)
        .map_err(AppError::Io)?;  // ? 操作符自动转换
    // ...
}

// ===== 错误信息模板 =====
/// 读取配置文件失败。
///
/// # Errors
///
/// 当文件不存在时返回 `AppError::Io`。
/// 当内容格式错误时返回 `AppError::Parse`。
pub fn load_config(path: &str) -> Result<Config> { ... }
```

### 6.5 布尔型函数

```rust
// 返回 bool 的函数通常用 is_ 或 has_ 前缀
fn is_empty(&self) -> bool { ... }
fn is_valid(&self) -> bool { ... }
fn has_key(&self, key: &str) -> bool { ... }
fn contains(&self, value: &T) -> bool { ... }

// 不要用 get_ 返回 bool：
// fn get_is_active(&self) -> bool { ... }  // ❌
```

### 6.6 Unsafe 函数

```rust
// unsafe 函数：函数名应该暗示"调用者需要满足什么前提条件"
unsafe fn from_raw_parts(ptr: *const u8, len: usize) -> String { ... }
//                                   ^^^^^^^^^^^^^^^^ 前提：ptr 有效且长度为 len

// unsafe 块的 SAFETY 注释：
impl<T> MyPtr<T> {
    fn deref(&self) -> &T {
        // SAFETY: `self.ptr` 在构造时保证非空，且指向一个已初始化的 T，
        // 调用者在 MyPtr 存活期间保证不会释放 T 的内存。
        unsafe { &*self.ptr }
    }
}
```

## 7. 常见场景速查

### 7.1 `let` 绑定 —— 需要标注类型吗？

```rust
// ✅ 编译器能推导时，不用标注
let user = User::new("Alice");
let count = users.len();
let mut map = HashMap::new();

// ✅ 无法推导或为了提高可读性时，标注
let config: AppConfig = serde_json::from_str(&json_str)?;
//         ^^^^^^^^^ 这里必须标注——编译器不知道要反序列化什么类型

let timeout: Duration = Duration::from_secs(30);
//           ^^^^^^^^ 可标注也可不标注——标注了更易读

// ✅ 长类型用 type 别名
type Result<T> = std::result::Result<T, MyError>;
```

### 7.2 `if` / `match` —— 使用表达式，不是语句

```rust
// ✅ Rust 中 if 和 match 都是表达式（返回值）
let status = if code == 200 { "OK" } else { "ERROR" };
let message = match code {
    200 => "OK",
    404 => "Not Found",
    500 => "Internal Server Error",
    _ => "Unknown",
};

// ❌ 不要写成 C 风格：
// let mut status = "ERROR";
// if code == 200 {
//     status = "OK";
// }
```

### 7.3 `Option` / `Result` 的简洁处理

```rust
// ✅ 用 ? 而不是 match + return Err
fn load() -> Result<Config> {
    let path = std::env::var("CONFIG_PATH")?;     // ? 自动处理 None/Err
    let content = std::fs::read_to_string(path)?;  // ? 自动处理 Err
    let config: Config = toml::from_str(&content)?;
    Ok(config)
}

// ✅ 用 Option 的组合子而不是 match
let user = find_user(id)?;
// 等价于：
// let user = match find_user(id) {
//     Some(u) => u,
//     None => return None,
// };

let name = user.map(|u| u.name.clone()).unwrap_or_default();
// 等价于复杂的 match + clone + default

// ✅ 用 if let 处理只关心一种模式的情况
if let Some(user) = find_user(id) {
    println!("Found: {}", user.name);
}
```

### 7.4 `self` 参数 —— 按需使用所有权

```rust
impl User {
    // &self：只读访问（最常用）
    fn name(&self) -> &str { &self.name }

    // &mut self：需要修改（借用）
    fn set_age(&mut self, age: u32) { self.age = age; }

    // self：消耗实例（转移所有权）
    fn into_name(self) -> String { self.name }
    // 调用后 user 不再可用
}
```

### 7.5 不要做的事情

```rust
// ❌ 不要用匈牙利命名法
// struct m_user { m_name: String }  // 不要！

// ❌ 不要给私有字段加 pub_ 前缀
// struct User { pub_name: String }  // 不要！

// ❌ 不要在 Rust 中模仿 C#/Java 的接口命名
// trait IReadable { ... }           // 不要 I 前缀！

// ❌ 不要用负逻辑的布尔名（避免 !）
// if !not_found { ... }              // 费解
// if found { ... }                   // ✅ 清晰

// ❌ 尽量不要缩写（除非是极其通用的缩写）
// let usr = get_usr();              // ❌ 不是节约字符的比赛
// let user = get_user();            // ✅

// ✅ 可以接受的缩写：
// - fn, mod, impl, config, err, ctx (context)
// - 标准库里的：iter, mem, ptr, sync
```

---

> **一句话总结**：Rust 的命名和格式化不是"你喜欢哪种风格"——整个生态用 `rustfmt` 统一格式，用 API Guidelines 统一命名。`snake_case` 给函数和变量，`CamelCase` 给类型，`SCREAMING_SNAKE_CASE` 给常量，4 空格缩进，100 字符行宽。当你犹豫怎么写时——跑一下 `cargo fmt`，答案就有了。
