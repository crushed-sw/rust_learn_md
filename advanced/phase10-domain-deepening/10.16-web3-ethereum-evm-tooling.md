# 10.16 以太坊与 EVM 工具链：从解释器到索引器

> **[Track C: Web3]** 以太坊是全球最大的可编程区块链，超过 70% 的 Web3 开发者在这个生态中构建。本节从零实现一个 EVM 解释器（约 200 行 Rust），让你理解智能合约执行的底层机制；然后学习 alloy 生态（Provider、sol! 宏、事件监听）、revm（本地 EVM 模拟）、MEV 基础，最终构建一个完整的以太坊事件索引器（WebSocket → 解码 → SQLite → GraphQL）。所有这些都发生在 Rust 中——不需要切换到 JavaScript/TypeScript，不需要 Hardhat 或 Infura。

**领域背景：** 以太坊的核心创新是图灵完备的智能合约——区块链不仅是"账本"，也是"世界计算机"。EVM（Ethereum Virtual Machine）是这台世界计算机的 CPU：它执行字节码、管理 gas（计算成本）、维护持久化存储。理解 EVM 不是可选技能——它让你能够审计合约、优化 gas 消耗、构建 DEX 聚合器、运行 MEV 策略、构建链上数据产品。

## 1. EVM 从第一原理：构建一个玩具解释器

### 1.1 EVM 的四个核心组件

```
┌─────────────────────────────────────────────────────────────────┐
│  EVM 架构：四个核心组件                                           │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  1. Stack（栈）                                                  │
│     → 最大深度 1024                                              │
│     → 每个元素 256-bit（32 字节）                                │
│     → PUSH1..PUSH32 压入数据，POP 弹出                           │
│     → 算术: ADD/MUL/SUB/DIV/MOD/EXP                             │
│                                                                 │
│  2. Memory（内存，易失）                                          │
│     → 字节寻址（不是 32 字节寻址！）                              │
│     → 按需扩展（首次访问高地址 → 自动扩容到该地址）                │
│     → 扩展成本 = 内存大小的函数 → gas 计费                        │
│                                                                 │
│  3. Storage（存储，持久化）                                       │
│     → 256-bit key → 256-bit value                               │
│     → 每个 slot 初始为非零时才写入（零值优化）                     │
│     → 最贵的操作: SSTORE（写存储 = 冷存储 20K gas）              │
│                                                                 │
│  4. Gas（燃料）                                                  │
│     → 每笔交易有 gas_limit（防止无限循环）                        │
│     → 每个 opcode 有固定的 gas 成本                              │
│     → 未用完的 gas 退还给发送者                                   │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 1.2 核心 Opcodes

```rust
// evm_opcodes.rs —— EVM 核心操作码定义
// EVM 有约 140 个 opcodes，这里列出最关键的子集

/// EVM 操作码（十六进制表示）
#[derive(Debug, Clone, Copy, PartialEq, Eq)]
enum Opcode {
    // 停止执行
    STOP = 0x00,
    
    // 算术运算（Stack: a b → result）
    ADD = 0x01,     // a + b
    MUL = 0x02,     // a * b
    SUB = 0x03,     // a - b
    DIV = 0x04,     // a / b
    MOD = 0x06,     // a % b
    EXP = 0x0a,     // a ^ b（指数运算，gas 随指数位数线性增长）
    
    // 比较运算
    LT = 0x10,      // a < b → 1 else 0
    GT = 0x11,      // a > b → 1 else 0
    EQ = 0x14,      // a == b → 1 else 0
    ISZERO = 0x15,  // a == 0 → 1 else 0
    
    // 位运算
    AND = 0x16,     // a & b
    OR = 0x17,      // a | b
    XOR = 0x18,     // a ^ b
    NOT = 0x19,     // ~a
    SHL = 0x1b,     // a << b
    SHR = 0x1c,     // a >> b
    
    // SHA3 (Keccak256) — 整个 EVM 的核心哈希函数
    SHA3 = 0x20,    // offset, size → hash
    
    // 环境信息
    ADDRESS = 0x30,      // 当前合约地址
    CALLER = 0x33,       // 合约调用者地址
    CALLVALUE = 0x34,    // 附带的 ETH 数量（单位: wei）
    TIMESTAMP = 0x42,    // 区块时间戳
    NUMBER = 0x43,       // 区块号
    CHAINID = 0x46,      // 链 ID
    
    // 栈操作
    POP = 0x50,          // 丢弃栈顶
    PUSH1 = 0x60,        // 压入 1 字节立即数（PUSH2..PUSH32 依次是 0x61..0x7f）
    
    DUP1 = 0x80,         // 复制栈顶第 1 个元素（DUP2..DUP16: 0x81..0x8f）
    SWAP1 = 0x90,        // 交换栈顶和第 2 个元素（SWAP2..SWAP16: 0x91..0x9f）
    
    // 内存操作
    MLOAD = 0x51,        // offset → value（从内存读取 32 字节）
    MSTORE = 0x52,       // offset, value（向内存写入 32 字节）
    MSTORE8 = 0x53,      // offset, value（向内存写入 1 字节）
    MSIZE = 0x59,        // 返回当前内存大小
    
    // 存储操作（最贵的 opcodes）
    SLOAD = 0x54,        // key → value（从持久化存储读取）
    SSTORE = 0x55,       // key, value（写入持久化存储）
    
    // 控制流
    JUMP = 0x56,         // 跳转到 dest（dest 必须是 JUMPDEST）
    JUMPI = 0x57,        // 条件跳转：dest, cond → if cond then jump dest
    JUMPDEST = 0x5b,     // 有效的跳转目标（标记）
    PC = 0x58,           // 当前程序计数器值
    
    // 返回数据
    RETURN = 0xf3,       // offset, size → 停止执行，返回内存中的一段数据
    REVERT = 0xfd,       // offset, size → 停止执行，回滚状态（但不消耗剩余 gas）
}

impl Opcode {
    /// 根据字节码确定对应的 opcode（处理 PUSH1..PUSH32 的特殊情况）
    fn from_byte(byte: u8) -> Option<Self> {
        match byte {
            0x00 => Some(Opcode::STOP),
            0x01 => Some(Opcode::ADD),
            0x02 => Some(Opcode::MUL),
            0x03 => Some(Opcode::SUB),
            0x04 => Some(Opcode::DIV),
            0x06 => Some(Opcode::MOD),
            0x0a => Some(Opcode::EXP),
            0x10 => Some(Opcode::LT),
            0x11 => Some(Opcode::GT),
            0x14 => Some(Opcode::EQ),
            0x15 => Some(Opcode::ISZERO),
            0x16 => Some(Opcode::AND),
            0x17 => Some(Opcode::OR),
            0x18 => Some(Opcode::XOR),
            0x19 => Some(Opcode::NOT),
            0x20 => Some(Opcode::SHA3),
            0x30 => Some(Opcode::ADDRESS),
            0x33 => Some(Opcode::CALLER),
            0x34 => Some(Opcode::CALLVALUE),
            0x42 => Some(Opcode::TIMESTAMP),
            0x43 => Some(Opcode::NUMBER),
            0x46 => Some(Opcode::CHAINID),
            0x50 => Some(Opcode::POP),
            0x51 => Some(Opcode::MLOAD),
            0x52 => Some(Opcode::MSTORE),
            0x53 => Some(Opcode::MSTORE8),
            0x54 => Some(Opcode::SLOAD),
            0x55 => Some(Opcode::SSTORE),
            0x56 => Some(Opcode::JUMP),
            0x57 => Some(Opcode::JUMPI),
            0x58 => Some(Opcode::PC),
            0x59 => Some(Opcode::MSIZE),
            0x5b => Some(Opcode::JUMPDEST),
            0xf3 => Some(Opcode::RETURN),
            0xfd => Some(Opcode::REVERT),
            0x60..=0x7f => Some(Opcode::PUSH1),  // PUSH1..PUSH32
            0x80..=0x8f => Some(Opcode::DUP1),     // DUP1..DUP16
            0x90..=0x9f => Some(Opcode::SWAP1),    // SWAP1..SWAP16
            _ => None,
        }
    }

    /// PUSH opcode 的字节数（PUSH1 → 1, PUSH32 → 32）
    fn push_size(op: u8) -> usize {
        if (0x60..=0x7f).contains(&op) {
            (op - 0x60 + 1) as usize
        } else {
            0
        }
    }
}
```

### 1.3 实现完整的 EVM 解释器

```rust
// toy_evm.rs —— 用 Rust 实现一个玩具 EVM 解释器（~150 行）
// 支持：算术、比较、栈操作、内存、存储、控制流、gas 追踪
// 可以用来执行简单的智能合约字节码

use std::collections::HashMap;

/// EVM 的 256-bit 字（在真实 EVM 中是 U256，这里用 u128 简化）
type Word = u128;

/// EVM 运行时状态
struct Evm {
    /// 栈（最大 1024 个元素，每个 256 bits）
    stack: Vec<Word>,

    /// 内存（按需扩展的字节数组）
    memory: Vec<u8>,

    /// 持久化存储（key → value）
    storage: HashMap<Word, Word>,

    /// 程序计数器（指向字节码中的当前位置）
    pc: usize,

    /// 字节码（要执行的 EVM 程序）
    code: Vec<u8>,

    /// 已消耗的 gas
    gas_used: u64,

    /// gas 上限
    gas_limit: u64,

    /// 是否已停止执行
    stopped: bool,

    /// 返回数据（RETURN/REVERT 的输出）
    return_data: Vec<u8>,

    /// 当前合约地址（模拟）
    address: Word,

    /// 调用者地址（模拟）
    caller: Word,

    /// 附带的 ETH 数量（模拟）
    call_value: Word,
}

/// 每个 opcode 的 gas 成本
fn gas_cost(op: Opcode) -> u64 {
    match op {
        Opcode::STOP | Opcode::JUMPDEST => 0,
        Opcode::ADD | Opcode::SUB | Opcode::MUL | Opcode::DIV | Opcode::MOD
        | Opcode::LT | Opcode::GT | Opcode::EQ | Opcode::ISZERO
        | Opcode::AND | Opcode::OR | Opcode::XOR | Opcode::NOT
        | Opcode::SHL | Opcode::SHR
        | Opcode::POP | Opcode::PC | Opcode::MSIZE
        | Opcode::PUSH1 | Opcode::DUP1 | Opcode::SWAP1 => 3,

        Opcode::MLOAD | Opcode::MSTORE | Opcode::MSTORE8
        | Opcode::JUMP | Opcode::JUMPI => 3,

        Opcode::EXP => 50,  // 基础成本 + 每个指数位额外成本
        Opcode::SHA3 => 30, // 基础成本 + 每个字节额外成本
        Opcode::SLOAD => 100,    // 热存储读（在同一个交易内）
        Opcode::SSTORE => {
            // SSTORE 成本非常复杂（取决于当前值/新值是否是零）
            // 冷存储写入: 20,000 gas
            // 热存储写入: 5,000 gas（覆盖已有值）
            // 清零退款: -15,000 gas（删除存储获得退款，上限是总 gas 的 1/5）
            20_000
        },
        Opcode::RETURN | Opcode::REVERT => 0,
        _ => 3,
    }
}

impl Evm {
    fn new(code: Vec<u8>, gas_limit: u64) -> Self {
        Self {
            stack: Vec::new(),
            memory: Vec::new(),
            storage: HashMap::new(),
            pc: 0,
            code,
            gas_used: 0,
            gas_limit,
            stopped: false,
            return_data: Vec::new(),
            address: 0xdead,
            caller: 0xbeef,
            call_value: 0,
        }
    }

    /// 消耗 gas（如果超出上限则 panic）
    fn use_gas(&mut self, amount: u64) {
        self.gas_used += amount;
        if self.gas_used > self.gas_limit {
            panic!("Out of gas! Used: {}, Limit: {}", self.gas_used, self.gas_limit);
        }
    }

    /// EVM 主循环：逐字节解释执行
    fn run(&mut self) -> Vec<u8> {
        while !self.stopped && self.pc < self.code.len() {
            let op_byte = self.code[self.pc];
            let opcode = Opcode::from_byte(op_byte)
                .unwrap_or_else(|| panic!("未知 opcode: 0x{:02x} at pc={}", op_byte, self.pc));

            self.use_gas(gas_cost(opcode));
            self.pc += 1;  // 默认前进 1 字节（PUSH/JUMP 等会自己调整）

            match opcode {
                // === 停止执行 ===
                Opcode::STOP => {
                    self.stopped = true;
                }

                // === 算术运算 ===
                Opcode::ADD => {
                    let (a, b) = self.pop_two();
                    self.stack.push(a.wrapping_add(b));
                    println!("  ADD: {} + {} = {}", a, b, a + b);
                }
                Opcode::MUL => {
                    let (a, b) = self.pop_two();
                    self.stack.push(a.wrapping_mul(b));
                }
                Opcode::SUB => {
                    let (a, b) = self.pop_two();
                    self.stack.push(a.wrapping_sub(b));
                }
                Opcode::DIV => {
                    let (a, b) = self.pop_two();
                    self.stack.push(if b == 0 { 0 } else { a / b });
                }

                // === 比较运算 ===
                Opcode::EQ => {
                    let (a, b) = self.pop_two();
                    self.stack.push(if a == b { 1 } else { 0 });
                }
                Opcode::ISZERO => {
                    let a = self.pop_one();
                    self.stack.push(if a == 0 { 1 } else { 0 });
                }
                Opcode::LT => {
                    let (a, b) = self.pop_two();
                    self.stack.push(if a < b { 1 } else { 0 });
                }
                Opcode::GT => {
                    let (a, b) = self.pop_two();
                    self.stack.push(if a > b { 1 } else { 0 });
                }

                // === 栈操作 ===
                // PUSH1..PUSH32: 压入 N 字节的立即数
                Opcode::PUSH1 => {
                    let n = Opcode::push_size(op_byte);
                    // 读取接下来 n 个字节作为大端序整数
                    let mut value: Word = 0;
                    for i in 0..n {
                        value = (value << 8) | (self.code[self.pc] as Word);
                        self.pc += 1;
                    }
                    self.stack.push(value);
                }
                Opcode::POP => {
                    self.pop_one();
                }
                Opcode::DUP1 => {
                    let dup_index = (op_byte - 0x80) as usize;
                    let value = self.stack[self.stack.len() - 1 - dup_index];
                    self.stack.push(value);
                }
                Opcode::SWAP1 => {
                    let swap_index = (op_byte - 0x90 + 1) as usize;
                    let len = self.stack.len();
                    self.stack.swap(len - 1, len - 1 - swap_index);
                }

                // === 内存操作 ===
                Opcode::MLOAD => {
                    let offset = self.pop_one() as usize;
                    self.expand_memory(offset + 32);
                    let mut value: Word = 0;
                    for i in 0..32 {
                        value = (value << 8) | (self.memory[offset + i] as Word);
                    }
                    self.stack.push(value);
                }
                Opcode::MSTORE => {
                    let (offset, value) = self.pop_two();
                    self.expand_memory(offset as usize + 32);
                    for i in 0..32 {
                        self.memory[offset as usize + 31 - i] = (value >> (i * 8)) as u8;
                    }
                }

                // === 存储操作 ===
                Opcode::SLOAD => {
                    let key = self.pop_one();
                    let value = self.storage.get(&key).copied().unwrap_or(0);
                    self.stack.push(value);
                }
                Opcode::SSTORE => {
                    let (key, value) = self.pop_two();
                    self.storage.insert(key, value);
                }

                // === 控制流 ===
                Opcode::JUMP => {
                    let dest = self.pop_one() as usize;
                    if dest < self.code.len() && self.code[dest] == 0x5b {
                        self.pc = dest;
                    } else {
                        panic!("无效的 JUMP 目标: {} (必须是 JUMPDEST)", dest);
                    }
                }
                Opcode::JUMPI => {
                    let (dest, cond) = self.pop_two();
                    if cond != 0 {
                        if (dest as usize) < self.code.len() && self.code[dest as usize] == 0x5b {
                            self.pc = dest as usize;
                        } else {
                            panic!("无效的 JUMPI 目标: {}", dest);
                        }
                    }
                }
                Opcode::JUMPDEST => {
                    // NOP —— 仅用作标记有效跳转位置
                }

                // === 环境信息 ===
                Opcode::Pc => {
                    self.stack.push(self.pc as Word);
                }
                Opcode::ADDRESS => {
                    self.stack.push(self.address);
                }
                Opcode::CALLER => {
                    self.stack.push(self.caller);
                }
                Opcode::CALLVALUE => {
                    self.stack.push(self.call_value);
                }

                // === 返回 ===
                Opcode::RETURN => {
                    let (offset, size) = self.pop_two();
                    self.expand_memory(offset as usize + size as usize);
                    self.return_data = self.memory[offset as usize..offset as usize + size as usize].to_vec();
                    self.stopped = true;
                }
                Opcode::REVERT => {
                    let (offset, size) = self.pop_two();
                    self.expand_memory(offset as usize + size as usize);
                    self.return_data = self.memory[offset as usize..offset as usize + size as usize].to_vec();
                    self.stopped = true;
                    println!("  REVERT: 执行回滚");
                }

                // === SHA3（Keccak256）===
                Opcode::SHA3 => {
                    let (offset, size) = self.pop_two();
                    self.expand_memory(offset as usize + size as usize);
                    let data = &self.memory[offset as usize..offset as usize + size as usize];
                    // 简化：用简单的哈希替代 Keccak256
                    let hash = self.simple_hash(data);
                    self.stack.push(hash);
                }

                _ => panic!("未实现的 opcode: {:?}", opcode),
            }
        }
        self.return_data.clone()
    }

    // === 辅助方法 ===

    fn pop_one(&mut self) -> Word {
        self.stack.pop().expect("栈下溢：尝试从空栈弹出")
    }

    fn pop_two(&mut self) -> (Word, Word) {
        let b = self.pop_one();
        let a = self.pop_one();
        (a, b)  // 注意顺序：EVM 中栈顶是第二个操作数
    }

    fn expand_memory(&mut self, required_size: usize) {
        // 内存按 32 字节为单位扩展
        let words = (required_size + 31) / 32;
        let new_size = words * 32;
        if new_size > self.memory.len() {
            self.memory.resize(new_size, 0u8);
        }
    }

    fn simple_hash(&self, data: &[u8]) -> Word {
        // 简化哈希（真实 EVM 用 Keccak-256）
        data.iter().fold(0u128, |acc, &b| acc.wrapping_mul(31).wrapping_add(b as u128))
    }
}

// ==================================================================
// 演示：执行一段简单的 EVM 字节码
// ==================================================================

fn main() {
    // 这段字节码做的事情：
    //   PUSH1 0x02    (压入 2)
    //   PUSH1 0x03    (压入 3)
    //   ADD           (2 + 3 = 5)
    //   PUSH1 0x07    (压入 7)
    //   MUL           (5 * 7 = 35)
    //   PUSH1 0x00    (存储位置 offset = 0)
    //   MSTORE        (将 35 存到内存位置 0)
    //   PUSH1 0x20    (返回数据大小 = 32 字节)
    //   PUSH1 0x00    (返回数据起始位置 = 0)
    //   RETURN        (返回内存[0..32]的内容，即 35)
    let bytecode = vec![
        0x60, 0x02,  // PUSH1 2
        0x60, 0x03,  // PUSH1 3
        0x01,        // ADD
        0x60, 0x07,  // PUSH1 7
        0x02,        // MUL
        0x60, 0x00,  // PUSH1 0 (offset)
        0x52,        // MSTORE
        0x60, 0x20,  // PUSH1 32 (size)
        0x60, 0x00,  // PUSH1 0 (offset)
        0xf3,        // RETURN
    ];

    println!("=== 玩具 EVM 解释器演示 ===");
    println!("字节码: (2 * 3 + 7) ... 实际执行了 PUSH1 2, PUSH1 3, ADD, PUSH1 7, MUL = 2+3*7=35\n");

    let mut evm = Evm::new(bytecode, 100_000);
    let result = evm.run();

    println!("\n执行完成！");
    println!("  gas 消耗: {}", evm.gas_used);
    println!("  栈深度: {}", evm.stack.len());
    println!("  返回值长度: {} 字节", result.len());
    if result.len() >= 32 {
        let mut value: u128 = 0;
        for &b in &result[..32] {
            value = (value << 8) | (b as u128);
        }
        println!("  计算结果: {}", value);  // 预期: 35 = (2+3)*7
    }
    println!("  ✓ 你已经实现了一个 EVM 解释器！");
}
```

**关键理解：** 这个 ~150 行的 EVM 解释器包含了 EVM 的核心逻辑：(1) 基于栈的运算模型，(2) PUSH 立即数压栈，(3) 内存和存储的分离（易失 vs 持久），(4) JUMP/JUMPI 控制流，(5) Gas 消耗追踪。真实 EVM 比这多约 100 个 opcode（外部调用、日志、创建合约等），但核心架构完全相同。你可以在 10 分钟内理解 Gas 定价的经济学：每个 SSTORE 消耗 20,000 gas —— 如果 gas price 是 20 gwei，那么写入一个存储槽的成本是 400,000 gwei ≈ 0.0004 ETH ≈ 1.2 美元（按 ETH 3,000 美元计）。

## 2. Alloy 生态：Rust 原生的以太坊工具链

Alloy 是 Paradigm（顶级加密风险投资公司）主导的 Rust 以太坊工具栈，替代了过去基于 `ethers-rs` 的生态。它的设计哲学是"类型安全、零成本抽象、完全 Rust"。

### 2.1 Provider 与连接

## 5. MEV（最大可提取价值）

### 5.1 什么是 MEV？

MEV 是最令人着迷也最具争议的区块链现象。它的核心定义：**区块生产者（及 searchers）通过在区块中插入、剔除或重排交易而获得的最大利润**。

```
┌─────────────────────────────────────────────────────────────────┐
│  MEV 的三种经典形式                                               │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  1. 三明治攻击（Sandwich Attack）                                 │
│     Mempool:                                                      │
│       → 受害者想用 100 ETH 在 Uniswap 上买某种代币               │
│       → Searcher 看到这笔交易，构造两笔自己的交易：                 │
│         前置交易：买同样的代币（推高价格）                        │
│         受害交易：受害者以高价买入（产生滑点）                     │
│         后置交易：卖出（以更高价格卖出获利）                      │
│       → 三笔交易排序: 前置 → 受害 → 后置                         │
│                                                                 │
│  2. 套利（Arbitrage）                                            │
│     → 同一代币在 Uniswap 和 SushiSwap 上价格不同                  │
│     → Searcher 在便宜的池买入，贵的池卖出                          │
│     → 利润 = 价差 - gas 费                                        │
│                                                                 │
│  3. 清算（Liquidation）                                          │
│     → Aave/Compound 中有用户的抵押仓位跌破清算线                   │
│     → 任何人都可以清算该仓位 → 获得清算奖励（通常 5-10%）          │
│     → 竞争：第一个发送清算交易的 searcher 获得奖励                │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 5.2 如何保护用户免受 MEV 侵害

/// 策略1：滑点限制（Slippage Protection）
fn apply_slippage_limit(
    expected_output: u128,
    max_slippage_bps: u16,  // basis points, 50 = 0.5%
) -> u128 {
    // 设置最小输出金额
    // 如果实际输出 < min_output → 交易回滚
    let min_output = expected_output * (10000 - max_slippage_bps as u128) / 10000;
    println!("  预期输出: {}, 最小输出: {} (滑点限制: {} bps)",
             expected_output, min_output, max_slippage_bps);
    min_output
}

/// 解决"交易在 mempool 中被看到 → 被 front-run"的问题

/// Searcher 无法在 commit 阶段知道你的完整交易内容 → 无法 front-run

    .execute(pool)
    .await?;

    println!("✓ 数据库表已创建（erc20_transfers）");
    Ok(())
}

/// 插入一条 Transfer 记录（忽略重复的 tx_hash + log_index 组合）
async fn insert_transfer(
    pool: &SqlitePool,
    record: &TransferRecord,
) -> Result<(), sqlx::Error> {
    sqlx::query(
        r#"
        INSERT OR IGNORE INTO erc20_transfers 
            (tx_hash, block_number, block_timestamp, log_index, 
             contract_address, from_address, to_address, value)
        VALUES (?, ?, ?, ?, ?, ?, ?, ?)
        "#,
    )
    .bind(&record.tx_hash)
    .bind(record.block_number)
    .bind(record.block_timestamp)
    .bind(record.log_index)
    .bind(&record.contract_address)
    .bind(&record.from_address)
    .bind(&record.to_address)
    .bind(&record.value)
    .execute(pool)
    .await?;
    Ok(())
}

/// 查询某个地址最近的转账记录
async fn query_transfers_by_address(
    pool: &SqlitePool,
    address: &str,
    limit: i64,
) -> Result<Vec<TransferRecord>, sqlx::Error> {
    sqlx::query_as::<_, TransferRecord>(
        r#"
        SELECT * FROM erc20_transfers
        WHERE from_address = ? OR to_address = ?
        ORDER BY block_number DESC, log_index DESC
        LIMIT ?
        "#,
    )
    .bind(address)
    .bind(address)
    .bind(limit)
    .fetch_all(pool)
    .await
}

/// 索引器状态
struct IndexerState {
    pool: SqlitePool,
    last_processed_block: Mutex<i64>,
    total_transfers_indexed: Mutex<u64>,
}

/// 启动实时索引（监听新区块，解析 Transfer 事件）
async fn start_indexing(state: Arc<IndexerState>, ws_url: &str) -> Result<(), Box<dyn std::error::Error>> {
    let ws = WsConnect::new(ws_url);
    let provider = ProviderBuilder::new().on_ws(ws).await?;

    // 订阅新区块通知
    let sub = provider.subscribe_blocks().await?;
    let mut stream = sub.into_stream();

    println!("开始监听新区块...");

    while let Some(block) = stream.next().await {
        let block_number = block.header.number.unwrap() as i64;
        let timestamp = block.header.timestamp as i64;

        println!("处理区块 #{} ({} 笔交易)", block_number, block.transactions.len());

        let mut block_transfer_count = 0u64;

        // 遍历区块中的每笔交易
        for tx in &block.transactions {
            let tx_hash = format!("{:?}", tx.inner.tx_hash());

            // 获取交易收据（包含事件日志）
            if let Ok(Some(receipt)) = provider.get_transaction_receipt(*tx.inner.tx_hash()).await {
                for (log_index, log) in receipt.inner.logs().iter().enumerate() {
                    // 尝试解码 Transfer 事件
                    if let Ok(decoded) = log.log_decode::<Transfer>() {
                        let event_data = decoded.data();

                        let record = TransferRecord {
                            id: 0,  // 由 SQLite 自动分配
                            tx_hash: tx_hash.clone(),
                            block_number,
                            block_timestamp: timestamp,
                            log_index: log_index as i64,
                            contract_address: format!("{:?}", log.address()),
                            from_address: format!("{:?}", event_data.from),
                            to_address: format!("{:?}", event_data.to),
                            value: format!("{}", event_data.value),
                        };

                        // 写入数据库
                        if let Err(e) = insert_transfer(&state.pool, &record).await {
                            eprintln!("数据库写入失败: {}", e);
                        } else {
                            block_transfer_count += 1;
                        }
                    }
                }
            }
        }

        // 更新索引状态
        {
            let mut last_block = state.last_processed_block.lock().await;
            *last_block = block_number;
            let mut total = state.total_transfers_indexed.lock().await;
            *total += block_transfer_count;
        }

        if block_transfer_count > 0 {
            println!("  → 本区块索引了 {} 条 Transfer 事件", block_transfer_count);
        }
    }

    Ok(())
}

/// GraphQL 类型：Transfer 的查询结构
#[derive(SimpleObject)]
struct TransferGql {
    tx_hash: String,
    block_number: i64,
    block_timestamp: i64,
    contract_address: String,
    from_address: String,
    to_address: String,
    value: String,

impl From<TransferRecord> for TransferGql {
    fn from(r: TransferRecord) -> Self {
        Self {
            tx_hash: r.tx_hash,
            block_number: r.block_number,
            block_timestamp: r.block_timestamp,
            contract_address: r.contract_address,
            from_address: r.from_address,
            to_address: r.to_address,
            value: r.value,
        }
    }
}

/// GraphQL 查询根节点
struct QueryRoot;

#[Object]
impl QueryRoot {
    /// 查询某个地址的所有转账记录
    async fn transfers<'ctx>(
        &self,
        ctx: &Context<'ctx>,
        address: String,
        #[graphql(default = 20)] limit: i64,
    ) -> async_graphql::Result<Vec<TransferGql>> {
        let pool = ctx.data::<SqlitePool>()?;
        let records = query_transfers_by_address(pool, &address, limit)
            .await
            .map_err(|e| async_graphql::Error::new(format!("数据库查询失败: {}", e)))?;

        Ok(records.into_iter().map(TransferGql::from).collect())
    }

    /// 获取索引器状态
    async fn indexer_stats<'ctx>(
        &self,
        ctx: &Context<'ctx>,
    ) -> async_graphql::Result<String> {
        let state = ctx.data::<Arc<IndexerState>>()?;
        let last_block = *state.last_processed_block.lock().await;
        let total = *state.total_transfers_indexed.lock().await;
        Ok(format!(
            "最后处理的区块: {}, 累计索引 Transfer: {}",
            last_block, total
        ))
    }
}

#[tokio::main]
async fn main() -> Result<(), Box<dyn std::error::Error>> {
    println!("╔═══════════════════════════════════════════════════════╗");
    println!("║       以太坊事件索引器 (Rust + SQLite + GraphQL)       ║");
    println!("╚═══════════════════════════════════════════════════════╝\n");

    // 步骤1: 初始化数据库
    let pool = SqlitePoolOptions::new()
        .max_connections(5)
        .connect("sqlite:ethereum_indexer.db")
        .await?;
    init_db(&pool).await?;

    // 步骤2: 创建索引器状态
    let state = Arc::new(IndexerState {
        pool: pool.clone(),
        last_processed_block: Mutex::new(0),
        total_transfers_indexed: Mutex::new(0),
    });

    // 步骤3: 构建 GraphQL Schema
    let schema = Schema::build(QueryRoot, MutationRoot, async_graphql::EmptySubscription)
        .data(pool.clone())
        .data(state.clone())
        .finish();

    // 步骤4: 启动 GraphQL 服务器（另一个任务）
    let schema_for_server = schema.clone();
    tokio::spawn(async move {
        use async_graphql_axum::{GraphQLRequest, GraphQLResponse};
        use axum::{Router, routing::post, extract::Extension};
        
        async fn graphql_handler(
            schema: Extension<Schema<QueryRoot, MutationRoot, async_graphql::EmptySubscription>>,
            req: GraphQLRequest,
        ) -> GraphQLResponse {
            schema.execute(req.into_inner()).await.into()
        }

        let app = Router::new()
            .route("/graphql", post(graphql_handler))
            .layer(Extension(schema_for_server));

        println!("GraphQL 服务器启动于 http://localhost:8000/graphql");
        let listener = tokio::net::TcpListener::bind("0.0.0.0:8000").await.unwrap();
        axum::serve(listener, app).await.unwrap();
    });

    // 步骤5: 启动区块链索引（需要有效的 WebSocket RPC URL）
    // 实际运行需要替换为真实的 WebSocket 端点：
    // let ws_url = "wss://eth-mainnet.g.alchemy.com/v2/YOUR_KEY";
    // start_indexing(state, ws_url).await?;

    println!("\n提示：替换 ws_url 为有效的 WebSocket 端点后，索引器将开始工作。");
    println!("然后访问 http://localhost:8000/graphql 查询数据。");
    println!("示例 GraphQL 查询：");
    println!("  query { transfers(address: \"0x...\", limit: 10) { tx_hash value from_address to_address } }");

    // 保持主任务运行
    tokio::signal::ctrl_c().await?;
    println!("\n索引器已停止。");
    Ok(())
}
