# 10.0 领域深化概述：选择你的战场

> Phase 1-6 让你精通 Rust 语言本身。Phase 7-8 让你理解编译器和生态的设计。Phase 9 让你成为开源贡献者。现在的问题是：**你想把 Rust 用到哪里去？** 这是 Phase 10 要回答的问题——不是"再学点什么"，而是"选一个方向，深耕下去"。本 Phase 提供三条赛道，每条都足够深、足够真、足够让你在就业市场或创业赛道中脱颖而出。

---

## 1. Phase 10 的定位：从 "T 型人才" 的横杠到竖杠

```
T 型人才模型：

横杠：广度 — Phase 1-9 已经覆盖
  → Rust 语言全特性、编译器内部、生态关键 crate、开源协作
  → 这让你成为一个"全面的 Rust 开发者"

竖杠：深度 — Phase 10 要建立的
  → 选择一个领域，把横杠上的某个点往下深钻
  → 这让你从"会用 Rust 的人"变成"用 Rust 解决某领域最难问题的人"

就业市场的现实：
  → "Rust 开发者"的岗位很少——但"用 Rust 做量化交易"的岗位更稀缺也更值钱
  → 通用 Rust 技能 = 没有差异化。领域专精 + Rust 技能 = 不可替代。
```

Phase 10 提供**三个方向**，你可以选一个深耕，也可以两个组合：

| 方向 | 定位 | 目标 | 课数 | 起点要求 |
|------|------|------|------|----------|
| **Track A: 性能工程** | 就业通用技能 | 性能工程师 / 系统优化专家 | 6 课 (10.1-10.6) | 已完成 Phase 4/6 性能课程 |
| **Track B: 量化金融** | 就业专精方向 | Quant Dev / 交易系统工程师 | 7 课 (10.7-10.13) | 零金融知识要求，强 Rust 基础 |
| **Track C: Web3** | 创业方向 | 区块链产品开发者 | 6 课 (10.14-10.19) | 零区块链知识要求，强 Rust 基础 |

## 4. Track B 前置依赖：零金融知识，全 Rust 硬核

Track B（量化金融）不需要你提前懂金融。每课都是"先讲领域概念，再写 Rust 实现"的并行教学。

```
你需要会的（来自 Phase 1-6）：
  → Rust 基础：所有权、生命周期、trait、泛型（Phase 1-2）
  → 并发：Arc/Mutex、Channel、Atomic、tokio（Phase 3）
  → 工程：错误处理、测试、性能剖析（Phase 6）
  → 底层：FFI、自定义分配器、SIMD（Phase 4，选学但有用）

你不需要会的（Track B 从零教）：
  → 什么是 bid/ask/spread/liquidity
  → 什么是 order book 和 matching engine
  → 什么是 FIX 协议
  → 什么是 backtesting 和 strategy
  → 什么是 market data feed

Track B 的哲学：
  "不会金融不丢人——Rust 程序员进入量化领域的第一步永远是理解领域概念。
   好消息是：大部分量化问题本质上都是数据结构+并发+性能优化问题，
   而你已经会了 Rust 的部分。"
```

## 5. Track C 前置依赖：零区块链知识，产品思维导向

Track C（Web3）不需要你提前懂区块链或密码学。每课从第一性原理出发，用 Rust 构建理解。

```
你需要会的（来自 Phase 1-6）：
  → Rust 基础：所有权、trait、泛型（Phase 1-2）
  → 并发与网络：tokio、channel（Phase 3）
  → 工程：API 设计、错误处理、CICD（Phase 6）
  → 可选加分：FFI（Phase 4），宏（Phase 5）

你不需要会的（Track C 从零教）：
  → 什么是 hash、签名、Merkle tree
  → 什么是 account 模型 vs UTXO 模型
  → 什么是共识机制（PoW/PoS）
  → 什么是 Solana program / Anchor 框架
  → 什么是 EVM / alloy / revm
  → 什么是零知识证明 / R1CS / PLONK

Track C 的哲学：
  "区块链看起来很神秘——但本质上，它是一组密码学原语+分布式协议的组合。
   用 Rust 从零实现它们，神秘感就会消失。"
```

## 6. 推荐学习路径

### 组合拳

```
推荐组合 1: A + B（就业最强组合）
  Track A (性能工程) → Track B (量化金融)
  为什么：量化金融是性能优化最极致的应用场景。
         先学通用的性能方法论，再应用到金融领域，事半功倍。
  目标岗位：Quantitative Developer @ Hedge Fund / Prop Trading Firm

推荐组合 2: A + C（全栈 Web3 开发者）
  Track A (性能工程) → Track C (Web3)
  为什么：Web3 基础设施（节点、索引器、MEV bot）对性能敏感。
         性能工程能力让你在 Web3 领域有不可替代的竞争优势。
  目标岗位：Blockchain Infrastructure Engineer / Web3 Founding Engineer

推荐组合 3: 仅 C（纯创业者路径）
  Track C (Web3) 单独
  为什么：如果你想快速进入产品开发，不深入底层优化。
         先用 Anchor 写 Solana 程序，用 alloy 做以太坊交互。
  目标：Technical Co-founder / Solo Web3 Builder

推荐组合 4: 全栈（不推荐，除非你有大量时间）
  A → B → C 全部
  为什么：20 课 × 每课预计 2-4 小时 = 40-80 小时。
         如果你有时间且有决心，这是最全面的路径。
  目标：成为 Rust 领域 T 型人才的最完整形态。
```

## 8. 热身：了解你的"战场"

在进入具体轨道之前，花 10 分钟做三件事：

### 8.1 明确你的"为什么"

```
拿一张纸，写下：

Track A（性能工程）吸引我的点：_________________
  → 是喜欢"把东西做到极限"的感觉？
  → 还是想拿性能工程师的高薪？
  → 还是两者都有？

Track B（量化金融）吸引我的点：_________________
  → 是对金融市场感兴趣？
  → 还是被收入吸引？
  → 如果你对金融本身无感——Track B 可能不适合你

Track C（Web3）吸引我的点：_________________
  → 是相信去中心化的理念？
  → 还是想抓住 Web3 的创业机会？
  → 如果你只是"觉得 Web3 很酷"但没有产品想法——Track C 仍然值得学
```

### 8.2 快速自检：你的 Rust 够强吗？

```rust
// 如果你能解释以下所有代码——你的 Rust 基础足够进入 Phase 10

use std::sync::Arc;
use std::cell::UnsafeCell;
use std::ops::Deref;

// 1. 这个 trait bound 意味着什么？
fn process<T: Send + Sync + 'static>(data: T) -> Arc<T> {
    Arc::new(data)
}

// 2. 为什么这样写是 unsafe 的？需要满足什么 invariant？
struct MyCell<T> {
    value: UnsafeCell<T>,
}

// 3. 这个 async block 捕获了什么？生命周期多长？
async fn fetch_and_process(url: &str) -> String {
    let body = reqwest::get(url).await.unwrap().text().await.unwrap();
    tokio::task::spawn_blocking(move || {
        // body 在这里能用吗？为什么？
        body.to_uppercase()
    }).await.unwrap()
}
```

如果有疑问——回到 Phase 1-3 复习基础。Phase 10 假设你能流畅地用 Rust 表达复杂想法。

### 8.3 规划时间

```
每课预计投入：2-4 小时（阅读 + 理解 + 动手）
每个 Track 的 Capstone：额外 4-8 小时（综合实战）

Track A 完整时间：6 课 × 3h + 6h capstone = ~24 小时
Track B 完整时间：7 课 × 3h + 6h capstone = ~27 小时
Track C 完整时间：6 课 × 3h + 6h capstone = ~24 小时

建议节奏：
  → 每周 2-3 课（如果你有全职工作）
  → 每周 5-6 课（如果你全职学习）
  → 一个 Track 大约需要 3-6 周完成

不要赶——这是 Phase 10，不是 Phase 1。
你不再是"学习 Rust"，你是在"用 Rust 成为某领域的专家"。
慢就是快。
```
