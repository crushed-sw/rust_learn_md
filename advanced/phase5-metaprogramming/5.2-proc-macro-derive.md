# 5.2 Derive 宏 —— 自动生成代码的艺术

> 过程宏让你像写普通 Rust 代码一样操作 Rust 代码——不再是以字符串拼接的方式生成代码，而是通过 syn 解析出完整的 AST，用 quote 将 Rust 语法树"还原"为代码。这一转变的意义如同从汇编跳转到高级语言：你不再纠结于 token 的顺序，而是操作有结构、有类型的抽象语法树。

---

## 1. proc-macro crate 的隔离世界

### 1.1 为什么过程宏必须在独立的 crate 中？

过程宏有一个硬性约束：**必须放在 `proc-macro = true` 的独立 crate 中**。这不是 Rust 的"设计偏好"，而是编译器架构的限制。

```toml
# my-derive/Cargo.toml
[lib]
proc-macro = true

[dependencies]
syn = { version = "2", features = ["full"] }
quote = "1"
proc-macro2 = "1"
```

原因是 Rust 的编译模型：过程宏不是"展开后丢弃"的元编程，而是**编译好的动态库**，编译器在运行时加载它并调用它。过程宏 crate 被编译成一个 `.so` / `.dll` / `.dylib` 文件，编译器进程在编译其他 crate 时动态加载这个库，调用其中的 `#[proc_macro_derive]` 函数。

```
普通的编译过程：
  源码 → AST → HIR → MIR → LLVM IR → 机器码
                         ↑
                   所有阶段在编译器进程内

过程宏的编译过程：
  my-derive 源码 → 编译成 .dylib (libmy_derive.so)
                                    ↓
  user crate 源码 → 编译器加载 my_derive.so
                   调用 my_derive 函数 → 获得 TokenStream
                   将生成的 TokenStream 合并到 AST 中
                   继续正常编译流程
```

### 1.2 两个世界的隔离

`proc_macro` crate（编译器提供的）和 `proc-macro2` crate（社区提供的）是两个不同的世界：

| | `proc_macro` | `proc-macro2` |
|:---|:---|:---|
| **来源** | 编译器内置，rustc 提供 | crates.io 上的库 |
| **TokenStream 类型** | `proc_macro::TokenStream` | `proc_macro2::TokenStream` |
| **可用范围** | **仅** proc-macro crate | 任何 crate（包括测试） |
| **Span** | 编译器内部 Span | 可模拟的 Span，可用于测试 |
| **调试支持** | 无（编译器内部类型） | 可通过 feature 开启 Debug/Display |

**为什么需要 proc-macro2？** 因为 `proc_macro` 只能在 `proc-macro = true` 的 crate 中使用——你不能在单元测试里导入它，不能构建 AST 在测试中验证。proc-macro2 解决了这个问题：它提供了完全兼容的 TokenStream 类型，加上丰富的 Span 操作 API。

### 1.3 典型项目结构

```
my-derive/
├── Cargo.toml       # proc-macro = true, 依赖 syn/quote/proc-macro2
├── src/
│   └── lib.rs       # #[proc_macro_derive(...)] —— 入口函数

my-derive-core/
├── Cargo.toml       # 普通 lib crate，定义共享 trait
├── src/
│   └── lib.rs       # pub trait MyDerive { ... } —— 被 derive 宏生成的代码调用

user-crate/
├── Cargo.toml       # 依赖 my-derive 和 my-derive-core
├── src/
│   └── main.rs      # #[derive(MyDerive)]
```

这种分离不是可选的——**trait 的定义必须在非 proc-macro crate 中**，因为 proc-macro crate 只能导出 `proc_macro::TokenStream`，不能导出 trait、类型等正常 Rust 项。把 trait 放在 `-core` crate 里，两边都能引用它。

```
┌─────────────────────┐      依赖       ┌───────────────────────┐
│   my-derive         │ ──────────────→ │   my-derive-core      │
│   (proc-macro)      │                 │   (normal lib)        │
│   生成 impl 代码     │                 │   定义 trait           │
└─────────────────────┘                 └───────────────────────┘
          │                                        ↑
          │ 编译时被加载                            │ 依赖
          ↓                                        │
┌─────────────────────┐                           │
│   user-crate        │ ──────────────────────────┘
│   (normal binary)   │
│   #[derive(MyDerive)]│
└─────────────────────┘
```

## 2. #[proc_macro_derive] 入口

### 2.1 函数签名

```rust
// my-derive/src/lib.rs
use proc_macro::TokenStream;

#[proc_macro_derive(HelloMacro)]
pub fn derive_hello(input: TokenStream) -> TokenStream {
    // input 是用户标注的 struct/enum 的完整 token 流
    // 例如：struct Foo { x: i32, y: String }
    //        ↑ 这一整段都作为 input 传入

    // 在这里解析、转换、生成新代码
    // 返回的 TokenStream 会被拼接到用户代码中
    TokenStream::new()  // 空输出
}
```

`input` 的内容是**整个 item**（struct 或 enum 的定义），包括属性、可见性、泛型、所有字段。注意：**不是你标注的那一行 `#[derive(HelloMacro)]`**，而是标注所在的 `struct` 定义。

### 2.2 proc_macro → proc_macro2 桥接

在入口函数中，你几乎总是立刻做两件事：

```rust
use proc_macro::TokenStream;

#[proc_macro_derive(HelloMacro)]
pub fn derive_hello(input: TokenStream) -> TokenStream {
    // 第一步：proc_macro::TokenStream → proc_macro2::TokenStream
    let input = proc_macro2::TokenStream::from(input);

    // 第二步：用 syn 解析
    // ...

    // 最后：proc_macro2::TokenStream → proc_macro::TokenStream
    TokenStream::from(output)
}
```

**为什么需要这两步转换？** 因为 syn/quote 都使用 `proc_macro2::TokenStream`，而入口函数签名要求 `proc_macro::TokenStream`。这两个类型通过 `.into()` / `From` trait 互相转换，开销几乎为零。

### 2.3 `attributes(helper_attr)` —— 捕获字段级属性

```rust
// 声明：除了 Derive 宏本身，还捕获 #[my_helper] 属性
#[proc_macro_derive(MyTrait, attributes(my_helper))]
pub fn derive_my_trait(input: TokenStream) -> TokenStream {
    // 现在，在 struct 或字段上的 #[my_helper(...)] 都会被传给 input
    // 不会被编译器当作"未知属性"而报错
}

// 用户侧代码：
#[derive(MyTrait)]
struct User {
    name: String,
    #[my_helper(skip)]      // ← 这个属性会被保留在 AST 中
    password: String,
    #[my_helper(rename = "email_address")]
    mail: String,
}
```

**为什么需要 `attributes(...)`？** 因为 Rust 编译器默认会把不认识的自定义属性当作错误删除（或至少发出 warning）。在 `attributes(...)` 中声明后，编译器知道"这个属性是给 derive 宏用的"，会把它保留在 token 流中。

**一个 derive 宏可以捕获多个属性：**

```rust
#[proc_macro_derive(Builder, attributes(builder, builder_default))]
pub fn derive_builder(input: TokenStream) -> TokenStream { ... }
```

### 2.4 错误处理：唯一的信号路径

过程宏不能返回 `Result`——函数签名就是 `TokenStream -> TokenStream`。有三种方式处理错误：

```rust
// 方式 1：返回 compile_error! token 流（推荐）
use proc_macro::TokenStream;
use quote::quote;

#[proc_macro_derive(MyTrait)]
pub fn derive_my_trait(input: TokenStream) -> TokenStream {
    let input = syn::parse_macro_input!(input as syn::DeriveInput);

    // 只允许 struct
    let _struct_data = match &input.data {
        syn::Data::Struct(s) => s,
        _ => {
            return quote! {
                compile_error!("MyTrait only supports structs");
            }.into();
        }
    };

    // ...
    TokenStream::new()
}

// 方式 2：通过 syn::Error 发射到编译器诊断系统（推荐）
#[proc_macro_derive(MyTrait)]
pub fn derive_my_trait(input: TokenStream) -> TokenStream {
    let input = syn::parse_macro_input!(input as syn::DeriveInput);

    let name = &input.ident;

    // 如果出现问题，把错误转为 compile_error!
    match derive_impl(&input) {
        Ok(tokens) => tokens.into(),
        Err(e) => e.to_compile_error().into(),
    }
}

fn derive_impl(input: &syn::DeriveInput) -> syn::Result<proc_macro2::TokenStream> {
    // 使用 ? 传播 syn::Error
    let _ = input.ident;
    Err(syn::Error::new_spanned(
        input,
        "derivation failed for some reason",
    ))
}

// 方式 3：panic（最粗暴，缺点是无法提供漂亮的诊断信息）
#[proc_macro_derive(MyTrait)]
pub fn derive_my_trait(input: TokenStream) -> TokenStream {
    panic!("not implemented");
}
```

### 2.5 最小的 "Hello World" derive 宏

```rust
// my-derive/Cargo.toml
// [lib]
// proc-macro = true
// [dependencies]
// syn = { version = "2", features = ["full"] }
// quote = "1"

use proc_macro::TokenStream;
use quote::quote;
use syn::{parse_macro_input, DeriveInput};

#[proc_macro_derive(HelloMacro)]
pub fn derive_hello(input: TokenStream) -> TokenStream {
    let input = parse_macro_input!(input as DeriveInput);
    let name = &input.ident;  // 类型名称，如 Foo, Bar

    let expanded = quote! {
        impl HelloMacro for #name {
            fn hello() {
                println!("Hello, my type name is {}!", stringify!(#name));
            }
        }
    };

    TokenStream::from(expanded)
}
```

用户侧：

```rust
trait HelloMacro {
    fn hello();
}

#[derive(HelloMacro)]
struct World;

fn main() {
    World::hello();  // 输出：Hello, my type name is World!
}
```

## 3. syn 解析：TokenStream → AST

### 3.1 `syn::parse_macro_input!` —— 解析的正确姿势

```rust
use proc_macro::TokenStream;
use syn::{parse_macro_input, DeriveInput};

#[proc_macro_derive(MyTrait)]
pub fn derive_my_trait(input: TokenStream) -> TokenStream {
    // 这行代码做了三件事：
    // 1. 把 proc_macro::TokenStream 转为 proc_macro2::TokenStream
    // 2. 尝试解析为 DeriveInput
    // 3. 如果解析失败，自动生成 compile_error! 并返回
    let input = parse_macro_input!(input as DeriveInput);

    // input 现在是一个结构化的 AST 节点
    // ...
    TokenStream::new()
}
```

`parse_macro_input!(input as DeriveInput)` 等价于：

```rust
let input = match syn::parse::<DeriveInput>(input.into()) {
    Ok(parsed) => parsed,
    Err(err) => return err.to_compile_error().into(),
};
```

### 3.2 DeriveInput 结构全景

`DeriveInput` 是 derive 宏中最核心的类型——它代表一个顶层 item（struct 或 enum）的完整语法树。

```rust
pub struct DeriveInput {
    pub attrs: Vec<Attribute>,       // 属性：#[...]
    pub vis: Visibility,             // 可见性：pub, pub(crate), ...
    pub ident: Ident,                // 标识符：struct Foo ← 这个 Foo
    pub generics: Generics,          // 泛型参数：<T: Debug, const N: usize>
    pub data: Data,                  // 数据体：Struct / Enum / Union
}

pub enum Data {
    Struct(DataStruct),    // struct S { ... } 或 struct S(...);
    Enum(DataEnum),        // enum E { ... }
    Union(DataUnion),      // union U { ... }
}

pub struct DataStruct {
    pub struct_token: Token![struct],
    pub fields: Fields,    // 字段集合
    pub semi_token: Option<Token![;]>,  // 单元结构体末尾的分号
}

pub enum Fields {
    Named(FieldsNamed),       // struct S { a: i32, b: String }
    Unnamed(FieldsUnnamed),   // struct S(i32, String);
    Unit,                     // struct S;
}

pub struct FieldsNamed {
    pub brace_token: token::Brace,
    pub named: Punctuated<Field, Token![,]>,
}

pub struct FieldsUnnamed {
    pub paren_token: token::Paren,
    pub unnamed: Punctuated<Field, Token![,]>,
}

pub struct Field {
    pub attrs: Vec<Attribute>,       // 字段上的属性
    pub vis: Visibility,             // 字段的可见性
    pub mutability: FieldMutability,
    pub ident: Option<Ident>,        // 字段名（元组结构体字段无名字）
    pub colon_token: Option<Token![:]>,
    pub ty: Type,                    // 字段类型
}
```

### 3.3 解析不同类型

```rust
use syn::{DeriveInput, Data, Fields};

fn process_derive_input(input: &DeriveInput) {
    match &input.data {
        Data::Struct(data_struct) => {
            match &data_struct.fields {
                // 具名字段：struct Point { x: i32, y: i32 }
                Fields::Named(fields_named) => {
                    for field in &fields_named.named {
                        let field_name = &field.ident;  // Some("x"), Some("y")
                        let field_type = &field.ty;
                        println!("named field: {:?}: {}", field_name, quote!(#field_type));
                    }
                }
                // 元组字段：struct Point(i32, i32);
                Fields::Unnamed(fields_unnamed) => {
                    for (i, field) in fields_unnamed.unnamed.iter().enumerate() {
                        let field_type = &field.ty;
                        println!("unnamed field {}: {}", i, quote!(#field_type));
                    }
                }
                // 单元结构体：struct Unit;
                Fields::Unit => {
                    println!("unit struct");
                }
            }
        }
        Data::Enum(data_enum) => {
            for variant in &data_enum.variants {
                let variant_name = &variant.ident;
                match &variant.fields {
                    Fields::Named(fields) => { /* 枚举变体有具名字段 */ }
                    Fields::Unnamed(fields) => { /* 枚举变体有元组字段 */ }
                    Fields::Unit => { /* 枚举变体无字段 */ }
                }
            }
        }
        Data::Union(data_union) => {
            // union 的处理
            for field in &data_union.fields.named {
                // ...
            }
        }
    }
}
```

### 3.4 Ident 和 Type —— 最常用的 syn 类型

```rust
use syn::{Ident, Type, TypePath, Path};

// Ident: 标识符——变量名、类型名、函数名
let ident: Ident = parse_quote!(my_variable);
// ident.to_string() == "my_variable"

// Type: 类型——i32, Vec<String>, &'a str
let ty: Type = parse_quote!(Vec<String>);

// TypePath: 路径类型——std::collections::HashMap
let type_path: TypePath = parse_quote!(std::collections::HashMap<K, V>);

// Path: 路径表达式——std::collections::HashMap::new
let path: Path = parse_quote!(std::collections::HashMap::new);
```

`parse_quote!` 是一个极其有用的宏（来自 syn），它让你可以用 Rust 语法直接构造 syn 的 AST 节点。

## 4. quote 生成：AST → TokenStream

### 4.1 `quote!` 的基本使用

`quote!` 是 derive 宏的"输出端"——syn 负责解析，quote 负责生成。它提供了一个模板语言，让你可以混合"静态的 Rust 代码"和"动态的 AST 节点"。

```rust
use quote::quote;

let name = syn::Ident::new("World", proc_macro2::Span::call_site());

// 静态部分直接写 Rust 代码
// 动态部分用 #variable 插值
let generated = quote! {
    impl HelloMacro for #name {
        fn hello() {
            println!("Hello, {}!", stringify!(#name));
        }
    }
};
```

生成的代码等价于：

```rust
impl HelloMacro for World {
    fn hello() {
        println!("Hello, {}!", "World");
    }
}
```

### 4.2 插值完整清单

```rust
use quote::quote;

let ident = syn::parse_str::<syn::Ident>("MyStruct").unwrap();
let ty = syn::parse_str::<syn::Type>("i32").unwrap();
let expr = syn::parse_str::<syn::Expr>("1 + 1").unwrap();
let tokens: proc_macro2::TokenStream = quote! { fn foo() {} };

let generated = quote! {
    // #ident — 插入 Ident（变成标识符）
    struct #ident {
        // #ty — 插入 Type（变成类型）
        field: #ty,
    }

    // #expr — 插入 Expr（变成表达式）
    const VALUE: i32 = #expr;

    // #tokens — 插入任意 TokenStream（拼接 token）
    #tokens
};
```

**可插值的类型**：任何实现了 `quote::ToTokens` 的类型都可以用 `#` 插值。syn 的几乎所有类型（`Ident`、`Type`、`Expr`、`ItemFn`、`Generics` 等）都实现了 `ToTokens`。

### 4.3 插值——不插值的陷阱

```rust
// ❌ 常见错误：忘记 # 直接写变量名
let type_name = syn::Ident::new("Foo", proc_macro2::Span::call_site());
let wrong = quote! {
    impl type_name {  // 生成的是字面量 "type_name"，不是 "Foo"！
        fn hello() {}
    }
};

// ✅ 正确：用 # 插值
let correct = quote! {
    impl #type_name {  // 生成的是 "Foo"
        fn hello() {}
    }
};
```

### 4.4 迭代插值 `#(...)*`

这是 derive 宏中最强大的模式——为集合中的每个元素生成重复的代码片段。

```rust
use quote::quote;

// 假设有三个字段：x: i32, y: String, z: bool
let field_names: Vec<&syn::Ident> = /* 从 struct 解析 */;

let generated = quote! {
    fn print_fields(&self) {
        #(
            // 为每个 field_name 生成一行打印语句
            println!("{} = {:?}", stringify!(#field_names), self.#field_names);
        )*
    }
};

// 生成结果（展开后）：
// fn print_fields(&self) {
//     println!("{} = {:?}", "x", self.x);
//     println!("{} = {:?}", "y", self.y);
//     println!("{} = {:?}", "z", self.z);
// }
```

**三种迭代模式：**

```rust
use quote::quote;
use syn::Ident;

let names: Vec<Ident> = vec![
    syn::parse_quote!(x),
    syn::parse_quote!(y),
];

// 1. #(#var),* — 用逗号分隔（最后一个后面没有逗号）
let with_comma = quote! {
    const NAMES: &[&str] = &[#(stringify!(#names)),*];
};
// 生成：&["x", "y"]

// 2. #(#var);* — 用分号分隔
let with_semicolon = quote! {
    #(println!("{}", stringify!(#names));)*
};
// 生成：
// println!("{}", "x");
// println!("{}", "y");

// 3. #(#var)* — 无分隔符（粘在一起）
let no_sep = quote! {
    #(#names)*
};
// 生成：xy
```

**多变量迭代——同时迭代多个集合时，所有集合必须有相同的长度：**

```rust
let field_names: Vec<Ident> = /* ... */;
let field_types: Vec<Type> = /* ... */;

quote! {
    #(
        fn #field_names(&self) -> &#field_types { &self.#field_names }
    )*
}
// 同时为每个字段生成一个 getter
```

### 4.5 `quote_spanned!` —— 保留 Span 信息

`Span` 是 Rust 编译器用来追踪"这段代码来自哪个文件、哪些行"的元数据。保留它对于编译错误信息的准确性至关重要。

```rust
use quote::quote_spanned;

// 标准 quote! 使用 Span::call_site()（宏被调用的位置）
let a = quote! { 1 + 2 };

// quote_spanned! 可以指定 Span
let span = field.ty.span();  // 使用字段类型的原始 Span
let b = quote_spanned! { span =>
    impl Trait for #field_name { }
};
// 现在如果生成的代码报错，错误信息会指向原始字段的位置
```

**什么时候需要 `quote_spanned!`？**

- 当你处理字段级错误时——你希望编译错误指向具体的字段，而不是整个 struct
- 当生成的代码逻辑复杂时——帮助 IDE 跳转到正确的定义位置

### 4.6 完整示例：derive Hello trait

```rust
use proc_macro::TokenStream;
use quote::quote;
use syn::{parse_macro_input, DeriveInput};

#[proc_macro_derive(HelloMacro)]
pub fn derive_hello(input: TokenStream) -> TokenStream {
    let input = parse_macro_input!(input as DeriveInput);

    let name = &input.ident;
    let name_str = name.to_string();

    let expanded = match &input.data {
        syn::Data::Struct(s) => {
            let field_info: Vec<_> = match &s.fields {
                syn::Fields::Named(fields) => {
                    fields.named.iter().map(|f| {
                        let name = &f.ident;
                        let ty = &f.ty;
                        (quote!(#name), quote!(#ty))
                    }).collect()
                }
                syn::Fields::Unnamed(fields) => {
                    fields.unnamed.iter().enumerate().map(|(i, f)| {
                        let idx = syn::Index::from(i);
                        let ty = &f.ty;
                        (quote!(#idx), quote!(#ty))
                    }).collect()
                }
                syn::Fields::Unit => vec![],
            };

            let print_fields = if field_info.is_empty() {
                quote! {
                    println!("  (no fields)");
                }
            } else {
                quote! {
                    #(
                        println!("  {}: {} = {:?}", stringify!(#field_info.0), stringify!(#field_info.1), &self.#field_info.0);
                    )*
                }
            };

            quote! {
                impl HelloMacro for #name {
                    fn hello() {
                        println!("Hello! I'm a struct named {} with {} fields.", #name_str, field_info.len());
                    }

                    fn debug_fields(&self) {
                        #print_fields
                    }
                }
            }
        }
        syn::Data::Enum(e) => {
            let variant_count = e.variants.len();
            let variant_names: Vec<_> = e.variants.iter().map(|v| {
                let name = &v.ident;
                name.to_string()
            }).collect();

            quote! {
                impl HelloMacro for #name {
                    fn hello() {
                        println!("Hello! I'm an enum named {} with {} variants: {:?}", #name_str, variant_count, &[#(#variant_names),*]);
                    }
                }
            }
        }
        syn::Data::Union(_) => {
            quote! {
                compile_error!("HelloMacro does not support unions");
            }
        }
    };

    TokenStream::from(expanded)
}
```

### 5.2 在字段上查找属性

```rust
use syn::{Field, Attribute, Meta};

fn find_custom_attr<'a>(field: &'a Field, attr_name: &str) -> Option<&'a Attribute> {
    field.attrs.iter().find(|attr| {
        attr.path().is_ident(attr_name)
    })
}

    find_custom_attr(field, "my_attr")

                .map(|ident| ident == "skip")
                .unwrap_or(false)
        })
```

### 5.3 解析复杂的属性参数

属性可以携带复杂的参数——类似函数调用的语法。

```rust
// 假设用户写：
// #[my_attr(name = "Alice", age = 30, active = true)]

use syn::{Attribute, Meta, MetaNameValue, Expr};

fn parse_my_attr(attr: &Attribute) -> syn::Result<MyAttrConfig> {
    // 获取 Meta 形式
    let meta = &attr.meta;

    // Meta 有三种形式：
    // Meta::Path(Path)           — #[attr]          （无参数）
    // Meta::List(MetaList)       — #[attr(a, b)]    （括号列表）
    // Meta::NameValue(MetaNameValue) — #[attr = value] （等号赋值）

    match meta {
        Meta::List(list) => {
            // 解析 #[my_attr(key1 = "value1", key2 = 42)] 中的参数
            let args: Punctuated<MetaNameValue, Token![,]> =
                list.parse_args_with(Punctuated::parse_terminated)?;

            let mut name = None;
            let mut age = None;

            for nv in args {
                if nv.path.is_ident("name") {
                    if let Expr::Lit(expr_lit) = &nv.value {
                        if let Lit::Str(s) = &expr_lit.lit {
                            name = Some(s.value());
                        }
                    }
                } else if nv.path.is_ident("age") {
                    if let Expr::Lit(expr_lit) = &nv.value {
                        if let Lit::Int(n) = &expr_lit.lit {
                            age = Some(n.base10_parse::<u32>()?);
                        }
                    }
                }
            }

            Ok(MyAttrConfig { name, age })
        }
        _ => Err(syn::Error::new_spanned(attr, "expected #[my_attr(key = value, ...)]")),
    }
}

struct MyAttrConfig {
    name: Option<String>,
    age: Option<u32>,
}
```

### 5.4 syn::Meta 完整指南

```rust
// #[name_value = "hello"]           → Meta::NameValue

}
```

## 6. 泛型参数处理

### 6.1 泛型处理的必要性

初学者最常见的 derive 宏错误：**在泛型 struct 上生成的 impl 块缺少泛型参数**。

```rust
// ❌ 错误的 derive 宏输出
#[derive(MyDerive)]
struct Foo<T: Debug> {
    value: T,
}

// 错误代码生成了：
// impl MyDerive for Foo { ... }  // 缺少 <T>
// 而编译器期望的是：
// impl<T: Debug> MyDerive for Foo<T> { ... }

// ✅ 正确处理泛型后的输出：
// impl<T: Debug> MyDerive for Foo<T> { ... }
```

### 6.2 `syn::Generics` API

```rust
use syn::{DeriveInput, Generics, GenericParam, WherePredicate};

fn handle_generics(input: &DeriveInput) -> proc_macro2::TokenStream {
    let name = &input.ident;
    let generics = &input.generics;  // syn::Generics

    // 分割泛型参数——impl 块需要三部分：
    // 1. 声明部分：<T: Debug, 'a>
    // 2. where 子句：where T: Clone
    // 3. 使用部分：Foo<T>
    let (impl_generics, ty_generics, where_clause) = generics.split_for_impl();

    // impl_generics:  <T: Debug, 'a>
    // ty_generics:    <T>
    // where_clause:   where T: Clone

    let expanded = quote! {
        impl #impl_generics MyDerive for #name #ty_generics #where_clause {
            fn my_method(&self) {
                // ...
            }
        }
    };

    expanded
}
```

`split_for_impl()` 是处理泛型的核心方法——它自动处理了：
- 泛型参数名（`T`、`'a` 等）
- trait bound（`T: Debug`）
- where 子句的重新排列

### 6.3 给生成的 impl 添加额外的 trait bound

有时你的宏要求泛型参数满足额外的 constraint：

```rust
fn add_bounds_to_generics(generics: &mut syn::Generics) {
    for param in &mut generics.params {
        if let syn::GenericParam::Type(type_param) = param {
            // 为每个泛型类型参数添加 Default bound
            type_param.bounds.push(syn::parse_quote!(Default));
        }
    }
}

// 使用：
let mut generics = input.generics.clone();
add_bounds_to_generics(&mut generics);
let (impl_generics, ty_generics, where_clause) = generics.split_for_impl();

let expanded = quote! {
    impl #impl_generics MyDerive for #name #ty_generics #where_clause {
        fn create_default() -> Self {
            Self { /* 使用 T::default() */ }
        }
    }
};
```

### 6.4 处理生命周期

```rust
// 生命周期参数在 split_for_impl() 中自动处理

#[derive(MyDerive)]
struct Borrowed<'a> {
    data: &'a str,
}

// split_for_impl 输出：
// impl_generics = <'a>
// ty_generics = <'a>
// 自动生成：impl<'a> MyDerive for Borrowed<'a> { ... }
```

### 6.5 完整示例：derive DebugLike（支持泛型）

```rust
use proc_macro::TokenStream;
use quote::quote;
use syn::{parse_macro_input, DeriveInput, Data, Fields};

#[proc_macro_derive(DebugLike)]
pub fn derive_debug_like(input: TokenStream) -> TokenStream {
    let input = parse_macro_input!(input as DeriveInput);
    let name = &input.ident;
    let generics = &input.generics;
    let (impl_generics, ty_generics, where_clause) = generics.split_for_impl();

    let debug_body = match &input.data {
        Data::Struct(s) => {
            match &s.fields {
                Fields::Named(fields) => {
                    let field_names: Vec<_> = fields.named.iter()
                        .filter_map(|f| f.ident.as_ref())
                        .collect();
                    quote! {
                        #(
                            builder.field(stringify!(#field_names), &self.#field_names);
                        )*
                    }
                }
                Fields::Unnamed(fields) => {
                    let indices: Vec<_> = (0..fields.unnamed.len())
                        .map(syn::Index::from)
                        .collect();
                    quote! {
                        #(
                            builder.field(&format!("field_{}", stringify!(#indices)), &self.#indices);
                        )*
                    }
                }
                Fields::Unit => {
                    quote! { let _ = builder; }
                }
            }
        }
        Data::Enum(_) => {
            return quote! {
                compile_error!("DebugLike does not support enums");
            }.into();
        }
        Data::Union(_) => {
            return quote! {
                compile_error!("DebugLike does not support unions");
            }.into();
        }
    };

    let expanded = quote! {
        impl #impl_generics std::fmt::Debug for #name #ty_generics #where_clause {
            fn fmt(&self, f: &mut std::fmt::Formatter<'_>) -> std::fmt::Result {
                let mut builder = f.debug_struct(stringify!(#name));
                #debug_body
                builder.finish()
            }
        }
    };

    TokenStream::from(expanded)
}
```

## 7. 实战：#[derive(Builder)] 完整实现

### 7.1 设计目标

为任意 struct 自动生成 Builder 模式代码。目标功能：
1. 为每个字段生成 setter 方法（`fn field_name(mut self, value: Type) -> Self`）
2. 自动检测 `Option<T>` 字段——为用户提供自动包装
3. 支持 `#[builder(default = expr)]` 设置默认值
4. `build()` 方法返回完整 struct

### 7.2 完整实现

```rust
// my-builder-derive/src/lib.rs
use proc_macro::TokenStream;
use quote::{quote, format_ident};
use syn::{
    parse_macro_input, DeriveInput, Data, Fields, Field, Type,
    GenericArgument, PathArguments, Lit, Meta,
};

#[proc_macro_derive(Builder, attributes(builder))]
pub fn derive_builder(input: TokenStream) -> TokenStream {
    let input = parse_macro_input!(input as DeriveInput);
    let name = &input.ident;

    // 生成 Builder 的名称
    let builder_name = format_ident!("{}Builder", name);

    // 处理泛型
    let generics = &input.generics;
    let (impl_generics, ty_generics, where_clause) = generics.split_for_impl();

    // 只支持 struct
    let fields = match &input.data {
        Data::Struct(s) => &s.fields,
        _ => {
            return quote! {
                compile_error!("Builder only supports structs");
            }.into();
        }
    };

    // 提取字段列表
    let named_fields = match fields {
        Fields::Named(fields) => &fields.named,
        _ => {
            return quote! {
                compile_error!("Builder only supports named fields");
            }.into();
        }
    };

    // ============ 处理每个字段 ============

    // 字段名（用于生成 setter 方法名）
    let field_names: Vec<_> = named_fields.iter()
        .map(|f| f.ident.as_ref().unwrap())
        .collect();

    // 字段类型（用于 Builder 结构体字段类型）
    let field_types: Vec<_> = named_fields.iter()
        .map(|f| f.ty.clone())
        .collect();

    // 检查每个字段是否是 Option<T>
    let is_option: Vec<bool> = named_fields.iter()
        .map(|f| is_type_option(&f.ty))
        .collect();

    // 获取每个字段的 #[builder(default = expr)] 属性
    let default_values: Vec<Option<syn::Expr>> = named_fields.iter()
        .map(|f| get_builder_default(f))
        .collect();

    // ============ Builder 结构体的字段类型 ============

    // Builder 字段的类型是把原始类型包装在 Option 中
    // （因为 builder 模式中字段可能是未设置的）
    let builder_field_types: Vec<proc_macro2::TokenStream> = field_types.iter().enumerate()
        .map(|(i, ty)| {
            if is_option[i] {
                // 原始字段已经是 Option<T>
                // Builder 字段直接存储 T（未设置的），不需要额外 Option
                if let Some(inner_ty) = extract_option_inner(ty) {
                    quote! { Option<#inner_ty> }
                } else {
                    quote! { Option<#ty> }
                }
            } else {
                quote! { Option<#ty> }
            }
        })
        .collect();

    // ============ setter 方法 ============

    let setters: Vec<proc_macro2::TokenStream> = named_fields.iter().enumerate()
        .map(|(i, field)| {
            let field_name = &field.ident;
            let ty = &field.ty;

            if is_option[i] {
                if let Some(inner_ty) = extract_option_inner(ty) {
                    // Option<T> 字段：提供两个 setter
                    // 1. 接受 T 的那个
                    let setter_t = format_ident!("set_{}", field_name.as_ref().unwrap());
                    // 2. 接受 Option<T> 的那个
                    let setter_opt = field_name.as_ref().unwrap().clone();

                    quote! {
                        /// Set the field with a value (wraps in Some)
                        pub fn #setter_t(mut self, value: #inner_ty) -> Self {
                            self.#field_name = Some(value);
                            self
                        }
                        /// Set the field with an Option
                        pub fn #setter_opt(mut self, value: #inner_ty) -> Self {
                            self.#field_name = Some(value);
                            self
                        }
                    }
                } else {
                    quote! {
                        pub fn #field_name(mut self, value: #ty) -> Self {
                            self.#field_name = Some(value);
                            self
                        }
                    }
                }
            } else {
                quote! {
                    pub fn #field_name(mut self, value: #ty) -> Self {
                        self.#field_name = Some(value);
                        self
                    }
                }
            }
        })
        .collect();

    // ============ build() 方法 ============

    // 为每个字段生成 .ok_or("...") 逻辑
    let build_fields: Vec<proc_macro2::TokenStream> = named_fields.iter().enumerate()
        .map(|(i, field)| {
            let field_name = &field.ident;
            let err_msg = format!("field `{}` not set", field_name.as_ref().unwrap());

            if let Some(ref default_expr) = default_values[i] {
                // 有默认值——unwrap_or(default)
                quote! {
                    #field_name: self.#field_name.unwrap_or_else(|| #default_expr)
                }
            } else {
                // 无默认值——必须设置
                quote! {
                    #field_name: self.#field_name.ok_or(#err_msg)?
                }
            }
        })
        .collect();

    // ============ 输出 ============

    let expanded = quote! {
        impl #impl_generics #name #ty_generics #where_clause {
            /// Create a new Builder
            pub fn builder() -> #builder_name #ty_generics {
                #builder_name {
                    #(
                        #field_names: None,
                    )*
                }
            }
        }

        /// Builder for [`#name`]
        pub struct #builder_name #impl_generics #where_clause {
            #(
                #field_names: #builder_field_types,
            )*
        }

        impl #impl_generics #builder_name #ty_generics #where_clause {
            #(#setters)*

            /// Build the final struct
            pub fn build(self) -> Result<#name #ty_generics, &'static str> {
                Ok(#name {
                    #(#build_fields,)*
                })
            }
        }
    };

    TokenStream::from(expanded)
}

// ============ 辅助函数 ============

/// 检查类型是否是 Option<T> 的形式
fn is_type_option(ty: &Type) -> bool {
    if let Type::Path(type_path) = ty {
        if let Some(segment) = type_path.path.segments.last() {
            if segment.ident == "Option" {
                return true;
            }
        }
    }
    false
}

/// 从 Option<T> 中提取 T
fn extract_option_inner(ty: &Type) -> Option<&Type> {
    if let Type::Path(type_path) = ty {
        if let Some(segment) = type_path.path.segments.last() {
            if segment.ident == "Option" {
                if let PathArguments::AngleBracketed(ref args) = segment.arguments {
                    if let Some(GenericArgument::Type(inner_ty)) = args.args.first() {
                        return Some(inner_ty);
                    }
                }
            }
        }
    }
    None
}

/// 从字段上获取 #[builder(default = expr)] 的值
fn get_builder_default(field: &Field) -> Option<syn::Expr> {
    for attr in &field.attrs {
        if !attr.path().is_ident("builder") {
            continue;
        }

        // 解析 #[builder(default = expr)]
        if let Meta::List(list) = &attr.meta {
            // list.tokens 的内容是：default = expr
            // 解析为 MetaNameValue 列表
            let parse_result: Result<Vec<_>, _> = list.parse_args_with(
                syn::punctuated::Punctuated::<syn::MetaNameValue, syn::Token![,]>::parse_terminated,
            );

            if let Ok(nv_list) = parse_result {
                for nv in nv_list {
                    if nv.path.is_ident("default") {
                        return Some(nv.value);
                    }
                }
            }
        }
    }
    None
}
```

## 8. 🔧 调试本节：Derive 宏的排错方法

derive 宏调试困难的原因在于：代码在编译器内部运行，你无法直接 `println!` 到 stdout，也无法用 `dbg!()`。本节介绍 4 种核心调试技巧。

### 8.1 `eprintln!` —— 打印到 stderr

```rust
    eprintln!("=== END RAW INPUT ===");

    let input = parse_macro_input!(input as DeriveInput);

    eprintln!("=== PARSED AST ===");
    eprintln!("{:#?}", input);
    eprintln!("=== END PARSED AST ===");

    TokenStream::new()
}
```

### 8.3 Span 和错误定位

**Span 是 syn 最重要的概念之一**——它记录了每个 AST 节点的源位置信息。在生成代码时保留 Span，可以确保编译错误指向正确的位置。

```rust
use syn::spanned::Spanned;

#[proc_macro_derive(MyTrait)]
pub fn derive_my_trait(input: TokenStream) -> TokenStream {
    let input = parse_macro_input!(input as DeriveInput);

    // 从 DeriveInput 获取 Span
    let span = input.span();

    // 使用 syn::Error 报告编译错误（保留 Span）
    match process_derive(&input) {
        Ok(output) => output.into(),
        Err(err) => {
            // Error 的 span 决定错误信息指向哪里
            let compile_err = syn::Error::new(
                span,
                "MyTrait derivation failed: check your fields",
            );
            compile_err.to_compile_error().into()
        }
    }
}
```

**Span API 速查：**

```rust
use syn::spanned::Spanned;

let ident: syn::Ident = /* ... */;

// 获取 Span

// 创建指向特定位置的错误

// 或者直接从任何 syn 类型

// 合并多个错误

errors.push(syn::Error::new(span2, "error 2"));
```

### 8.4 常见陷阱及排错

**陷阱 1：proc_macro 和 proc_macro2 的 TokenStream 转换遗漏**

```rust
// ❌ 错误——忘记转换回来
#[proc_macro_derive(MyTrait)]
pub fn derive_my_trait(input: TokenStream) -> TokenStream {
    let input = proc_macro2::TokenStream::from(input);
    // ... 处理，得到 proc_macro2::TokenStream ...
    let output = quote! { ... };
    output  // ❌ 类型不匹配：expected proc_macro::TokenStream
            //               found proc_macro2::TokenStream
}

// ✅ 正确——入口转换进，出口转换出
#[proc_macro_derive(MyTrait)]
pub fn derive_my_trait(input: TokenStream) -> TokenStream {
    // 方式 A：使用 parse_macro_input! 自动处理
    let input = parse_macro_input!(input as DeriveInput);
    let output = quote! { ... };
    TokenStream::from(output)  // 手动转回

    // 方式 B：使用 .into() 自动处理
    let output: proc_macro2::TokenStream = quote! { ... };
    output.into()  // From<proc_macro2::TokenStream> for proc_macro::TokenStream
}
```

**陷阱 2：quote! 中的 `#` 遗忘**

```rust
// ❌ 忘记 # — 生成了字面量标识符而非变量值
let method_name: syn::Ident = parse_quote!(my_method);
let bad = quote! {
    fn method_name() {}  // 生成的是 fn method_name()，不是 fn my_method()
};

// ✅ 加 #
let good = quote! {
    fn #method_name() {}  // 生成 fn my_method()
};
```

**陷阱 3：不处理泛型导致 trait bound 错误**

```rust
// ❌ 直接使用 name，没有 split_for_impl
quote! {
    impl MyTrait for #name { ... }
}
// 应用于 Foo<T> 时生成：impl MyTrait for Foo<T> { ... }
// 但 Foo<T> 中的 T 没有声明——编译错误

```

**陷阱 4：属性被编译器静默删除**

```rust

    // field.attrs 中找不到 my_attr
}

// ✅ 声明属性

pub fn derive_my_trait(input: TokenStream) -> TokenStream {
    // 现在 #[my_attr(skip)] 被保留
}
```

**陷阱 5：syn 类型选择错误**

```rust
// ❌ 在处理 ItemStruct 时尝试解析为 DeriveInput 的子项
// 或反过来：以为是 DeriveInput 但其实是另一个 syn 类型
```

## 答案

### 练习 1 答案

**问题诊断：**

泛型参数 `T` 在 `impl` 块中没有被声明。当用户写：

```rust
#[derive(Describe)]
struct Point<T> {
    x: T,
    y: T,
}
```

宏生成了：

```rust
impl Describe for Point { ... }
//                    ^^^^^ 应该是 Point<T>
```

而不是：

```rust
impl<T> Describe for Point<T> { ... }
//  ^^^ 需要声明泛型参数 T
```

**根因：** 代码中完全没有引用 `input.generics`——泛型参数被丢失了。`split_for_impl()` 没有出现在任何地方。

**修复：**

```rust
use syn::{parse_macro_input, DeriveInput, Data, Fields};

#[proc_macro_derive(Describe)]
pub fn derive_describe(input: TokenStream) -> TokenStream {
    let input = parse_macro_input!(input as DeriveInput);
    let name = &input.ident;

    // === 修复的关键：处理泛型 ===
    let generics = &input.generics;
    let (impl_generics, ty_generics, where_clause) = generics.split_for_impl();

    let field_descriptions = match &input.data {
        Data::Struct(s) => match &s.fields {
            Fields::Named(fields) => {
                fields.named.iter().map(|f| {
                    let name = f.ident.as_ref().unwrap().to_string();
                    let ty = f.ty.to_token_stream().to_string();
                    format!("  {}: {}", name, ty)
                }).collect::<Vec<_>>().join("\n")
            }
            Fields::Unnamed(_) => String::from("  (tuple struct fields)"),
            Fields::Unit => String::from("  (no fields)"),
        },
        _ => String::from("(not a struct)"),
    };

    // === 修复：使用 impl_generics, ty_generics, where_clause ===
    let expanded = quote! {
        impl #impl_generics Describe for #name #ty_generics #where_clause {
            fn describe() -> String {
                format!("Struct {}:\n{}", stringify!(#name), #field_descriptions)
            }
        }
    };

    TokenStream::from(expanded)
}
```

**关键理解：** `split_for_impl()` 返回三个部分：
- `impl_generics`：`<T>` 或 `<T: Debug>`——放在 `impl` 关键字后面
- `ty_generics`：`<T>`——放在类型名称后面
- `where_clause`：`where T: Something`——放在 impl 块的大括号前面

### 练习 2 答案

```rust
// my-debug-derive/src/lib.rs
use proc_macro::TokenStream;
use quote::quote;
use syn::{parse_macro_input, DeriveInput, Data, Fields};

#[proc_macro_derive(MyDebug, attributes(my_debug))]
pub fn derive_my_debug(input: TokenStream) -> TokenStream {
    let input = parse_macro_input!(input as DeriveInput);
    let name = &input.ident;

    // 处理泛型
    let generics = &input.generics;
    let (impl_generics, ty_generics, where_clause) = generics.split_for_impl();

    let debug_body = match &input.data {
        Data::Struct(s) => {
            match &s.fields {
                Fields::Named(fields) => {
                    // 具名字段：生成 Point { x: ..., y: ..., label: ... }
                    let field_entries: Vec<_> = fields.named.iter()
                        .filter(|f| !has_my_debug_skip(f))
                        .map(|f| {
                            let name = f.ident.as_ref().unwrap();
                            quote! {
                                builder.field(
                                    stringify!(#name),
                                    &self.#name,
                                );
                            }
                        })
                        .collect();

                    quote! {
                        let mut builder = f.debug_struct(stringify!(#name));
                        #(#field_entries)*
                        builder.finish()
                    }
                }
                Fields::Unnamed(fields) => {
                    // 元组结构体：生成 Color(255, 128, 0)
                    let field_entries: Vec<_> = fields.unnamed.iter()
                        .enumerate()
                        .filter(|(_, f)| !has_my_debug_skip(f))
                        .map(|(i, _f)| {
                            let idx = syn::Index::from(i);
                            quote! {
                                builder.field(&self.#idx);
                            }
                        })
                        .collect();

                    quote! {
                        let mut builder = f.debug_tuple(stringify!(#name));
                        #(#field_entries)*
                        builder.finish()
                    }
                }
                Fields::Unit => {
                    quote! {
                        f.debug_struct(stringify!(#name)).finish()
                    }
                }
            }
        }
        _ => {
            return quote! {
                compile_error!("MyDebug only supports structs");
            }.into();
        }
    };

    let expanded = quote! {
        impl #impl_generics std::fmt::Debug for #name #ty_generics #where_clause {
            fn fmt(&self, f: &mut std::fmt::Formatter<'_>) -> std::fmt::Result {
                #debug_body
            }
        }
    };

    TokenStream::from(expanded)
}

/// 检查字段是否有 #[my_debug(skip)] 属性
fn has_my_debug_skip(field: &syn::Field) -> bool {
    field.attrs.iter().any(|attr| {
        if !attr.path().is_ident("my_debug") {
            return false;
        }
        // 尝试解析 #[my_debug(skip)]
        if let Ok(arg) = attr.parse_args::<syn::Ident>() {
            return arg == "skip";
        }
        false
    })
}
```

**核心设计思想：**

1. **属性过滤**：`has_my_debug_skip()` 在生成代码之前过滤字段，带 `#[my_debug(skip)]` 的字段不参与 Debug 输出
2. **泛型处理**：必须使用 `split_for_impl()`，否则泛型 struct 无法编译
3. **Named vs Unnamed**：具名字段用 `debug_struct`，元组字段用 `debug_tuple`，输出格式不同
4. **Span 保留**：`stringify!(#name)` 和字段索引都使用 syn 原始类型，保留 Span 信息

### 练习 3 答案

**A. 核心数据结构设计思路**

Builder 模式的关键是将"字段值是否已设置"从类型层面退化为运行时的 Option：

```
原始 struct 的字段类型 T  →  Builder 中的字段类型 Option<T>
                         →  None 表示"未设置"
                         →  Some(v) 表示"已设置为 v"
```

对于本身是 `Option<T>` 的字段：
- Builder 中存储 `Option<T>`（内层 T，不是 `Option<Option<T>>`）
- setter 接受 `T`（而非 `Option<T>`），自动包装 `Some(value)`
- 这样用户写 `.database("mydb")` 而非 `.database(Some("mydb"))`

**B. setter 生成的伪代码**

```
for each field in struct.fields:
    if field.type is Option<InnerType>:
        // Option 字段的特殊处理
        生成 setter:
            fn 字段名(mut self, value: InnerType) -> Self {
                self.字段名 = Some(value);  // 自动包装
                self
            }
    else:
        // 普通字段
        生成 setter:
            fn 字段名(mut self, value: FieldType) -> Self {
                self.字段名 = Some(value);  // Builder 内部用 Option 包装
                self
            }
```

**C. build() 方法的生成伪代码**

```
fn build(self) -> Result<OriginalStruct, &'static str>:
    for each field in struct.fields:
        if field has #[builder(default = expr)]:
            字段名: self.字段名.unwrap_or_else(|| expr)
            // 有默认值——永远成功
        else:
            字段名: self.字段名.ok_or("field `字段名` not set")?
            // 无默认值——未设置则返回 Err

    Ok(OriginalStruct { 所有字段 })
```

**完整实现要点（供参考）：**

```rust
// Builder 结构体的生成模式：
// struct FooBuilder {
//     field1: Option<Type1>,     // 普通字段
//     field2: Option<InnerType>,  // Option 字段——存内层类型
// }

// setter 生成的核心区别：
// 普通字段：fn field1(mut self, value: Type1) -> Self
//           { self.field1 = Some(value); self }
// Option字段：fn field2(mut self, value: InnerType) -> Self
//             { self.field2 = Some(value); self }

// build() 的核心区别：
// 有 default：self.field.unwrap_or_else(|| default_expr)
// 无 default：self.field.ok_or("field `field` not set")?
```
