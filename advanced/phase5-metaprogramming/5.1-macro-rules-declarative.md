# 5.1 声明宏 macro_rules! —— Rust 的代码模板引擎

> "声明宏是 Rust 的'代码模板引擎'——你描述模式，编译器填充细节。与 C 的 `#define` 不同，macro_rules! 是部分卫生的、模式驱动的、类型感知的代码生成系统。"

---

## 一、设计哲学

### 1.1 为什么需要 macro_rules!？

编写 Rust 代码时，你迟早会遇到这些痛点：

- 想为 `Vec<T>` 和 `HashMap<K,V>` 写一个通用的初始化语法
- 想消除重复的 `match` 分支或 trait 实现
- 想创建一个 DSL（领域特定语言）嵌入到 Rust 中
- 想在编译期根据条件生成不同的代码

**macro_rules! 为这些问题而生。** 它允许你在编译期进行"模式匹配 → 代码替换"，是 Rust 元编程的第一道门户。

### 1.2 设计取舍

```rust
// C 的 #define —— 纯文本替换，不安全、不卫生
// #define SQUARE(x) x * x
// SQUARE(1+2) → 1+2*1+2 → 5 (错误!)

// Rust 的 macro_rules! —— 模式驱动，部分卫生
macro_rules! square {
    ($x:expr) => { $x * $x };
}
// square!(1+2) → (1+2)*(1+2) → 9 (正确!)
```

核心设计哲学：

| 维度 | C `#define` | Rust `macro_rules!` |
|------|-------------|---------------------|
| 匹配机制 | 文本替换 | 基于 token tree 的模式匹配 |
| 类型安全 | 无 | 展开后通过类型检查 |
| 卫生性 | 不卫生（变量捕获） | 部分卫生 |
| 递归支持 | 不支持 | 支持 |
| 语法约束 | 无（可产生非法语法） | 展开结果必须是合法 AST |

## 二、深度原理

### 2.1 macro_rules! 基础语法

#### 基本形式

```rust
macro_rules! 宏名 {
    (模式1) => { 展开1 };
    (模式2) => { 展开2 };
    // ...更多分支
}
```

编译器在遇到 `宏名!(...)` 调用时，**按从上到下的顺序**匹配每条规则。第一个匹配成功的规则被执行，展开后的代码被插入调用处。如果没有任何规则匹配，编译报错。

#### 第一个例子：`print_something!`

```rust
macro_rules! print_something {
    // 匹配一个字面量字符串
    ($msg:expr) => {
        println!("[LOG] {}", $msg);
    };
    // 匹配两个字面量
    ($prefix:expr, $msg:expr) => {
        println!("[{}] {}", $prefix, $msg);
    };
}

fn main() {
    print_something!("hello");
    // 展开为: println!("[LOG] {}", "hello");

    print_something!("WARN", "disk full");
    // 展开为: println!("[{}] {}", "WARN", "disk full");
}
```

#### `vec!` 的简化实现

```rust
// 标准库 vec! 背后的核心思想
macro_rules! my_vec {
    // 空 Vec
    () => {
        Vec::new()
    };
    // 单个元素
    ($elem:expr) => {{
        let mut v = Vec::new();
        v.push($elem);
        v
    }};
    // 多个元素 —— 使用重复模式（后面的章节详解）
    ($($elem:expr),+ $(,)?) => {{
        let mut v = Vec::new();
        $(
            v.push($elem);
        )+
        v
    }};
}

fn main() {
    let v1: Vec<i32> = my_vec!();
    let v2 = my_vec!(42);
    let v3 = my_vec!(1, 2, 3, 4);
    let v4 = my_vec!(1, 2, 3,); // 尾部逗号也支持
    println!("{:?} {:?} {:?} {:?}", v1, v2, v3, v4);
}
```

#### 多分支匹配与顺序

```rust
macro_rules! describe {
    // 最具体的规则放在前面
    (zero) => { "you said zero" };
    ($x:expr) => { "you said something else" };
}

fn main() {
    dbg!(describe!(zero));        // "you said zero" —— 匹配第一条
    dbg!(describe!(42));           // "you said something else" —— 匹配第二条
}
```

**重要**：如果有两个规则都能匹配，先写的那条胜出。因此，把更具体的规则放在更通用的规则前面。

### 2.2 捕获类型详解（Capture Types）

macro_rules! 的威力来自它的匹配系统。每个 `$名称:类型` 告诉编译器"这里应该匹配什么类型的语法片段"。

#### 常见捕获类型速查表

| 捕获类型 | 匹配内容 | 示例 |
|----------|----------|------|
| `expr` | 表达式 | `1 + 2`，`foo()`，`x`，`{ let a = 1; a }` |
| `ty` | 类型标注 | `i32`，`Vec<String>`，`&str` |
| `ident` | 标识符 | `foo`，`x`，`HashMap`，`new` |
| `tt` | token tree（单个 token 或括号组） | `x`，`()`，`{ }`，`struct` |
| `pat` | 模式 | `Some(x)`，`1..=5`，`_` |
| `stmt` | 语句（可带分号） | `let x = 1;`，`foo()` |
| `block` | 代码块 | `{ ... }` |
| `item` | 顶层项 | `fn foo() {}`，`struct Bar;` |
| `meta` | 属性内容 | `derive(Debug)` |
| `lifetime` | 生命周期 | `'a`，`'static` |
| `vis` | 可见性 | `pub`，`pub(crate)` |
| `literal` | 字面量 | `42`，`"hello"`，`true` |
| `path` | 路径 | `std::collections::HashMap` |

#### 详细示例

```rust
// --- expr: 匹配任意表达式 ---
macro_rules! debug_expr {
    ($e:expr) => {
        // stringify! 把表达式变成字符串，用于调试
        println!("{} = {:?}", stringify!($e), $e);
    };
}

fn expr_demo() {
    let x = 42;
    debug_expr!(x);
    debug_expr!(x + 1);
    debug_expr!(vec![1, 2, 3].len());
    debug_expr!({
        let a = 10;
        let b = 20;
        a + b
    });
    // 输出:
    // x = 42
    // x + 1 = 43
    // vec![1, 2, 3].len() = 3
    // { let a = 10; let b = 20; a + b } = 30
}

// --- ty: 匹配类型标注 ---
macro_rules! make_default {
    ($t:ty) => {
        <$t as Default>::default()
    };
}

fn ty_demo() {
    let x: i32 = make_default!(i32);        // 0
    let y: String = make_default!(String);   // ""
    let z: Vec<u8> = make_default!(Vec<u8>); // []
    println!("{:?} {:?} {:?}", x, y, z);
}

// --- ident: 匹配标识符 ---
macro_rules! make_getter_setter {
    ($field:ident, $t:ty) => {
        fn $field(&self) -> &$t {
            &self.$field
        }

        // paste 风格的拼接头通常需要 paste crate
        // 这里用间接的方式演示
        fn set_ident(&mut self, val: $t) {
            self.$field = val;
        }
    };
}

// --- tt: token tree —— 万能匹配器 ---
macro_rules! count_tt {
    () => { 0usize };
    ($head:tt $($tail:tt)*) => {
        1usize + count_tt!($($tail)*)
    };
}

fn tt_demo() {
    // 每个 token（标识符、标点、关键字等）都算一个
    let n1 = count_tt!(a b c);             // 3
    let n2 = count_tt!(let x = 1;);        // 5 (let/x/=/1/;)
    let n3 = count_tt!(fn foo() {});       // 5 (fn/foo/()/{})
    println!("{} {} {}", n1, n2, n3);
}

// --- pat: 匹配模式 ---
macro_rules! match_arms {
    ($val:expr, $($pat:pat => $res:expr),+ $(,)?) => {
        match $val {
            $($pat => $res),+
        }
    };
}

fn pat_demo() {
    let x = Some(42);
    let result = match_arms!(x,
        Some(0) => "zero",
        Some(1..=10) => "small",
        Some(n) => "other",
        None => "nothing",
    );
    println!("{}", result);
}

// --- stmt: 匹配语句 ---
macro_rules! timed {
    ($($stmt:stmt);* $(;)?) => {
        let start = std::time::Instant::now();
        $($stmt;)*
        println!("elapsed: {:?}", start.elapsed());
    };
}

fn stmt_demo() {
    timed! {
        let x = (0..10000).sum::<i64>();
        let y = (0..10000).product::<i64>();
    };
}

// --- block: 匹配代码块 ---
macro_rules! with_lock {
    ($mutex:expr, $block:block) => {{
        let guard = $mutex.lock().unwrap();
        let result = $block;
        drop(guard);
        result
    }};
}

// --- item: 匹配完整的项（函数、结构体等） ---
macro_rules! wrap_debug {
    ($item:item) => {
        #[derive(Debug)]
        $item
    };
}

wrap_debug! {
    struct Point {
        x: i32,
        y: i32,
    }
}

fn item_demo() {
    let p = Point { x: 1, y: 2 };
    println!("{:?}", p);
}

// --- meta: 匹配属性参数 ---
macro_rules! attr_sniffer {
    ($(#[$meta:meta])* $item:item) => {{
        println!("Found {} attributes", 0 $(+ { stringify!($meta); 1 })*);
        $item
    }};
}

// --- lifetime: 匹配生命周期标注 ---
macro_rules! make_ref_fn {
    ($lt:lifetime) => {
        fn get_ref<$lt, T>(input: &$lt T) -> &$lt T {
            input
        }
    };
}

make_ref_fn!('a);

// --- vis: 匹配可见性 ---
macro_rules! make_module_item {
    ($vis:vis fn $name:ident() -> $ret:ty $body:block) => {
        $vis fn $name() -> $ret $body
    };
}

mod demo {
    make_module_item!(pub(crate) fn greet() -> String { "hello".to_string() });
}

// --- path: 匹配路径 ---
macro_rules! use_type {
    ($p:path) => {
        let x: $p = Default::default();
    };
}

fn path_demo() {
    use_type!(std::collections::HashSet<i32>);
}

// --- literal: 匹配字面量 ---
macro_rules! literal_match {
    (int: $i:literal) => { println!("integer literal: {}", $i); };
    (str: $s:literal) => { println!("string literal: {}", $s); };
    (bool: $b:literal) => { println!("bool literal: {}", $b); };
}

fn literal_demo() {
    literal_match!(int: 42);
    literal_match!(str: "hello");
    literal_match!(bool: true);
}
```

#### 常见陷阱：expr 匹配与双括号

```rust
// expr 匹配不会自动包裹表达式，但宏展开中常常需要额外的括号
macro_rules! bad_twice {
    ($x:expr) => { $x * 2 };
}

macro_rules! good_twice {
    ($x:expr) => {{ $x * 2 }};  // 双括号：外层是 block 的值，内层保护优先级
}

fn ambiguity_demo() {
    // bad_twice!(1 + 2) 展开为 1 + 2 * 2 = 5（错误！）
    // 因为展开后没有括号保护运算优先级
    //
    // 解决方案：在展开中显式加括号
}

macro_rules! safe_twice {
    ($x:expr) => { ($x) * 2 };  // 括号保护
}
// safe_twice!(1+2) → (1+2) * 2 = 6 ✓
```

### 2.3 重复模式（Repetition Patterns）

重复是 macro_rules! 最强大的特性之一。语法规则：

| 语法 | 含义 |
|------|------|
| `$( ... )*` | 重复零次或多次 |
| `$( ... )+` | 重复一次或多次 |
| `$( ... ),*` | 零次或多次，用逗号分隔 |
| `$( ... );+` | 一次或多次，用分号分隔 |
| `$( ... )?` | 重复零次或一次（nightly-only） |

#### 基本的重复示例

```rust
// 打印任意多个值（类似 println!）
macro_rules! print_all {
    // $()* 表示其中的内容可以重复零次或多次
    // 在展开侧，$()* 表示"对每次匹配，重复这里的代码"
    ($($val:expr),* $(,)?) => {
        $(
            println!("value = {}", $val);
        )*
    };
}

fn basic_repeat_demo() {
    print_all!();            // 零次 —— 什么都不打印
    print_all!(1);           // 一次
    print_all!(1, 2, 3, 4);  // 多次
    print_all!(1, 2, 3,);    // 尾部逗号 OK（因为有 $(,)? 可选项）
}
```

#### 分隔符是在重复模式内部的

```rust
// 注意：逗号在 $( ) 内部，不在外部
macro_rules! sum {
    // $( $x:expr ),*  表示"一个 expr，后跟逗号，这个组合可重复"
    ($($x:expr),* $(,)?) => {
        0 $( + $x )*
    };
    // 展开逻辑：
    // sum!(1, 2, 3)
    // → 0 + 1 + 2 + 3 = 6
}

fn separator_demo() {
    let s1: i32 = sum!();
    let s2: i32 = sum!(10, 20, 30);
    println!("s1={}, s2={}", s1, s2); // s1=0, s2=60
}
```

#### `vec!` 宏的完整实现分析

```rust
// 标准库 vec! 的精简版实现
// 来源：Rust 源码中 vec! 的简化版本
macro_rules! vec {
    // 规则1: 空 Vec
    () => {
        Vec::new()
    };

    // 规则2: vec![elem; n] —— 重复模式
    // 如 vec![0; 10] 展开为:
    //   let mut v = Vec::with_capacity(10);
    //   v.resize(10, 0);
    //   v
    ($elem:expr; $n:expr) => {{
        let mut v = Vec::with_capacity($n);
        v.resize($n, $elem);
        v
    }};

    // 规则3: vec![a, b, c] —— 逗号分隔的值列表
    // $( $x:expr ),* $( ,)? 解析:
    //   $($x:expr),*  —— 匹配 "expr," 的重复序列
    //   $(,)?         —— 可选的尾随逗号
    ($($x:expr),* $(,)?) => {{
        // 计算元素个数（通过临时数组 trick）
        // 这里也可以用 Vec::new() + push，但标准库用这个技巧:
        let mut v = Vec::new();
        $(
            v.push($x);
        )*
        v
    }};
}
```

#### `hashmap!` 宏实现

```rust
// 创建 HashMap 字面量
macro_rules! hashmap {
    // 空 map
    () => {
        ::std::collections::HashMap::new()
    };
    // key => value 对，以逗号分隔
    ($($key:expr => $value:expr),* $(,)?) => {{
        let mut map = ::std::collections::HashMap::new();
        $(
            map.insert($key, $value);
        )*
        map
    }};
}

fn hashmap_demo() {
    let ages = hashmap! {
        "Alice" => 30,
        "Bob" => 25,
        "Charlie" => 35,
    };
    println!("Alice's age: {:?}", ages.get("Alice")); // Some(30)
}
```

#### 嵌套重复

```rust
// 嵌套重复允许同时遍历多个序列
macro_rules! zip_map {
    // 两个独立的重复组：keys 和 values
    ($($key:expr => $value:expr),* $(,)?) => {{
        let mut map = ::std::collections::HashMap::new();
        $(
            map.insert($key, $value);
        )*
        map
    }};
    // 另一种风格：并行列表
    // keys: [ $( $k:expr ),* ] values: [ $( $v:expr ),* ]
}

// 更复杂的嵌套：创建多个结构体
macro_rules! make_structs {
    (
        $(
            $name:ident {
                $($field:ident: $ty:ty),* $(,)?
            }
        )*
    ) => {
        $(
            #[derive(Debug)]
            struct $name {
                $($field: $ty),*
            }
        )*
    };
}

make_structs! {
    Point { x: i32, y: i32 }
    Rectangle { width: f64, height: f64 }
}

fn nested_repeat_demo() {
    let p = Point { x: 1, y: 2 };
    let r = Rectangle { width: 10.0, height: 5.0 };
    println!("{:?} {:?}", p, r);
}
```

### 2.4 $crate 与 Hygiene（卫生性）

#### 什么是宏卫生？

"宏卫生"（Macro Hygiene）指的是**宏展开不会意外地与调用处的变量或路径发生冲突**。

```rust
// C 的宏不卫生 —— 经典 bug:
// #define SWAP(a, b) { int tmp = a; a = b; b = tmp; }
//
// int tmp = 10;
// SWAP(x, tmp);  // 灾难！内部的 tmp 和外部的 tmp 冲突

// Rust 的 macro_rules! 是部分卫生的：
macro_rules! swap {
    ($a:expr, $b:expr) => {
        let tmp = $a;
        $a = $b;
        $b = tmp;
    };
}

fn hygiene_demo() {
    let mut x = 1;
    let mut y = 2;
    let tmp = 42; // 外部的 tmp

    swap!(x, y);
    // 展开后，宏内部的 tmp 和外部的 tmp 是不同的变量
    // Rust 编译器会为宏内部引入的标识符创建"卫生屏障"

    println!("x={}, y={}, tmp={}", x, y, tmp);
    // 输出: x=2, y=1, tmp=42  —— tmp 没有被修改！
}
```

#### 卫生性的"部分"之处

macro_rules! 的卫生性主要体现在**局部变量**和**标签**上，但对于**路径**和**类型名**并不完全卫生：

```rust
// 好的情况：局部变量是卫生的
macro_rules! make_fn {
    () => {
        fn helper() -> i32 { 42 }
    };
}

// 不卫生的情况1：路径引用
// 如果宏定义在 crate A 中，被 crate B 使用，
// 宏内部的路径需要在调用处 resolve —— 可能导致失败

// 不卫生的情况2：宏可以"捕获"外部标识符
macro_rules! use_outer {
    () => {
        println!("{}", outer_var); // 会引用调用处的 outer_var
    };
}

fn partial_hygiene_demo() {
    let outer_var = "hello";
    use_outer!(); // 输出 "hello" —— 宏"看到"了外部变量
}
```

#### `$crate` —— 解决跨 crate 路径问题

当宏被**重新导出**时，内部的 crate 路径会出错：

```rust
// ============ 问题场景 ============
// crate: my_lib
pub mod helpers {
    pub struct Helper;
}

#[macro_export]
macro_rules! bad_make_helper {
    () => {
        // 直接写 crate 路径 —— 当宏被其他 crate 使用时，这个路径不解析
        crate::helpers::Helper
    };
}

#[macro_export]
macro_rules! good_make_helper {
    () => {
        // $crate 总是展开为定义该宏的 crate
        $crate::helpers::Helper
    };
}
// =================================

// 在 my_lib 内部使用，两者都可以工作
// 但如果在 external_crate 中使用，bad_make_helper 会失败
// 因为 external_crate 中没有 crate::helpers::Helper
// 而 good_make_helper 中的 $crate 指向 my_lib，正确！
```

**`$crate` 使用规则**：
- `$crate` 只能在宏展开中使用
- 它展开为定义该宏的 crate 根路径
- 当宏在本地 crate 使用时，`$crate` 就是 `crate`
- 当宏被外部 crate 使用时，`$crate` 指向该外部 crate 的名字

```rust
// 实用示例：导出一个宏，该宏使用了本 crate 的内部类型
#[macro_export]
macro_rules! my_vec {
    ($($x:expr),* $(,)?) => {
        // 使用 $crate 引用内部的构造器
        $crate::raw_vec::from_elements(&[$($x),*])
    };
}
```

#### paste crate 与标识符拼接

macro_rules! 本身**不支持标识符拼接**。但是 `paste` crate 提供了解决方案：

```rust
// Cargo.toml:
// [dependencies]
// paste = "1"

use paste::paste;

// 使用 paste! 宏拼接标识符
macro_rules! make_getters {
    ($($field:ident: $ty:ty),* $(,)?) => {
        paste! {
            $(
                // [< >] 中的内容会被拼接
                fn [<get_ $field>](&self) -> &$ty {
                    &self.$field
                }
            )*
        }
    };
}

struct Person {
    name: String,
    age: u32,
}

impl Person {
    // 这会生成 get_name() 和 get_age() 方法
    make_getters!(name: String, age: u32);
}
```

### 2.5 递归宏

macro_rules! 支持递归调用——一个宏在展开中调用自身，直到匹配到基础情况。

#### `count!()` —— 统计参数个数

```rust
// 递归统计传入的参数数量
macro_rules! count {
    // 基础情况：没有参数
    () => { 0usize };

    // 递归情况：剥离第一个参数，对剩余参数递归计数
    ($head:expr $(, $tail:expr)* $(,)?) => {
        1usize + count!($($tail),*)
    };
}

fn count_demo() {
    let n0: usize = count!();
    let n1: usize = count!(a);
    let n3: usize = count!(a, b, c);
    assert_eq!(n0, 0);
    assert_eq!(n1, 1);
    assert_eq!(n3, 3);

    println!("count!() = {}", n0);
    println!("count!(a, b, c) = {}", n3);
}
```

#### 利用递归实现 `hashmap!`

```rust
// 使用递归 + 内部规则模式（下节详解）
macro_rules! hashmap_recursive {
    // 公共入口：接收键值对列表
    ($($key:expr => $value:expr),* $(,)?) => {
        hashmap_recursive!(@build $($key => $value),*)
    };

    // 内部规则：递归构建
    // 基础情况：空列表
    (@build) => {
        ::std::collections::HashMap::new()
    };

    // 递归情况：处理第一对，然后递归处理剩余的
    (@build $key:expr => $value:expr $(, $rest_key:expr => $rest_value:expr)* $(,)?) => {{
        let mut map = hashmap_recursive!(@build $($rest_key => $rest_value),*);
        map.insert($key, $value);
        map
    }};
}

fn hashmap_recursive_demo() {
    let map = hashmap_recursive!(
        "a" => 1,
        "b" => 2,
        "c" => 3,
    );
    assert_eq!(map.get("a"), Some(&1));
    assert_eq!(map.get("b"), Some(&2));
    println!("hashmap_recursive: {:?}", map);
}
```

### 2.6 TT Muncher 模式

TT Muncher 是 declarative macro 中最强大和灵活的模式。核心理念：

> **将一个 token tree 视为链表，逐个"啃掉" head，递归处理 tail，在递归的归阶段组装结果。**

#### 为什么需要 TT Muncher？

普通的重复模式 `$( ... )*` 只能处理**简单的平坦列表**。当遇到嵌套结构（如 JSON 对象中的数组、数组中的对象）时，重复模式不够用。TT Muncher 可以处理任意深度的嵌套。

#### 基础 TT Muncher 模板

```rust
macro_rules! tt_muncher_template {
    // 基础情况：全部处理完毕
    () => {
        // 最终结果
    };

    // 递归情况：处理第一个 token，然后继续
    ($head:tt $($tail:tt)*) => {
        // 处理 $head
        // 递归调用处理 $tail
        tt_muncher_template!($($tail)*)
    };
}
```

#### 用 TT Muncher 重新实现 `vec!`

```rust
macro_rules! vec_ttm {
    // 公共入口
    ($($tokens:tt)*) => {
        // 开始 munching，用一个内部栈记录收集到的元素
        vec_ttm!(@collect [] $($tokens)*)
    };

    // 基础情况：所有 token 处理完毕，构建 Vec
    (@collect [$($elem:expr),*]) => {{
        let mut v = Vec::new();
        $(
            v.push($elem);
        )*
        v
    }};

    // 遇到逗号：跳过
    (@collect [$($elem:expr),*] , $($tail:tt)*) => {
        vec_ttm!(@collect [$($elem),*] $($tail)*)
    };

    // 处理一个元素
    (@collect [$($elem:expr),*] $head:expr $(, $($tail:tt)*)?) => {
        vec_ttm!(@collect [$($elem,)* $head] $($($tail)*)?)
    };

    // 处理末尾可能的逗号
    (@collect [$($elem:expr),*] ,) => {
        vec_ttm!(@collect [$($elem),*])
    };
}

fn tt_muncher_demo() {
    let v = vec_ttm!(1, 2, 3, 4, 5);
    println!("TT muncher vec: {:?}", v);
}
```

#### TT Muncher 处理嵌套结构

```rust
// 一个简化的 S-表达式解析器
// 能够处理 (a (b c) d) 这样的嵌套结构
macro_rules! sexpr {
    // 外部入口
    ($($t:tt)*) => {
        sexpr!(@parse [] $($t)*)
    };

    // 遇到 ( 开始新的列表
    (@parse [$($acc:tt)*] ( $($inner:tt)* ) $($rest:tt)*) => {
        sexpr!(@parse [$($acc)* [ sexpr!(@parse [] $($inner)*) ]] $($rest)*)
    };

    // 遇到原子
    (@parse [$($acc:tt)*] $atom:ident $($rest:tt)*) => {
        sexpr!(@parse [$($acc)* $atom] $($rest)*)
    };

    // 结束
    (@parse [$($acc:tt)*]) => {
        vec![$($acc),*]
    };
}
```

#### TT Muncher vs 重复模式对比

| 特性 | 重复模式 `$(...)*` | TT Muncher |
|------|---------------------|------------|
| 语法复杂度 | 低 | 高 |
| 处理平坦列表 | 优秀 | 可以（但没必要） |
| 处理嵌套结构 | 困难/不可能 | 强大 |
| 编译时间 | 快 | 较慢 |
| 展开深度 | 恒定 O(1) | O(n)，受递归限制 |

### 2.7 内部规则模式（Internal Rules）

内部规则是 macro_rules! 的设计模式：用特殊标记（通常是 `@`）将公共接口与内部实现分离。

#### 基本模式

```rust
macro_rules! my_macro {
    // ======== 公共入口 ========
    ($user_input:expr) => {
        // 调用内部规则，@marker 用于区分
        my_macro!(@internal $user_input, initial_state)
    };

    // ======== 内部规则 ========
    (@internal $input:expr, $state:ident) => {
        // 核心实现逻辑
    };
}
```

`@` 只是一个约定，你可以用其他标记。它不会被匹配为表达式、标识符或其他捕获类型——它只是一个 token tree。这确保了公共接口和内部规则不会意外冲突。

#### 用内部规则实现 `json!` 宏

```rust
#[derive(Debug, Clone, PartialEq)]
enum JsonValue {
    Null,
    Bool(bool),
    Number(f64),
    String(String),
    Array(Vec<JsonValue>),
    Object(Vec<(String, JsonValue)>),
}

macro_rules! json_internal {
    // ======== 公共入口 ========
    // 对象: { "key": value, ... }
    ({ $($input:tt)* }) => {
        json_internal!(@object [] $($input)*)
    };
    // 数组: [ elements... ]
    ([ $($input:tt)* ]) => {
        json_internal!(@array [] $($input)*)
    };
    // 字面量
    (null) => { JsonValue::Null };
    (true) => { JsonValue::Bool(true) };
    (false) => { JsonValue::Bool(false) };
    ($num:literal) => { JsonValue::Number($num as f64) };
    ($s:literal) => { JsonValue::String($s.to_string()) };

    // ======== 内部规则：解析对象 ========
    // 基础情况：对象解析完毕
    (@object [$(($key:expr, $val:expr)),*]) => {
        JsonValue::Object(vec![
            $(($key.to_string(), $val)),*
        ])
    };

    // 处理 key: value 对
    (@object [$(($key:expr, $val:expr)),*] $k:literal : $v:tt $($rest:tt)*) => {
        json_internal!(@object [$(($key, $val)),* ( $k, json_internal!($v) )] $($rest)*)
    };

    // 跳过逗号
    (@object [$($pairs:tt)*] , $($rest:tt)*) => {
        json_internal!(@object [$($pairs)*] $($rest)*)
    };

    // ======== 内部规则：解析数组 ========
    // 基础情况：数组解析完毕
    (@array [$($elem:expr),*]) => {
        JsonValue::Array(vec![$($elem),*])
    };

    // 处理元素
    (@array [$($elem:expr),*] $first:tt $($rest:tt)*) => {
        json_internal!(@array [$($elem,)* json_internal!($first)] $($rest)*)
    };

    // 跳过逗号
    (@array [$($elem:expr),*] , $($rest:tt)*) => {
        json_internal!(@array [$($elem),*] $($rest)*)
    };
}

fn json_internal_demo() {
    let obj = json_internal!({
        "name": "Alice",
        "age": 30,
        "scores": [95, 87, 92],
        "address": {
            "city": "Beijing",
            "zip": "100000"
        },
        "active": true,
        "middle_name": null
    });
    println!("{:#?}", obj);
}
```

#### `@` 标记的其他用途

```rust
// 用途1: 状态机标记
macro_rules! parser {
    (@start $input:expr) => { /* 初始状态 */ };
    (@parsing_key $input:expr) => { /* 解析键 */ };
    (@parsing_value $input:expr) => { /* 解析值 */ };
    (@done $result:expr) => { $result };

// 用途2: 调试标记
macro_rules! debug_build {
    (@debug $msg:expr) => { eprintln!("[DEBUG] {}", $msg); };
    (@release $msg:expr) => { /* 什么都不做 */ };

        } else {
            debug_build!(@release $msg);
        }
    };
}

// 用途3: 累加器模式
macro_rules! accum {
    // 公共接口：接收输入，从空累加器开始
    ($($input:tt)*) => {
        accum!(@acc [] $($input)*)
    };
    // 内部规则：累加器处理
    (@acc [$($processed:tt)*]) => {
        // 全部处理完毕，输出结果
        stringify!($($processed)*)
    };
    (@acc [$($processed:tt)*] $next:tt $($rest:tt)*) => {
        // 处理 $next，加入累加器
        accum!(@acc [$($processed)* # $next] $($rest)*)
    };
}
```

**Bug 1: 宏引入的变量覆盖了调用者的变量**

```rust

        let tmp = $a;

    };

    let mut y = 2;
    let mut tmp = 999;

    // 即使展开后它们看起来同名，编译器会给宏内部变量"着色"

macro_rules! with_temp_file_buggy {

        // 前缀下划线 + 描述性名称

            std::env::temp_dir().join("temp.dat")
        }
        let path = __macro_generated_temp_file_helper();
        let result = $body;
        std::fs::remove_file(&path).ok();
        result
    }};
}

// 修复方案2: 让宏不引入名称，而是使用闭包或表达式

        let path = {
            std::env::temp_dir().join("temp.dat")

        let _ = std::fs::remove_file(&path);
        result
    }};
}
```

```rust
// ============ Bug 分析 ============
//
// 实际上，Rust macro_rules! 的局部变量是卫生的！
// 在 time_it! 宏中声明的 `start` 不会被外部同名变量影响，
```

### 练习 2 答案：TT Muncher 实现 `count!()`

```rust
/// count!() 宏 —— 使用 TT Muncher 统计参数个数
///
/// 设计思路:
/// 1. 公共入口接收任意 token 序列
/// 2. 内部规则 @count(accumulated_count) 维护已统计的数量
/// 3. 每次递归消费一个 token，累加器 +1
/// 4. 基础情况：没有 token 了，输出累加器值

macro_rules! count {
    // ======== 公共入口 ========
    // 接收任意 token 序列，从计数 0 开始
    ($($tokens:tt)*) => {
        count!(@count(0) $($tokens)*)
    };

    // ======== 内部规则 ========

    // 基础情况：所有 token 处理完毕，输出计数
    (@count($n:expr)) => {
        $n
    };

    // 处理逗号：跳过逗号（不计数）
    // 注意：逗号本身也是一个 tt，但我们不把它算作参数
    (@count($n:expr) , $($rest:tt)*) => {
        count!(@count($n) $($rest)*)
    };

    // 处理尾随逗号（逗号后面没有更多内容）
    (@count($n:expr) ,) => {
        count!(@count($n))
    };

    // 递归：消费一个 token，计数 +1
    // 这个规则必须放在逗号规则之后，因为如果放在前面
    // 逗号会被当作普通 token 计数，导致尾随逗号也被算作参数
    (@count($n:expr) $head:tt $($tail:tt)*) => {
        count!(@count($n + 1) $($tail)*)
    };
}

// ============ 测试 ============
fn main() {
    // 基础测试
    assert_eq!(count!(), 0);
    assert_eq!(count!(a), 1);
    assert_eq!(count!(a, b), 2);
    assert_eq!(count!(a, b, c, d, e), 5);

    // 不同类型 token
    assert_eq!(count!(let, x, =, 1), 4);
    assert_eq!(count!(fn, foo, (), {}), 4);
    assert_eq!(count!(a, b, c,), 3);  // 尾随逗号

    // 复杂 token
    assert_eq!(
        count!(struct Foo { x: i32, y: i32 }),
        7
    );
    // 分解: struct / Foo / { x : i32 , y : i32 } = 7 个 token tree

    println!("All count! tests passed!");
}

// ============ 展开追踪（理解递归过程） ============
//
// count!(a, b, c) 的展开过程：
//
// count!(@count(0) a , b , c)     // 入口
// → match $head:tt = a
// → count!(@count(0 + 1) , b , c) // a 被消费，计数 +1
// → match $head:tt = ,           // 逗号规则匹配
// → count!(@count(1) b , c)      // 逗号被跳过
// → match $head:tt = b
// → count!(@count(1 + 1) , c)    // b 被消费
// → match $head:tt = ,
// → count!(@count(2) c)          // 逗号被跳过
// → match $head:tt = c
// → count!(@count(2 + 1))        // c 被消费
// → 基础情况匹配 → 输出 3
```

### 练习 3 答案：完整的 `json!` 宏

```rust
use std::collections::HashMap;

// ============ JSON 值类型 ============
#[derive(Debug, Clone, PartialEq)]
pub enum Json {
    Null,
    Bool(bool),
    Number(f64),
    String(String),
    Array(Vec<Json>),
    Object(HashMap<String, Json>),
}

// ============ json! 宏实现 ============
// 架构：
//   公共入口层: json!({...}) / json!([...]) / json!(literal)
//   内部规则层: @object / @array / @literal
//
// 数据流：
//   输入 tokens → @object/@array 递归下降 → Json 值
//
// 使用 TT Muncher 逐 token 处理，用 @ 标记区分模式

macro_rules! json {
    // ==========================================
    // 公共入口层
    // ==========================================

    // 对象: { ... }
    ({ $($tt:tt)* }) => {
        json!(@object (::std::collections::HashMap::new()) [] $($tt)*)
    };

    // 数组: [ ... ]
    ([ $($tt:tt)* ]) => {
        json!(@array (Vec::new()) [] $($tt)*)
    };

    // 字面量: null, true, false, 数字, 字符串
    (null) => { Json::Null };
    (true) => { Json::Bool(true) };
    (false) => { Json::Bool(false) };

    // 数字 — 注意 literal 匹配，需要判断类型
    ($num:literal) => {
        // 使用 trait 方法在编译期做类型判断
        // 简化处理：所有字面量数字都转为 f64
        Json::Number(json!(@to_f64 $num))
    };

    // 字符串
    ($s:literal) => {
        // 注意：这里 $s 匹配的是任意字面量
        // 在 parse 阶段，我们已经使用了 tt 来区分字符串
        // 这个分支作为后备
        Json::String($s.to_string())
    };

    // ==========================================
    // 内部规则: @object — 解析 JSON 对象
    // ==========================================
    //
    // 状态: @object(正在构建的HashMap, 当前key: Option, 待处理的tokens...)
    //
    // 基础情况: 对象解析完毕
    (@object ($map:expr) [] ) => {
        $map
    };

    // 处理 "key": value 中的 key（字符串字面量）
    (@object ($map:expr) [] $key:literal : $($rest:tt)*) => {{
        let mut map = $map;
        let key = $key.to_string();
        // 回调自己来解析 value（可能是任何 JSON 类型）
        // 使用另一个内部辅助来简化
        json!(@object_value (map, key) [] $($rest)*)
    }};

    // 跳过逗号
    (@object ($map:expr) [] , $($rest:tt)*) => {
        json!(@object ($map) [] $($rest)*)
    };

    // 跳过尾随逗号
    (@object ($map:expr) [] ,) => {
        json!(@object ($map) [] )
    };

    // ==========================================
    // 内部规则: @object_value — 解析对象中的 value
    // ==========================================
    // 状态: @object_value((map, key), 待处理的tokens...)

    // value 是 null
    (@object_value ($map:expr, $key:expr) [] null $($rest:tt)*) => {{
        let mut map = $map;
        map.insert($key, Json::Null);
        json!(@object (map) [] $($rest)*)
    }};

    // value 是 true/false
    (@object_value ($map:expr, $key:expr) [] true $($rest:tt)*) => {{
        let mut map = $map;
        map.insert($key, Json::Bool(true));
        json!(@object (map) [] $($rest)*)
    }};

    (@object_value ($map:expr, $key:expr) [] false $($rest:tt)*) => {{
        let mut map = $map;
        map.insert($key, Json::Bool(false));
        json!(@object (map) [] $($rest)*)
    }};

    // value 是数字字面量
    (@object_value ($map:expr, $key:expr) [] $num:literal $($rest:tt)*) => {{
        let mut map = $map;
        map.insert($key, Json::Number(json!(@to_f64 $num)));
        json!(@object (map) [] $($rest)*)
    }};

    // value 是嵌套对象 { ... }
    // 需要找到匹配的 } —— 用 tt muncher 收集嵌套内容
    (@object_value ($map:expr, $key:expr) [] { $($inner:tt)* } $($rest:tt)*) => {{
        let mut map = $map;
        map.insert($key, json!({ $($inner)* }));
        json!(@object (map) [] $($rest)*)
    }};

    // value 是嵌套数组 [ ... ]
    (@object_value ($map:expr, $key:expr) [] [ $($inner:tt)* ] $($rest:tt)*) => {{
        let mut map = $map;
        map.insert($key, json!([ $($inner)* ]));
        json!(@object (map) [] $($rest)*)
    }};

    // 跳过 value 后面的逗号
    (@object_value ($map:expr, $key:expr) [] , $($rest:tt)*) => {
        json!(@object ($map) [] $($rest)*)
    };

    // ==========================================
    // 内部规则: @array — 解析 JSON 数组
    // ==========================================
    // 状态: @array(正在构建的Vec, 待处理的tokens...)

    // 基础情况: 空数组
    (@array ($vec:expr) [] ) => {
        $vec
    };

    // 数组元素: null
    (@array ($vec:expr) [] null $($rest:tt)*) => {{
        let mut vec = $vec;
        vec.push(Json::Null);
        json!(@array (vec) [] $($rest)*)
    }};

    // 数组元素: true
    (@array ($vec:expr) [] true $($rest:tt)*) => {{
        let mut vec = $vec;
        vec.push(Json::Bool(true));
        json!(@array (vec) [] $($rest)*)
    }};

    // 数组元素: false
    (@array ($vec:expr) [] false $($rest:tt)*) => {{
        let mut vec = $vec;
        vec.push(Json::Bool(false));
        json!(@array (vec) [] $($rest)*)
    }};

    // 数组元素: 数字
    (@array ($vec:expr) [] $num:literal $($rest:tt)*) => {{
        let mut vec = $vec;
        vec.push(Json::Number(json!(@to_f64 $num)));
        json!(@array (vec) [] $($rest)*)
    }};

    // 数组元素: 嵌套对象
    (@array ($vec:expr) [] { $($inner:tt)* } $($rest:tt)*) => {{
        let mut vec = $vec;
        vec.push(json!({ $($inner)* }));
        json!(@array (vec) [] $($rest)*)
    }};

    // 数组元素: 嵌套数组
    (@array ($vec:expr) [] [ $($inner:tt)* ] $($rest:tt)*) => {{
        let mut vec = $vec;
        vec.push(json!([ $($inner)* ]));
        json!(@array (vec) [] $($rest)*)
    }};

    // 跳过逗号
    (@array ($vec:expr) [] , $($rest:tt)*) => {
        json!(@array ($vec) [] $($rest)*)
    };

    // 跳过尾随逗号
    (@array ($vec:expr) [] ,) => {
        json!(@array ($vec) [] )
    };

    // ==========================================
    // 辅助: @to_f64 — 安全地将字面量转为 f64
    // ==========================================
    // 这个辅助利用 Rust 的 trait 解析
    (@to_f64 $num:literal) => {
        {
            // 利用 Rust 的类型推断：字面量可以是任意数值类型
            // 这里显式转为 f64 以确保一致性
            let n: f64 = $num as f64;
            n
        }
    };
}

// ============ 测试 ============
fn main() {
    // === 基础类型 ===
    let null_val = json!(null);
    assert_eq!(null_val, Json::Null);

    let bool_val = json!(true);
    assert_eq!(bool_val, Json::Bool(true));

    let num_val = json!(42);
    assert_eq!(num_val, Json::Number(42.0));

    // === 简单对象 ===
    let simple_obj = json!({
        "name": "Alice",
        "age": 30
    });

    if let Json::Object(ref map) = simple_obj {
        assert_eq!(map.get("name"), Some(&Json::String("Alice".to_string())));
        assert_eq!(map.get("age"), Some(&Json::Number(30.0)));
    } else {
        panic!("Expected object");
    }

    // === 简单数组 ===
    let simple_arr = json!([1, 2, 3, 4, 5]);
    if let Json::Array(ref arr) = simple_arr {
        assert_eq!(arr.len(), 5);
        assert_eq!(arr[0], Json::Number(1.0));
        assert_eq!(arr[4], Json::Number(5.0));
    } else {
        panic!("Expected array");
    }

    // === 嵌套结构 ===
    let complex = json!({
        "user": {
            "name": "Bob",
            "scores": [95, 87, 92],
            "address": {
                "city": "Shanghai",
                "zip": "200000"
            }
        },
        "active": true,
        "tags": ["rust", "programming"],
        "metadata": null
    });

    println!("{:#?}", complex);

    // 验证嵌套结构
    if let Json::Object(ref map) = complex {
        assert_eq!(map.get("active"), Some(&Json::Bool(true)));
        assert_eq!(map.get("metadata"), Some(&Json::Null));

        if let Some(Json::Object(ref user)) = map.get("user") {
            if let Some(Json::Array(ref scores)) = user.get("scores") {
                assert_eq!(scores.len(), 3);
            } else {
                panic!("Expected scores array");
            }
        } else {
            panic!("Expected user object");
        }
    } else {
        panic!("Expected root object");
    }

    println!("All json! tests passed!");
}
```
