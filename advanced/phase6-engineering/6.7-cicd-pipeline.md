# 6.7 CI/CD 管道设计 —— 让机器替你记住每一个检查

> 人类的注意力是稀缺资源。你不可能在每次提交前都记得跑 `cargo test && cargo clippy && cargo fmt --check && cargo audit && cargo deny check`。你更不可能手动构建 4 个平台的二进制文件然后上传到 GitHub Releases。CI/CD 的价值不在于"自动化"本身——而在于它把最佳实践从"靠自觉"变成了"强制要求"。一个好的 CI 管道是一份**可执行的质量标准**：lint 不过你就合并不了，测试失败你就发布不了，安全漏洞不修你就睡不着。本节以 GitHub Actions 为主线，构建一个生产级的 Rust CI/CD 管道——从快速的 PR 检查到慢速的 nightly 全面扫描，从手动触发发布到自动通知。

## 1. GitHub Actions 基础

### 1.1 最小的 Rust CI

```yaml
# .github/workflows/ci.yml
name: CI

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Install Rust
        uses: dtolnay/rust-toolchain@stable

      - name: Cache dependencies
        uses: actions/cache@v4
        with:
          path: |
            ~/.cargo/registry
            ~/.cargo/git
            target
          key: ${{ runner.os }}-cargo-${{ hashFiles('**/Cargo.lock') }}

      - name: Run tests
        run: cargo test --all-features

      - name: Run clippy
        run: cargo clippy --all-features -- -D warnings

      - name: Check formatting
        run: cargo fmt --all -- --check
```

**这个最简 CI 做了什么**：
1. 每次 push 到 main 或创建 PR 时触发
2. 安装 Rust 稳定版
3. 缓存编译产物（加速后续构建）
4. 运行测试
5. 运行 clippy（`-D warnings` 意味着任何 warning 都当作 error）
6. 检查代码格式（`--check` 意味着只检查不修改）

### 1.2 缓存策略详解

```yaml
# ━━━ 更好的缓存策略 ━━━
- name: Cache cargo registry
  uses: actions/cache@v4
  with:
    path: ~/.cargo/registry
    key: cargo-registry-${{ hashFiles('**/Cargo.lock') }}
    restore-keys: |
      cargo-registry-

- name: Cache cargo index
  uses: actions/cache@v4
  with:
    path: ~/.cargo/git
    key: cargo-index-${{ hashFiles('**/Cargo.lock') }}

- name: Cache build artifacts
  uses: actions/cache@v4
  with:
    path: target
    key: ${{ runner.os }}-target-${{ hashFiles('**/Cargo.lock') }}
    restore-keys: |
      ${{ runner.os }}-target-
```

**缓存分层策略的原因**：
- `~/.cargo/registry`：下载的 crate 源码（变化频率最低）
- `~/.cargo/git`：git 依赖（如果有的话）
- `target/`：编译产物（变化频率最高）
- 分层使得即使最常变化的 `target/` 缓存失效，registry 缓存仍然有效

## 2. Matrix 测试 —— 多维度覆盖

### 2.1 多平台 × 多 Rust 版本

```yaml
jobs:
  test-matrix:
    strategy:
      fail-fast: false  # ← 一个平台失败不取消其他平台
      matrix:
        os: [ubuntu-latest, macos-latest, windows-latest]
        rust: [stable, beta, "1.75"]  # MSRV 测试
        features: ["", "--no-default-features", "--all-features"]
        exclude:
          # 排除不合理的组合
          - os: windows-latest
            rust: beta

    runs-on: ${{ matrix.os }}
    steps:
      - uses: actions/checkout@v4
      - uses: dtolnay/rust-toolchain@master
        with:
          toolchain: ${{ matrix.rust }}

      - name: Test
        run: cargo test ${{ matrix.features }}
```

**Matrix 的价值**：
- **多平台**：你的代码在 Linux 上跑了 10000 次不代表在 Windows 上能编译
- **多版本**：`stable` 保证当前可用，`beta` 提前发现下个版本的问题，MSRV（1.75）保证你声明的"最低支持版本"是真的
- **多 features**：无 feature、默认 features、全部 features——三种都测到

### 2.2 Cargo Feature 矩阵

```yaml
jobs:
  feature-matrix:
    strategy:
      fail-fast: false
      matrix:
        feature:
          - "json"
          - "yaml"
          - "json,yaml"
          - "async"
          - "json,async"
    steps:
      - uses: actions/checkout@v4
      - uses: dtolnay/rust-toolchain@stable

      - name: Install cargo-hack
        uses: taiki-e/install-action@cargo-hack

      - name: Test feature combinations
        run: cargo hack test --feature-powerset --depth 2
        # --depth 2: 只测试 1-2 层的 feature 组合（避免组合爆炸）
```

## 3. 质量关卡 —— CI 中的"红灯"

### 3.1 所有质量检查整合

```yaml
jobs:
  # ━━━ 快速检查（每次 push）━━━
  quick-checks:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: dtolnay/rust-toolchain@stable
        with:
          components: rustfmt, clippy

      - name: Format check
        run: cargo fmt --all -- --check

      - name: Clippy
        run: cargo clippy --all-targets --all-features -- -D warnings

      - name: Doc tests
        run: cargo test --doc

  # ━━━ 完整测试（每次 push）━━━
  test:
    needs: quick-checks  # ← 格式检查过了才跑测试
    strategy:
      matrix:
        os: [ubuntu-latest, macos-latest, windows-latest]
    runs-on: ${{ matrix.os }}
    steps:
      - uses: actions/checkout@v4
      - uses: dtolnay/rust-toolchain@stable
      - run: cargo test --all-features

  # ━━━ 安全审计（每天定时 + Cargo.toml 变更时）━━━
  security:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: dtolnay/rust-toolchain@stable
      - uses: taiki-e/install-action@cargo-audit
      - run: cargo audit

  # ━━━ 依赖检查（每天定时）━━━
  deny:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: taiki-e/install-action@cargo-deny
      - run: cargo deny check

  # ━━━ 文档构建（每次 push）━━━
  docs:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: dtolnay/rust-toolchain@nightly
      - run: cargo doc --no-deps --all-features
        env:
          RUSTDOCFLAGS: "-D warnings"  # 文档中的警告也当作错误
```

### 3.2 什么时候运行什么检查

```
┌──────────────────┬─────────────┬──────────────────────────────────┐
│ 检查             │ 运行频率     │ 理由                              │
├──────────────────┼─────────────┼──────────────────────────────────┤
│ fmt              │ 每次 push   │ 格式检查极快（几秒），即时反馈     │
│ clippy           │ 每次 push   │ lint 是"代码味道"的第一道防线     │
│ test             │ 每次 push   │ 核心质量保证，必须每次跑           │
│ doc              │ 每次 push   │ 文档示例代码可能被重构破坏         │
│ cargo-audit      │ 每天 + PR   │ 每天 + Cargo.toml 变更时触发      │
│ cargo-deny       │ 每天 + PR   │ 同上，还检查许可证                 │
│ cargo-outdated   │ 每周       │ 不需要即时，但应该定期提醒          │
│ Miri             │ PR + 每天  │ 慢，但 unsafe 代码需要             │
│ Fuzz             │ 每天       │ 非常慢，nightly 即可                │
│ Cross-compile    │ 发版时     │ 耗时很长，只在发布前跑              │
└──────────────────┴─────────────┴──────────────────────────────────┘
```

## 4. 发布自动化

### 4.1 自动发布到 crates.io

```yaml
# .github/workflows/release.yml
name: Release

on:
  push:
    tags:
      - 'v*'  # 只在推送 v1.0.0 这类 tag 时触发

jobs:
  publish:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: dtolnay/rust-toolchain@stable

      - name: Publish to crates.io
        run: cargo publish --token ${{ secrets.CRATES_IO_TOKEN }}
```

### 4.2 自动构建多平台二进制 + GitHub Releases

```yaml
name: Release Binaries

on:
  push:
    tags: ['v*']

jobs:
  build:
    strategy:
      matrix:
        target:
          - x86_64-unknown-linux-gnu
          - x86_64-apple-darwin
          - x86_64-pc-windows-msvc
          - aarch64-unknown-linux-gnu
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: dtolnay/rust-toolchain@stable

      - name: Install cross
        run: cargo install cross

      - name: Build
        run: cross build --release --target ${{ matrix.target }}

      - name: Upload artifact
        uses: actions/upload-artifact@v4
        with:
          name: binary-${{ matrix.target }}
          path: target/${{ matrix.target }}/release/my-binary${{ matrix.target == 'x86_64-pc-windows-msvc' && '.exe' || '' }}

  release:
    needs: build
    runs-on: ubuntu-latest
    steps:
      - uses: actions/download-artifact@v4
      - name: Create GitHub Release
        uses: softprops/action-gh-release@v2
        with:
          files: binary-*/*
          generate_release_notes: true
```

## 5. PR 流程最佳实践

### 5.1 Branch Protection Rules

在 GitHub 仓库的 Settings → Branches → Add rule 中：

```
Branch name pattern: main

  ☑ Require a pull request before merging
    ☑ Require approvals (1)

  ☑ Require status checks to pass before merging
    ☑ test (ubuntu-latest)
    ☑ test (macos-latest)
    ☑ test (windows-latest)
    ☑ clippy
    ☑ fmt

  ☑ Require conversation resolution before merging
```

### 5.2 PR 模板

```markdown
<!-- .github/PULL_REQUEST_TEMPLATE.md -->

## Summary
<!-- 简要描述这个 PR 做了什么 -->

## Checklist
- [ ] 添加了必要的测试
- [ ] 更新了文档（如果有 API 变更）
- [ ] `cargo test --all-features` 通过
- [ ] `cargo clippy --all-features` 无 warning
- [ ] `cargo fmt --all` 通过
- [ ] 如果是 breaking change，在 CHANGELOG 中记录
```

## 7. 练习答案

```yaml
name: CI

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

jobs:
  test:
    strategy:
      matrix:
        os: [ubuntu-latest, macos-latest, windows-latest]
    runs-on: ${{ matrix.os }}
    steps:
      - uses: actions/checkout@v4
      - uses: dtolnay/rust-toolchain@stable
      - run: cargo test --all-features

  lint:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: dtolnay/rust-toolchain@stable

          components: rustfmt, clippy
      - run: cargo fmt --all -- --check

      - uses: actions/checkout@v4
```
