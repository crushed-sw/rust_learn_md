# 6.15 Phase 6 综合实战：工程化 Rust 项目从零到发布

> Phase 6 的 14 节课每一节都教了你一项具体技能。现在你需要把它们组合起来——不是"写一个测试 + 配一个 CI"，而是从头设计一个具有良好工程实践的完整 Rust 项目。这个综合实战项目是一个**命令行日志分析工具**（`rlog`）——输入 Apache/Nginx 格式的日志文件，输出统计报告。这个项目足够简单（不会花太多时间理解业务逻辑），又足够完整（需要解析、统计、输出、测试、性能、CI/CD）。通过这个项目，你会体会到工程实践不是"学完就忘的配置项"——它们是每个真实项目中自然生长出来的需求。

## 1. 项目概述：rlog —— Rust Log Analyzer

### 1.1 功能需求

```
输入：Apache/Nginx 格式的 access.log
```

- 错误：`thiserror`（库层） + `anyhow`（CLI 层）

- CLI：`clap`（参数解析）
- 基准：`criterion`
- 测试：单元 + 集成 + `proptest` + `insta`
- CI：GitHub Actions（test + clippy + audit + release）
- 发布：cargo publish + 多平台二进制

```rust
use thiserror::Error;

/// rlog 库的所有错误类型
#[derive(Error, Debug)]
pub enum RlogError {
    #[error("I/O error: {0}")]
    Io(#[from] std::io::Error),

    #[error("parse error at line {line}: {msg}")]
    Parse { line: usize, msg: String },

    #[error("invalid log format at line {line}: {raw}")]
    InvalidFormat { line: usize, raw: String },

    #[error("empty log file")]
    EmptyFile,
}

/// 库函数的 Result 别名
pub type RlogResult<T> = Result<T, RlogError>;
```

```rust
/// 从文件路径解析所有日志行
pub fn parse_file(path: &str) -> RlogResult<Vec<LogEntry>> {
    let content = std::fs::read_to_string(path)?;
    let entries: RlogResult<Vec<LogEntry>> = content
        .lines()
        .enumerate()
        .filter(|(_, line)| !line.is_empty())
        .map(|(i, line)| {
            parse_line(line)
                .map(|(_, entry)| entry)
                .map_err(|e| RlogError::Parse {
                    line: i + 1,
                    msg: format!("{}", e),
                })
        })
        .collect();

    entries
}
```

### 2.4 分析器 (`analyzer.rs`)

```rust
use std::collections::{HashMap, BTreeMap};
use crate::model::{AnalysisReport, LogEntry};

/// 分析日志条目，生成报告
pub fn analyze(entries: &[LogEntry]) -> AnalysisReport {
    let total = entries.len() as u64;

    // 唯一 IP 数
    let unique_ips: std::collections::HashSet<&str> = entries
        .iter()
        .map(|e| e.ip.as_str())
        .collect();

    // 成功请求比例
    let success_count = entries
        .iter()
        .filter(|e| e.status < 400)
        .count();

    // Top URLs
    let mut url_counts: HashMap<&str, u64> = HashMap::new();
    for entry in entries {
        *url_counts.entry(&entry.path).or_default() += 1;
    }
    let mut top_urls: Vec<(String, u64)> = url_counts
        .into_iter()
        .map(|(k, v)| (k.to_string(), v))
        .collect();
    top_urls.sort_by(|a, b| b.1.cmp(&a.1));
    top_urls.truncate(10);

    // 状态码分布
    let mut status_counts: HashMap<u16, u64> = HashMap::with_capacity(20);
    for entry in entries {
        *status_counts.entry(entry.status).or_default() += 1;
    }
    let mut status_dist: Vec<(u16, u64, f64)> = status_counts
        .into_iter()
        .map(|(code, count)| (code, count, count as f64 / total as f64 * 100.0))
        .collect();
    status_dist.sort_by_key(|(code, ..)| *code);

    // 每小时请求分布
    let mut hour_counts: BTreeMap<u32, u64> = BTreeMap::new();
    for entry in entries {
        *hour_counts.entry(entry.timestamp.hour()).or_default() += 1;
    }
    let requests_per_hour: Vec<(u32, u64)> = hour_counts.into_iter().collect();

    AnalysisReport {
        total_requests: total,
        unique_ips: unique_ips.len(),
        success_rate: success_count as f64 / total as f64 * 100.0,
        top_urls,
        status_distribution: status_dist,
        requests_per_hour,
    }
}
```

### 2.5 CLI 入口 (`main.rs`)

```rust
use clap::Parser;
use anyhow::Context;
use tracing::{info, error};

#[derive(Parser)]
#[command(name = "rlog", about = "Fast log analyzer")]
struct Cli {
    /// Input log file path
    #[arg(short, long)]
    input: String,

    /// Output format: text, json
    #[arg(short, long, default_value = "text")]
    format: String,

    /// Top N URLs to display
    #[arg(short = 'n', long, default_value = "10")]
    top_n: usize,
}

fn main() -> anyhow::Result<()> {
    tracing_subscriber::fmt::init();

    let cli = Cli::parse();
    info!("Reading log file: {}", cli.input);

    let entries = rlog::parser::parse_file(&cli.input)
        .context("Failed to parse log file")?;
    info!("Parsed {} entries", entries.len());

    let report = rlog::analyzer::analyze(&entries);

    // 输出报告
    match cli.format.as_str() {
        "json" => println!("{}", serde_json::to_string_pretty(&report)?),
        _ => rlog::reporter::print_text_report(&report, cli.top_n),
    }

    Ok(())
}
```

## 5. 性能基准

### 5.1 Criterion Benchmark

```rust
// benches/parse_bench.rs
use criterion::{black_box, criterion_group, criterion_main, Criterion};

fn generate_test_log(size: usize) -> String {
    // 生成 size 行的测试日志
    let mut s = String::with_capacity(size * 150);
    for i in 0..size {
        s.push_str(&format!(
            r#"127.0.0.1 - - [10/Oct/2023:13:55:36 +0800] "GET /page/{} HTTP/1.1" 200 2326\n"#,
            i
        ));
    }
    s
}

fn bench_parse(c: &mut Criterion) {
    let log_data = generate_test_log(100_000);

    c.bench_function("parse 100k lines", |b| {
        b.iter(|| {
            let lines: Vec<&str> = log_data.lines().collect();
            for line in lines {
                let _ = rlog::parser::parse_line(black_box(line));
            }
        })
    });
}

criterion_group!(benches, bench_parse);
criterion_main!(benches);
```
