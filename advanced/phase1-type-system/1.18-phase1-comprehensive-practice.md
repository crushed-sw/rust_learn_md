## 2. Demo 1: Typestate Builder —— 编译期保证"必填字段"完整性

### 2.1 设计哲学

> "如果你需要写 `if !self.url.is_set() { panic!() }`，说明你的 API 设计有缺陷。Typestate Builder 把运行时的检查变成编译时的类型约束——不合法的状态组合根本**无法表达**。"

这个模式被 Rust 生态中几乎所有 HTTP 客户端、数据库查询构建器、命令行解析器使用，是 Rust 类型系统"零成本抽象"理念的最佳展示。

### 2.2 问题场景

实现一个 HTTP 请求构建器，要求：
- `url` 和 `method` 是**必填字段**
- 在设置 `url` 和 `method` 之前，不能调用 `header()` 和 `body()`（必须先有目标和方法）
- 在调用 `build()` 之前，`url` 和 `method` 必须已设置
- 所有检查在**编译期**完成，零运行时开销

```rust
// 错误的 API（运行时检查）：
let req = Request::new()
    .header("Content-Type", "application/json")  // ← 该不该允许？
    .build();  // ← panic! runtime error!

// Typestate API（编译期检查）：
let req = RequestBuilder::new()
    // .header("...", "...")  // ← 编译错误！NoUrl 没有 header 方法
    .url("https://api.example.com/data")
    // .build()               // ← 编译错误！UrlOnly 没有 build 方法
    .method("POST")
    .header("Content-Type", "application/json")
    .build()                  // ← 只有 Ready 状态才有 build
    .send();
```

### 2.3 实现

#### Step 1: 定义状态标记类型 & Newtype

```rust
use std::marker::PhantomData;

// ━━━ 状态标记：零大小类型，仅在类型层面存在 ━━━
pub struct NoUrl;
pub struct UrlOnly;
pub struct UrlAndMethod;
pub struct Ready;

// ━━━ Newtype: 类型安全的 URL ━━━
// 不是简单的 String，而是一个有语义的类型
#[derive(Debug, Clone, PartialEq, Eq)]
pub struct Url(String);

// From/Into: 转换 trait（Phase 1 1.5）
impl From<&str> for Url {
    fn from(s: &str) -> Self {
        // 真实场景中这里会做 URL 校验
        Url(s.to_string())
    }
}

impl std::fmt::Display for Url {
    fn fmt(&self, f: &mut std::fmt::Formatter<'_>) -> std::fmt::Result {
        write!(f, "{}", self.0)
    }
}
```

#### Step 2: Supertrait —— 约束状态类型

```rust
// ━━━ Supertrait 体系（Phase 1 1.16）━━━
// BuildState 是 supertrait，所有具体状态都必须实现它
// sealed 模式防止外部 crate 实现这个 trait
pub trait BuildState: sealed::Sealed {}

mod sealed {
    pub trait Sealed {}
    impl Sealed for super::NoUrl {}
    impl Sealed for super::UrlOnly {}
    impl Sealed for super::UrlAndMethod {}
    impl Sealed for super::Ready {}
}

impl BuildState for NoUrl {}
impl BuildState for UrlOnly {}
impl BuildState for UrlAndMethod {}
impl BuildState for Ready {}
```

#### Step 3: Builder 核心结构 & 状态转换

```rust
#[derive(Debug, Clone)]
pub struct RequestBuilder<State: BuildState = NoUrl> {
    url: Option<Url>,
    method: Option<String>,
    headers: Vec<(String, String)>,
    body: Option<Vec<u8>>,
    _marker: PhantomData<State>,  // 仅在类型层面存在
}

// ━━━ State 0: NoUrl —— 只能设置 URL ━━━
impl RequestBuilder<NoUrl> {
    pub fn new() -> Self {
        Self {
            url: None,
            method: None,
            headers: Vec::new(),
            body: None,
            _marker: PhantomData,
        }
    }

    /// 消费 self，返回新的状态类型
    /// From<&str> for Url 自动生效（Phase 1 1.5）
    pub fn url(self, url: impl Into<Url>) -> RequestBuilder<UrlOnly> {
        RequestBuilder {
            url: Some(url.into()),
            method: self.method,
            headers: self.headers,
            body: self.body,
            _marker: PhantomData,
        }
    }
}

// ━━━ State 1: UrlOnly —— 只能设置 method ━━━
// 没有 header()、没有 body()、没有 build()——编译期杜绝误用
impl RequestBuilder<UrlOnly> {
    pub fn method(self, method: impl Into<String>) -> RequestBuilder<UrlAndMethod> {
        RequestBuilder {
            url: self.url,
            method: Some(method.into()),
            headers: self.headers,
            body: self.body,
            _marker: PhantomData,
        }
    }
}

// ━━━ State 2: UrlAndMethod —— 可以设置可选字段 ━━━
impl RequestBuilder<UrlAndMethod> {
    pub fn header(mut self, key: impl Into<String>, value: impl Into<String>) -> Self {
        self.headers.push((key.into(), value.into()));
        self
    }

    pub fn body(mut self, data: Vec<u8>) -> Self {
        self.body = Some(data);
        self
    }

    /// 进入 Ready 状态——此后不能再修改
    pub fn build(self) -> RequestBuilder<Ready> {
        RequestBuilder {
            url: self.url,
            method: self.method,
            headers: self.headers,
            body: self.body,
            _marker: PhantomData,
        }
    }
}

// ━━━ State 3: Ready —— 只能发送 ━━━
// 不可变借用：可以多次 send
impl RequestBuilder<Ready> {
    pub fn send(&self) -> String {
        let url = self.url.as_ref().unwrap();
        let method = self.method.as_ref().unwrap();

        let mut request = format!("{} {} HTTP/1.1\r\n", method, url);
        for (key, value) in &self.headers {
            request.push_str(&format!("{}: {}\r\n", key, value));
        }
        request.push_str("\r\n");
        if let Some(body) = &self.body {
            request.push_str(&format!("[{} bytes body]", body.len()));
        }
        request
    }
}
```

#### Step 4: 使用演示

```rust
fn main() {
    // ✓ 正确的使用路径
    let response = RequestBuilder::new()
        .url("https://api.example.com/users")
        .method("POST")
        .header("Content-Type", "application/json")
        .header("Authorization", "Bearer token123")
        .body(br#"{"name": "Alice"}"#.to_vec())
        .build()
        .send();

    println!("{}", response);
    // 输出:
    // POST https://api.example.com/users HTTP/1.1
    // Content-Type: application/json
    // Authorization: Bearer token123
    //
    // [19 bytes body]

    // ✓ Ready 状态可以多次发送（不可变引用）
    let req = RequestBuilder::new()
        .url("https://api.example.com/health")
        .method("GET")
        .build();
    println!("{}", req.send());
    println!("{}", req.send());  // 不会消费 self

    // ✗ 以下代码全部会编译失败（这就是 Typestate 的价值）：
    // RequestBuilder::new().build();
    //   → error[E0599]: no method named `build` found for `RequestBuilder<NoUrl>`
    //
    // RequestBuilder::new().method("GET");
    //   → error[E0599]: no method named `method` found for `RequestBuilder<NoUrl>`
    //
    // RequestBuilder::new().url("...").header("X", "Y");
    //   → error[E0599]: no method named `header` found for `RequestBuilder<UrlOnly>`
}
```

## 3. Demo 2: LendingIterator —— GAT + HRTB 实战

### 3.1 设计哲学

> "标准库的 `Iterator` 有个致命的局限：它要求 `next()` 返回的值必须独立于迭代器自身。这意味着你无法写出'迭代器借出自己数据'的模式。GAT 打破了这堵墙——而 LendingIterator 就是 GAT 存在的**理由**。"

这个模式是 GAT RFC（RFC 1598）的核心动机，也是理解"为什么 Rust 1.65 的 GAT 稳定化如此重要"的最佳切入点。

### 3.2 问题场景

实现一个 `Windows` 迭代器——对切片进行滑动窗口遍历：

```rust
let data = vec![1, 2, 3, 4, 5];
let mut windows = Windows::new(&data, 3);

// 期望：每次 next() 返回 &[i32]，借用自原始 data
// 标准 Iterator 做不到——&[i32] 的生命周期与 &data 绑定
assert_eq!(windows.next(), Some(&[1, 2, 3][..]));
assert_eq!(windows.next(), Some(&[2, 3, 4][..]));
assert_eq!(windows.next(), Some(&[3, 4, 5][..]));
```

**为什么标准 `Iterator` 不够？**

```rust
// 标准 Iterator trait：
pub trait Iterator {
    type Item;                       // ← 固定的关联类型
    fn next(&mut self) -> Option<Self::Item>;
}

// 如果你想表达 "next 返回 &[T]"，Item 的生命周期无法与 &mut self 关联：
impl<'a, T> Iterator for Windows<'a, T> {
    type Item = &'a [T];            // ← 'a 是借入 &data 的生命周期
    fn next(&mut self) -> Option<&'a [T]> {  // ← 但这里 &mut self 只有自己的生命周期
        // 编译器无法证明返回值的生命周期与 self 的关系
        todo!()
    }
}
```

**GAT 的解决**：让关联类型带生命周期参数，关联到 `Self`。

### 3.3 实现

#### Step 1: 定义 LendingIterator trait（GAT 核心）

```rust
/// 借出型迭代器：next 返回的值借用自迭代器自身
///
/// 这是 GAT (1.8) 的经典应用：关联类型 `Item<'a>` 带有生命周期参数，
/// 将返回值的生命周期与 `&mut self` 绑定。
pub trait LendingIterator {
    // ━━━ GAT (Phase 1 1.8): 关联类型带生命周期参数 ━━━
    // Item<'a> 中的 'a 代表 &mut self 的借用生命周期
    type Item<'a>
    where
        Self: 'a;  // ← HRTB 约束 (Phase 1 1.9)

    fn next(&mut self) -> Option<Self::Item<'_>>;
}
```

> **关键语法**：`where Self: 'a` 是 **Higher-Ranked Trait Bound**（Phase 1 1.9）的简化形式。它说："对于**所有**可能借用 self 的生命周期 `'a`，`Self` 都必须活过 `'a`。" 这是一个全称量化——不是某一个特定生命周期，而是**任意**生命周期。

#### Step 2: Windows 迭代器实现

```rust
pub struct Windows<'slice, T> {
    slice: &'slice [T],
    position: usize,
    window_size: usize,
}

impl<'slice, T> Windows<'slice, T> {
    pub fn new(slice: &'slice [T], window_size: usize) -> Self {
        assert!(window_size > 0, "window_size must be positive");
        Self {
            slice,
            position: 0,
            window_size,
        }
    }

    /// 返回窗口数量（编译期计算，const fn）
    pub fn window_count(&self) -> usize {
        if self.slice.len() < self.window_size {
            0
        } else {
            self.slice.len() - self.window_size + 1
        }
    }
}

impl<'slice, T> LendingIterator for Windows<'slice, T> {
    // ━━━ GAT 的 impl 语法 ━━━
    // Item<'a> = &'a [T]，其中 'a 是借用 self 的生命周期
    // 'slice: 'a 约束确保切片活得比借用长
    type Item<'a> = &'a [T]
    where
        'slice: 'a;

    fn next(&mut self) -> Option<Self::Item<'_>> {
        if self.position + self.window_size <= self.slice.len() {
            let window = &self.slice[self.position..self.position + self.window_size];
            self.position += 1;
            Some(window)
        } else {
            None
        }
    }
}
```

#### Step 3: 为什么 LendingIterator 不能是 Trait 对象

```rust
// ━━━ Trait 对象安全（Phase 1 1.2）━━━
// LendingIterator 不是对象安全的，因为 Item<'a> 是 GAT

// ✗ 编译失败：
// fn consume(iter: &mut dyn LendingIterator) { ... }
//   → error[E0038]: the trait `LendingIterator` cannot be made into an object
//     ...because it contains the generic associated type `Item`

// ✓ 只能用静态分发（Phase 1 1.15）：
fn consume(mut iter: impl LendingIterator) {
    while let Some(item) = iter.next() {
        // 处理 item...
    }
}
```

#### Step 4: Impl Trait 辅助方法（Phase 1 1.12）

```rust
impl<'slice, T: std::fmt::Debug> Windows<'slice, T> {
    /// RPIT (Return Position Impl Trait): 隐藏具体返回类型
    pub fn collect_windows<const W: usize>(&self) -> Vec<&[T]> {
        let mut windows = Vec::new();
        let mut pos = 0;
        while pos + W <= self.slice.len() {
            windows.push(&self.slice[pos..pos + W]);
            pos += 1;
        }
        windows
    }
}

/// 为所有 LendingIterator 实现 map 适配器（展示 blanket impl + GAT）
pub struct Map<L, F> {
    inner: L,
    f: F,
}

// 注意：这里 Item<'a> 的类型依赖于 F 的输出类型
impl<L: LendingIterator, F, B> LendingIterator for Map<L, F>
where
    F: for<'a> FnMut(L::Item<'a>) -> B,
    //  ↑ HRTB (Phase 1 1.9): F 必须对所有生命周期 'a 都有效
{
    type Item<'a> = B
    where
        L: 'a,
        F: 'a;

    fn next(&mut self) -> Option<Self::Item<'_>> {
        self.inner.next().map(&mut self.f)
    }
}
```

#### Step 5: FQS 实战——消除方法歧义（Phase 1 1.17）

```rust
// 当多个 trait 提供同名方法时，FQS 是唯一的解决方案

trait StringExt {
    fn length(&self) -> usize;
}

impl StringExt for &str {
    fn length(&self) -> usize {
        self.chars().count()  // 字符数，而不是字节数
    }
}

fn demonstrate_fqs(s: &str) {
    // 标准库的 len() 返回字节数
    let byte_len = s.len();                // 5 (for "hello")

    // StringExt 的 length() 返回字符数——同名冲突！
    // ✓ FQS 消除歧义（Phase 1 1.17）：
    let char_count = <&str as StringExt>::length(&s);  // 5 (for "hello")
    //                               ↑ FQS 语法：<Type as Trait>::method()

    // 或使用 UFCS 风格：
    let char_count2 = StringExt::length(&s);

    println!("bytes: {}, chars: {}", byte_len, char_count);
}
```

## 4. Demo 3: StackVec —— Const Generics + DST + 编译期安全

### 4.1 设计哲学

> "动态分配不是免费的。当你确定一个向量的元素数不会超过 64 时，把这个限制写进类型里——让编译器在**栈上**分配空间，用 const generics 在**编译期**验证容量。"

`smallvec`、`arrayvec`、`tinyvec` 等 crate 都基于这个理念。Servo 浏览器引擎大量使用这种模式来避免堆分配。

### 4.2 问题场景

实现 `StackVec<T, const N: usize>`：
- 元素存储在栈上的固定大小数组中（编译期确定容量）
- 提供类 `Vec` 的 API：`push`, `pop`, `as_slice`, `len`
- 容量在**编译期**被保证：`push` 超出 N 是编译错误（或编译期可见的 panic）
- 实现标准 trait，展示 negative impls 和 never type

### 4.3 实现

#### Step 1: 核心结构（Const Generics + Sized/DST）

```rust
use std::mem::{self, MaybeUninit};

/// 编译期定长向量
///
/// const N: usize 是 const generic（Phase 1 1.10）—— 容量在编译期确定。
/// [MaybeUninit<T>; N] 的大小 = size_of::<T>() * N，编译期就知道。
pub struct StackVec<T, const N: usize> {
    data: [MaybeUninit<T>; N],
    len: usize,  // 已初始化的元素数量
}

impl<T, const N: usize> StackVec<T, N> {
    /// const fn 构造（常量上下文可用）
    pub const fn new() -> Self {
        StackVec {
            // SAFETY: MaybeUninit::uninit() 在 const context 中合法
            // 因为我们尚未读取任何元素
            data: unsafe { MaybeUninit::uninit().assume_init() },
            len: 0,
        }
    }

    pub fn push(&mut self, value: T) {
        if self.len >= N {
            panic!(
                "StackVec capacity exceeded: tried to push when len={} but capacity={}",
                self.len, N
            );
        }
        self.data[self.len].write(value);
        self.len += 1;
    }

    pub fn pop(&mut self) -> Option<T> {
        if self.len == 0 {
            None
        } else {
            self.len -= 1;
            // SAFETY: len > 0 implies data[self.len] was initialized
            Some(unsafe { self.data[self.len].assume_init_read() })
        }
    }

    pub fn as_slice(&self) -> &[T] {
        // SAFETY: data[0..len] are initialized T values
        unsafe { std::slice::from_raw_parts(self.data.as_ptr() as *const T, self.len) }
    }

    pub fn as_mut_slice(&mut self) -> &mut [T] {
        unsafe { std::slice::from_raw_parts_mut(self.data.as_mut_ptr() as *mut T, self.len) }
    }

    pub fn is_empty(&self) -> bool {
        self.len == 0
    }

    /// 编译期可用的容量查询
    pub const fn capacity(&self) -> usize {
        N
    }

    pub const fn len(&self) -> usize {
        self.len
    }
}
```

#### Step 2: Drop 实现 & Sized/DST 深度分析

```rust
impl<T, const N: usize> Drop for StackVec<T, N> {
    fn drop(&mut self) {
        // 只 drop 已初始化的元素
        for i in 0..self.len {
            unsafe { self.data[i].assume_init_drop() };
        }
        // data 数组本身是栈分配，会自动回收
    }
}

// ━━━ Sized/DST 深度分析（Phase 1 1.1）━━━
// StackVec<T, 4>    → Sized（栈上，大小 = MaybeUninit<T> * 4 + usize）
// StackVec<[u8], 4> → 不合法！[u8] 是 DST，不能放在 MaybeUninit 中
// 编译器报错：the size for values of type `[u8]` cannot be known
//
// 但我们可以将 StackVec 转为 DST 引用：
// &StackVec<T, N>  → &[T]（通过 Deref 或 as_slice）——这就是胖指针
```

// ━━━ Never Type 的应用（Phase 1 1.13）━━━
//
// `!` 类型的核心特性：可以自动强制转换为**任何**类型。
// 这意味着它可以用在 "不可能发生" 的分支中。

use std::convert::Infallible;

/// TryFrom 用于栈数组转 StackVec——容量在编译期匹配时永远成功
impl<T, const N: usize> TryFrom<[T; N]> for StackVec<T, N> {
    // 错误类型是 Infallible（! 的 stable 别名）
    // 因为当数组大小 == N 时，push 永远不会失败
    type Error = Infallible;

    fn try_from(arr: [T; N]) -> Result<Self, Self::Error> {
        let mut vec = StackVec::new();
        // 编译器知道这个循环永远不会溢出
        for elem in arr {
            vec.push(elem);
        }
        Ok(vec)
    }
}

// 可以直接 unwrap——因为 Error 是 Infallible（不可能构造出来）
// let sv: StackVec<i32, 3> = [1, 2, 3].try_into().unwrap();  // 永远不会 panic

// ━━━ Negative Impls & Auto Trait 控制（Phase 1 1.14）━━━
//
// 使用 PhantomData<*const ()> 来阻止 Send/Sync 的自动实现。
// 这是标准库 UnsafeCell 的同款技巧。

use std::marker::PhantomData;
use std::cell::Cell;

/// 一个显式标记为 !Sync 的包装器
///
/// *const T 不是 Send/Sync 的，所以包含它的类型也不会是。
/// 这在标准库中是 stable 的做法（不需要 nightly 的 negative_impls feature）。
pub struct NotSyncWrapper<T> {
    inner: T,
    _not_sync: PhantomData<Cell<()>>,  // Cell<!> is !Sync
}

// 这类模式用于：
// - 构建自定义内部可变性容器
// - 显式标记某类型不能跨线程共享
// - FFI 中包装 C 库的线程不安全对象

## 5. Demo 4: Extension Trait 模式 —— 为外部类型赋能

### 5.1 设计哲学

> "孤儿规则（Orphan Rule）是 Rust coherence 系统的基石：你既不能为别人的 trait 实现别人的类型，也不能为你的 trait 实现别人的类型（除非类型或 trait 中有至少一个在你的 crate 中定义）。Extension Trait 是这个规则的**标准解法**——它在不破坏 coherence 的前提下，让你为任何类型添加方法。"

这个模式被 itertools、anyhow、tokio、futures 等几乎所有主流 crate 使用。理解 Extension Trait = 理解 Rust 生态的 API 设计惯例。

### 5.2 问题场景

为 `Vec<T>` 添加两个方法：
- `median()`：返回中位数
- `to_histogram()`：返回频率分布

这些都是外部类型（`Vec` 是 std 的）——孤儿规则阻止直接 `impl`。我们用三种策略解决，并对比优劣。

### 5.3 实现

#### Strategy 1: Extension Trait（最推荐）

```rust
use std::collections::HashMap;
use std::hash::Hash;

/// Extension Trait: 在**你的** crate 中定义 trait，
/// 然后为外部类型实现它——孤儿规则允许这么做。
///
/// 关键：impl<T> VecExt<T> for Vec<T> —— Vec 是外部类型，
/// 但 VecExt 在你的 crate 中定义，满足"至少一个在当前 crate"。
pub trait VecExt<T> {
    /// 返回中位数（对切片排序后取中间值）
    /// where 子句展示了 Supertrait-style bounds（Phase 1 1.16）：
    /// T 必须实现 Copy + PartialOrd
    fn median(&mut self) -> Option<T>
    where
        T: Copy + PartialOrd;

    /// 返回频率分布直方图
    fn to_histogram(&self) -> Vec<(T, usize)>
    where
        T: Eq + Hash + Clone;
}

impl<T> VecExt<T> for Vec<T> {
    fn median(&mut self) -> Option<T>
    where
        T: Copy + PartialOrd,
    {
        if self.is_empty() {
            return None;
        }
        // 使用 PartialOrd 而非 Ord——兼容 f32/f64
        self.sort_by(|a, b| a.partial_cmp(b).unwrap_or(std::cmp::Ordering::Equal));
        Some(self[self.len() / 2])
    }

    fn to_histogram(&self) -> Vec<(T, usize)>
    where
        T: Eq + Hash + Clone,
    {
        let mut counts: HashMap<T, usize> = HashMap::new();
        for item in self {
            *counts.entry(item.clone()).or_insert(0) += 1;
        }
        let mut result: Vec<_> = counts.into_iter().collect();
        result.sort_by(|a, b| b.1.cmp(&a.1));  // 按频率降序
        result
    }
}

// ━━━ 使用 ━━━
fn demo_extension_trait() {
    let mut numbers = vec![3.0, 1.0, 4.0, 1.0, 5.0, 9.0, 2.0];

    // use VecExt 来引入方法——标准做法
    // 如果不 use，方法不可见（不会污染命名空间）
    use VecExt;
    println!("median: {:?}", numbers.median());  // Some(3.0)

    let hist = numbers.to_histogram();
    println!("histogram: {:?}", hist);
    // [(1.0, 2), (2.0, 1), (3.0, 1), (4.0, 1), (5.0, 1), (9.0, 1)]
}
```

#### Strategy 2: Newtype Wrapper（当 Extension Trait 不够时）

```rust
/// 当你的 trait 方法与标准库方法**同名冲突**时，
/// Newtype 是唯一的解决方案（extension trait 会引发歧义）。
///
/// 示例：为 Vec 添加一个自定义的 `len()` 方法
/// 返回的是元素值之和而非元素个数——显然不能用 extension trait。

#[derive(Debug, Clone)]
pub struct SummableVec<T>(pub Vec<T>);  // Newtype (Phase 1 1.3)

// ━━━ From/Into 转换（Phase 1 1.5）━━━
impl<T> From<Vec<T>> for SummableVec<T> {
    fn from(v: Vec<T>) -> Self {
        SummableVec(v)
    }
}

// ━━━ AsRef/Deref 选择（Phase 1 1.3, 1.5）━━━
// AsRef: 轻量级引用转换，不引入方法名污染
impl<T> AsRef<Vec<T>> for SummableVec<T> {
    fn as_ref(&self) -> &Vec<T> {
        &self.0
    }
}

// 注意：这里**不**实现 Deref
// 原因：Deref 会引入方法解析混淆，且 SummableVec 不是 "is-a" Vec
// （这是一种有意的语义选择）

impl<T: std::ops::Add<Output = T> + Copy + Default> SummableVec<T> {
    /// 自定义的 len：返回元素值之和
    /// 这个方法名是 `len`，但语义完全不同——Newtype 让它合法存在
    pub fn len(&self) -> T {
        let mut sum = T::default();
        for item in &self.0 {
            sum = sum + *item;
        }
        sum
    }
}

// ━━━ FQS 演示（Phase 1 1.17）━━━
// 想要同时使用标准库 len() 和自己的 len()？用 FQS：
fn fqs_demo() {
    let sv = SummableVec(vec![1, 2, 3, 4, 5]);

    // 标准库的 len()（通过 AsRef）
    let std_len = <SummableVec<i32> as AsRef<Vec<i32>>>::as_ref(&sv).len();
    // 或更简单：
    let std_len2 = AsRef::<Vec<i32>>::as_ref(&sv).len();

    // 自己的 len()
    let custom_len = sv.len();

    println!("std len: {}, custom sum: {}", std_len2, custom_len);
    // std len: 5, custom sum: 15
}
```

#### Strategy 3: Blanket Impl（为满足条件的**所有**类型实现）

```rust
/// Blanket Impl: 为所有满足 trait bound 的 T 实现你的 trait
/// 这是 Supertrait 系统（Phase 1 1.16）的延伸

use std::fmt::Display;

/// 为所有实现 Display 的类型添加 `summarize_with_prefix` 方法
pub trait SummaryExt {
    fn summarize_with_prefix(&self, prefix: &str) -> String;
    fn summarize(&self) -> String {
        self.summarize_with_prefix("")
    }
}

// ━━━ Blanket impl（孤儿规则允许！）━━━
// T: Display 中的 T 包含**所有**实现了 Display 的类型——
// 它们都"在你的控制之下了"
impl<T: Display> SummaryExt for T {
    fn summarize_with_prefix(&self, prefix: &str) -> String {
        format!("{}: {}", prefix, self)
    }
}

// ━━━ 使用：String, i32, f64 等所有 Display 类型都能用 ━━━
fn demo_blanket() {
    // i32 实现了 Display → 自动获得 SummaryExt 的方法
    println!("{}", 42.summarize());                    // ": 42"
    println!("{}", 42.summarize_with_prefix("答案"));   // "答案: 42"

    // String 实现了 Display → 也可以用
    println!("{}", String::from("hello").summarize_with_prefix("msg"));
    // "msg: hello"
}
```

    pub const fn new() -> Self {
        KVStore {
            entries: unsafe { MaybeUninit::uninit().assume_init() },
            len: 0,
            _mode: PhantomData,
        }
    }

    pub const fn len(&self) -> usize { self.len }
    pub const fn is_empty(&self) -> bool { self.len == 0 }
    pub const fn capacity(&self) -> usize { N }

    pub fn get(&self, key: &Key) -> Option<&V>
    where
        K: PartialEq<Key>,
    {
        for i in 0..self.len {
            let entry = unsafe { &*self.entries[i].as_ptr() };
            if entry.0 == *key {
                return Some(&entry.1);
            }
        }
        None
    }

    pub fn contains_key(&self, key: &Key) -> bool
    where
        K: PartialEq<Key>,
    {
        self.get(key).is_some()
    }
}

// ━━━ ReadWrite-only methods ━━━
impl<K, V, const N: usize> KVStore<K, V, N, ReadWrite> {
    pub fn insert(&mut self, key: K, value: V)
    where
        K: PartialEq<Key>,
    {
        // 去重：如果已存在则替换
        for i in 0..self.len {
            let entry = unsafe { &mut *self.entries[i].as_mut_ptr() };
            if entry.0 == key {
                entry.1 = value;
                return;
            }
        }
        if self.len >= N {
            panic!("KVStore capacity exceeded: {}", N);
        }
        self.entries[self.len].write((key, value));
        self.len += 1;
    }

    pub fn remove(&mut self, key: &Key) -> Option<V>
    where
        K: PartialEq<Key>,
    {
        for i in 0..self.len {
            let entry = unsafe { &*self.entries[i].as_ptr() };
            if entry.0 == *key {
                // swap-remove
                let removed = unsafe { self.entries[i].assume_init_read() };
                self.entries[i] = unsafe { self.entries[self.len - 1].assume_init_read() };
                self.len -= 1;
                return Some(removed.1);
            }
        }
        None
    }

    /// Typestate 转换：ReadWrite → ReadOnly
    pub fn into_readonly(self) -> KVStore<K, V, N, ReadOnly> {
        KVStore {
            entries: unsafe {
                // SAFETY: 我们转移了所有权，ReadOnly 模式不会调用 drop
                // 实际上需要 ManuallyDrop 或 ptr::read——这里简化了
                std::ptr::read(&self.entries)
            },
            len: self.len,
            _mode: PhantomData,
        }
    }
}

// ━━━ ReadOnly-specific ━━━
impl<K, V, const N: usize> KVStore<K, V, N, ReadOnly> {
    /// Typestate 转换：ReadOnly → ReadWrite（需要所有权）
    pub fn into_readwrite(self) -> KVStore<K, V, N, ReadWrite> {
        KVStore {
            entries: unsafe { std::ptr::read(&self.entries) },
            len: self.len,
            _mode: PhantomData,
        }
    }
}

// ━━━ Drop ━━━
impl<K, V, const N: usize, Mode: StoreMode> Drop for KVStore<K, V, N, Mode> {
    fn drop(&mut self) {
        for i in 0..self.len {
            unsafe { self.entries[i].assume_init_drop() };
        }
    }
}

// ━━━ GAT LendingIterator ━━━
pub trait StoreIterator {
    type Item<'a>
    where
        Self: 'a;
    fn next(&mut self) -> Option<Self::Item<'_>>;
}

pub struct KVStoreIter<'a, K, V, const N: usize, Mode: StoreMode> {
    store: &'a KVStore<K, V, N, Mode>,
    position: usize,
}

impl<K, V, const N: usize, Mode: StoreMode> KVStore<K, V, N, Mode> {
    pub fn iter(&self) -> KVStoreIter<'_, K, V, N, Mode> {
        KVStoreIter { store: self, position: 0 }
    }
}

impl<'a, K, V, const N: usize, Mode: StoreMode> StoreIterator
    for KVStoreIter<'a, K, V, N, Mode>
{
    type Item<'b> = (&'b K, &'b V) where 'a: 'b;

    fn next(&mut self) -> Option<Self::Item<'_>> {
        if self.position < self.store.len {
            let entry = unsafe { &*self.store.entries[self.position].as_ptr() };
            self.position += 1;
            Some((&entry.0, &entry.1))
        } else {
            None
        }
    }
}

// ━━━ From conversion ━━━
impl<K, V, const N: usize> From<[(K, V); N]> for KVStore<K, V, N, ReadWrite> {
    fn from(arr: [(K, V); N]) -> Self {
        let mut store = KVStore::new();
        for (k, v) in arr {
            unsafe { store.entries[store.len].write((k, v)) };
            store.len += 1;
        }
        store
    }
}

// ━━━ Extension trait ━━━
pub trait IntoValueIter<V> {
    fn into_values(self) -> Vec<V>;
}

impl<K, V: Clone, const N: usize, Mode: StoreMode> IntoValueIter<V>
    for KVStore<K, V, N, Mode>
{
    fn into_values(self) -> Vec<V> {
        let mut values = Vec::with_capacity(self.len);
        for i in 0..self.len {
            let entry = unsafe { &*self.entries[i].as_ptr() };
            values.push(entry.1.clone());
        }
        values
    }
}

// ━━━ Comparison traits ━━━
impl<K: Eq, V: PartialEq, const N: usize, M1: StoreMode, M2: StoreMode>
    PartialEq<KVStore<K, V, N, M2>> for KVStore<K, V, N, M1>
{
    fn eq(&self, other: &KVStore<K, V, N, M2>) -> bool {
        if self.len != other.len {
            return false;
        }
        for i in 0..self.len {
            let a = unsafe { &*self.entries[i].as_ptr() };
            let b = unsafe { &*other.entries[i].as_ptr() };
            if a.0 != b.0 || a.1 != b.1 {
                return false;
            }
        }
        true
    }
}

impl<K: Eq, V: Eq, const N: usize, Mode: StoreMode> Eq for KVStore<K, V, N, Mode> {}
