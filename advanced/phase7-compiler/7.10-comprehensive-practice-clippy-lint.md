# 7.10 Phase 7 综合实战：为 Clippy 贡献一个 Lint

> Phase 7 的 9 节课带你走完了从 Token 到机器码的全流程——但现在这些都是"纸上谈兵"。真正理解编译器内部的最佳方式是把理论变成一行行会编译的代码。为 Clippy 写一个自定义 lint 是 Rust 编译器领域最友好的入门实战：它足够独立（不依赖整个编译管线）、有清晰的模板（参照已有 lint）、产出可验证（UI 测试通过或失败）、而且——如果你愿意——可以提交 PR 成为真正的 Rust 贡献者。本次综合实战将带你走完一整个"从 clone 源码到 lint 生效"的完整流程。

### 2.2 Stage0/1/2 Bootstrap

```
rustc 的构建使用"自举"（bootstrap）策略：

Stage0: 用你系统上已有的 rustc（beta 版本）
        ↓ 编译
Stage1: 一个用 Stage0 编译的新 rustc（功能完整但未优化）
        ↓ 用 Stage1 编译
Stage2: 用 Stage1 编译的最终 rustc（经过完整优化）
        ↓ 用 Stage2 编译

# 实践中你只需要：

## 3. 选题：找一个合适的 Lint

### 3.1 推荐选题（从易到难）

```rust
// ━━━ Level 1: 入门（模式匹配 + 简单建议）━━━
// lint 名: unnecessary_lazy_evaluations
// 目标: 检测不必要的闭包包装
// 示例:
//   let x = || 42;        → 警告：不需要闭包
//   let x = || { 42 };    → 警告：不需要闭包

// ━━━ Level 2: 进阶（类型检查 + 方法调用分析）━━━
// lint 名: unnecessary_to_owned
// 目标: 检测对已有引用的不必要 .to_owned() / .to_string()
// 示例:
//   let s: &str = "hello";
//   let owned = s.to_string();  // 如果只需要 &str → 不必要

// ━━━ Level 3: 挑战（控制流分析 + 复杂模式）━━━
// lint 名: redundant_else
// 目标: 检测 if 块最后是 return/break/continue 时不必要的 else
// 示例:
//   if cond { return x; } else { y }  → 不需要 else
```

### 3.2 从哪里找选题

```
1. Clippy issue tracker:
   https://github.com/rust-lang/rust-clippy/issues
   搜索标签: "good first issue", "E-easy", "E-medium"

2. Clippy 的 "lint request" 标签:
   社区提出的 lint 建议，等待实现

3. 观察自己代码中的 clippy 警告:
   哪些模式 clippy 还没有检测到？
```

> **本节实战推荐**：选择 Level 1 的 `unnecessary_lazy_evaluations`。它足够简单（约 50 行 lint 代码），但完整覆盖了 Clippy lint 开发的全部流程——从注册到测试。

## 4. 理解 Clippy 的 Lint 基础设施

### 4.1 两种 Lint Pass

```rust
// ━━━ EarlyLintPass: 在 AST 阶段运行 ━━━
// 适用场景: 不需要类型信息的检查（命名、格式、简单语法）
// 优点: 在宏展开之前运行，处理源代码级模式
// 缺点: 没有类型信息，不能做深层分析

// ━━━ LateLintPass: 在 HIR 阶段运行 ━━━
// 适用场景: 需要类型信息、方法解析、trait 判断的检查
// 优点: 可以访问完整的 HIR + 类型上下文
// 缺点: 在宏展开之后，处理的是展开后的代码
// → 大多数 lint 用这个

use rustc_lint::{LateContext, LateLintPass};

struct MyLint;
impl LateLintPass<'_> for MyLint {
    fn check_expr(&mut self, cx: &LateContext<'_>, expr: &Expr<'_>) {
        // 遍历每个表达式时调用
    }
}
```

### 4.2 Lint 声明与注册

```rust
// ━━━ 1. 声明 lint ━━━
// 在 clippy_lints/src/lib.rs 或对应模块中：
declare_clippy_lint! {
    /// ### What it does
    /// 检测不必要的闭包包装。
    ///
    /// ### Why is this bad?
    /// `|| 42` 和 `42` 是等价的，闭包增加了一层不必要的间接。
    ///
    /// ### Example
    /// ```rust
    /// // bad
    /// let x = || 42;
    ///
    /// // good
    /// let x = 42;
    /// ```
    #[clippy::version = "1.80.0"]
    pub UNNECESSARY_LAZY_EVALUATIONS,
    complexity,                    // ← lint 分类（complexity, style, perf, correctness, ...）
    "detects unnecessary closure wrappers"  // ← 简短描述
}

// ━━━ 2. 注册 lint ━━━
// 在 clippy_lints/src/lib.rs 的 register_lints 函数中：
store.register_late_pass(|_| Box::new(MyLint));

// 或者在模块的 impl_lint_pass 宏中：
impl_lint_pass!(MyLint => [UNNECESSARY_LAZY_EVALUATIONS]);
```

### 4.3 诊断 API

```rust
use rustc_lint::{LateContext, LintContext};
use rustc_span::sym;

impl LateLintPass<'_> for MyLint {
    fn check_expr(&mut self, cx: &LateContext<'_>, expr: &Expr<'_>) {
        // ━━━ 发出警告（无建议）━━━
        cx.span_lint(
            UNNECESSARY_LAZY_EVALUATIONS,
            expr.span,
            "unnecessary closure wrapper",
        );

        // ━━━ 发出警告 + 帮助信息 ━━━
        cx.span_lint_and_help(
            UNNECESSARY_LAZY_EVALUATIONS,
            expr.span,
            "unnecessary closure can be removed",
            None,
            "replace the closure with its body directly",
        );

        // ━━━ 发出警告 + 自动修复建议 ━━━
        // Applicability::MachineApplicable → 可以安全自动应用
        // Applicability::MaybeIncorrect → 需要人工确认
        // Applicability::Unspecified → 仅提示，不自动修复
        cx.span_lint_and_sugg(
            UNNECESSARY_LAZY_EVALUATIONS,
            expr.span,
            "unnecessary closure",
            "try",
            format!("{}", snippet(cx, body_expr.span, "..")),
            Applicability::MachineApplicable,
        );
    }
}
```

## 5. 实现 Lint 逻辑

### 5.1 完整示例：`unnecessary_lazy_evaluations`

```rust
// 文件: clippy_lints/src/unnecessary_lazy_eval.rs

use clippy_utils::diagnostics::span_lint_and_sugg;
use clippy_utils::source::snippet;
use clippy_utils::ty::is_type_diagnostic_item;
use rustc_errors::Applicability;
use rustc_hir::{Closure, Expr, ExprKind, Mutability};
use rustc_lint::{LateContext, LateLintPass};
use rustc_middle::ty;
use rustc_session::declare_lint_pass;
use rustc_span::sym;

declare_clippy_lint! {
    /// ### What it does
    /// 检测立即被调用的闭包（IIFE）可以简化为直接求值。
    ///
    /// ### Why is this bad?
    /// `(|| 42)()` 等价于 `42`，但多了一层无意义的闭包。
    ///
    /// ### Example
    /// ```rust
    /// // bad
    /// let x = (|| 42)();
    ///
    /// // good
    /// let x = 42;
    /// ```
    #[clippy::version = "1.80.0"]
    pub UNNECESSARY_LAZY_EVALUATIONS,
    complexity,
    "detects closures that are immediately evaluated"
}

declare_lint_pass!(UnnecessaryLazyEval => [UNNECESSARY_LAZY_EVALUATIONS]);

impl LateLintPass<'_> for UnnecessaryLazyEval {
    fn check_expr(&mut self, cx: &LateContext<'_>, expr: &Expr<'_>) {
        // ━━━ 模式匹配：找"立即调用的闭包" (|| body)() ━━━
        // expr 是一个函数调用 expr(args)
        if let ExprKind::Call(callee, args) = &expr.kind
            // callee 是一个闭包表达式 || body
            && let ExprKind::Closure(closure) = &callee.kind
            // 调用时没有传参数
            && args.is_empty()
        {
            // 检查闭包没有捕获任何变量（纯计算）
            // 如果有捕获 → 不是"无意义"的闭包，跳过
            let body = cx.tcx.hir().body(closure.body);
            if !body.params.is_empty() {
                return; // 有参数 → 不是我们要检测的模式
            }

            // 获取闭包体的源码片段（用于建议）
            let body_snippet = snippet(cx, body.value.span, "..");

            span_lint_and_sugg(
                cx,
                UNNECESSARY_LAZY_EVALUATIONS,
                expr.span,
                "this closure is immediately evaluated and can be simplified",
                "try removing the closure wrapper",
                body_snippet.to_string(),
                Applicability::MachineApplicable,
            );
        }
    }
}

// ━━━ 关键点 ━━━
// 1. ExprKind::Call → 匹配函数调用语法（f(args)）
// 2. ExprKind::Closure → 匹配闭包表达式（|| body）
// 3. 检查 args.is_empty() → 确保是 () 调用（没有参数）
// 4. snippet(cx, span, default) → 获取源码片段
// 5. Applicability::MachineApplicable → 标记为可自动修复
```

### 5.2 HIR 节点速查表

```rust
// 写 lint 时最常用的 HIR 节点（来自 7.1 的 HIR 知识）：

// ━━━ 表达式 (ExprKind) ━━━
ExprKind::Call(callee, args)     // f(x, y)
ExprKind::MethodCall(seg, rcvr, args, span)  // x.f(y)
ExprKind::Closure(closure)       // |x| x + 1
ExprKind::If(cond, then, else)   // if cond { then } else { else }
ExprKind::Match(scrutinee, arms, source)  // match x { ... }
ExprKind::Loop(body, label, ..)  // loop { body }
ExprKind::Block(block, label)    // { ... }
ExprKind::Lit(lit)               // 字面量: 42, "hello"
ExprKind::Binary(op, lhs, rhs)   // a + b, a == b
ExprKind::Unary(op, arg)         // -x, !x, *x

// ━━━ 语句 (StmtKind) ━━━
StmtKind::Let(local)             // let x = ...;
StmtKind::Expr(expr)             // expr;
StmtKind::Semi(expr)             // expr; (有分号的表达式语句)

// ━━━ 模式 (PatKind) ━━━
PatKind::Binding(mode, .., ident, sub)  // x, mut x, ref x
PatKind::Struct(qpath, fields, ..)      // Foo { x, y }
PatKind::TupleStruct(qpath, subs, ..)   // Foo(x, y)
PatKind::Wild                           // _
```

└── unnecessary_lazy_eval.stderr    # 期望的诊断输出

### 6.2 测试输入文件 (`.rs`)

```rust
// 文件: tests/ui/unnecessary_lazy_eval.rs

#![warn(clippy::unnecessary_lazy_evaluations)]

fn main() {
    // 应该触发 lint
    let _x = (|| 42)();          //~ ERROR: this closure is immediately evaluated

    // 应该触发 lint（多行）
    let _y = (|| {
        1 + 1                    //~ ERROR: this closure is immediately evaluated
    })();

    // 不应该触发（有参数）
    let _z = (|x: i32| x + 1)(5);

    // 不应该触发（有捕获）
    let msg = "hello";
    let _capture = (|| msg.len())();

    // 不应该触发（不是立即调用）
    let _f = || 42;
}
```

## 7. 进阶：提交 Pull Request

### 7.1 Clippy 贡献流程

```
1. 在 rust-lang/rust 仓库创建分支
   git checkout -b clippy-unnecessary-lazy-eval

2. 实现 lint（按照上面的模板）
   - clippy_lints/src/unnecessary_lazy_eval.rs
   - 在 clippy_lints/src/lib.rs 中注册模块
   - 在 clippy_lints/src/declared_lints.rs 中注册 lint 名

3. 运行测试
   ./x.py test clippy -- unnecessary_lazy_eval

4. 添加 changelog 条目
   clippy_changelog/clippy changelog

5. 提交
   git commit -m "Add unnecessary_lazy_evaluations lint"

6. 推送并创建 PR
   git push origin clippy-unnecessary-lazy-eval
   # 访问 GitHub 创建 PR
```

### 7.2 PR 描述模板

```markdown
Add new lint: `unnecessary_lazy_evaluations`

## What
Detects closures that are immediately evaluated, e.g. `(|| 42)()` -> `42`

## Why
This pattern adds unnecessary indirection and makes code harder to read.

## Category
complexity

changelog: [`unnecessary_lazy_evaluations`]: new lint to detect
immediately evaluated closures that can be simplified
```

### 7.3 常见 Review 反馈

```
1. "这个 lint 的误报率太高了" → 收紧触发条件
2. "应该用 Applicability::MachineApplicable" → 检查建议的安全性
3. "需要更多测试用例" → 补充边界条件测试
4. "和其他 lint 重叠了" → 检查是否已有类似的 lint
5. "Changelog 格式不对" → 按模板更新
```
