## 2. Demo 1: RawVec 从头写 + Custom Allocator — Unsafe Rust 全家桶

### 2.1 设计哲学

> "`Vec<T>` 的内部实现只有 ~100 行 unsafe 代码。不是因为魔法——而是因为 Rust 的 unsafe 原语恰好足够。本 Demo 带你完整手写 RawVec：Layout 管理、Grow 策略、drop 顺序、自定义分配器。写完你就不会再害怕 unsafe。"

### 2.2 RawVec —— 标准库 Vec 的内核

```rust
use std::alloc::{self, Layout};
use std::ptr::NonNull;
use std::mem::{self, ManuallyDrop};

/// ━━━ 4.2 裸指针: RawVec 是裸指针管理的最佳教材 ━━━
pub struct RawVec<T> {
    ptr: NonNull<T>,   // ━━━ NonNull: niche 优化的裸指针 ━━━
    cap: usize,        // 容量（分配的 T 数量）
}

impl<T> RawVec<T> {
    pub fn new() -> Self {
        // ━━━ 4.2 裸指针: ZST 不需要实际分配 ━━━
        // 对 ZST，用 dangling 指针——size=0 所以永远不解引用
        assert!(mem::size_of::<T>() != 0, "ZST 需要特殊处理");
        RawVec {
            ptr: NonNull::dangling(),
            cap: if mem::size_of::<T>() == 0 { usize::MAX } else { 0 },
        }
    }

    /// ━━━ 4.5 Intrinsics + 4.10 Allocator ━━━
    /// grow 策略：类似标准库 Vec（加倍直到足够大）
    pub fn grow(&mut self, old_len: usize, new_len: usize) {
        // 计算新的容量
        let new_cap = if self.cap == 0 {
            new_len.max(4)  // 初始分配 4 个
        } else {
            // 加倍直到 >= new_len
            let mut cap = self.cap;
            while cap < new_len {
                cap *= 2;
            }
            cap
        };

        // ━━━ 4.2 裸指针: Layout 精确描述内存需求 ━━━
        let new_layout = Layout::array::<T>(new_cap).unwrap();
        let old_layout = Layout::array::<T>(self.cap).unwrap();

        let new_ptr = if self.cap == 0 {
            // 首次分配
            unsafe { alloc::alloc(new_layout) }
        } else {
            // ━━━ 4.1 Unsafe: alloc::realloc 的 # Safety 契约 ━━━
            // Safety:
            // - self.ptr 之前由 alloc::alloc 分配
            // - old_layout 与分配时一致
            // - new_size > 0
            unsafe { alloc::realloc(self.ptr.as_ptr() as *mut u8, old_layout, new_layout.size()) }
        };

        // ━━━ 4.6 hint: 分支预测提示 ━━━
        if new_ptr.is_null() {
            // ━━━ 4.5 intrinsics: cold path 标记 ━━━
            alloc::handle_alloc_error(new_layout);
            // handle_alloc_error 内部使用 std::intrinsics::unlikely
        }

        self.ptr = unsafe { NonNull::new_unchecked(new_ptr as *mut T) };
        self.cap = new_cap;
    }

    /// ━━━ 4.5 Intrinsics: copy_nonoverlapping ━━━
    /// 在 grow 后需要把旧元素移到新分配的空间
    pub fn relocate(&self, old_len: usize) {
        if self.cap > 0 && old_len > 0 {
            // copy_nonoverlapping ≈ memcpy（src 和 dst 不重叠）
            // 比 ptr::copy 更快——编译器可以生成 SIMD 指令（4.11）
        }
    }
}

impl<T> Drop for RawVec<T> {
    fn drop(&mut self) {
        if self.cap > 0 && mem::size_of::<T>() != 0 {
            let layout = Layout::array::<T>(self.cap).unwrap();
            unsafe { alloc::dealloc(self.ptr.as_ptr() as *mut u8, layout) };
        }
    }
}

### 2.3 自定义 GlobalAlloc（Bump + 计数）

```rust
use std::alloc::{GlobalAlloc, Layout};
use std::sync::atomic::{AtomicUsize, Ordering};

/// ━━━ 4.10 Custom Allocator: GlobalAlloc trait ━━━
///
/// 这个分配器的策略：
/// 1. 用 mmap 分配一大块内存（模拟 bump allocator）
/// 2. 每次 alloc 递增偏移量——超快，但从不释放单个分配
/// 3. 用 AtomicUsize 追踪分配次数——零锁开销

struct CountingBumpAllocator {
    // 简化版：直接委托给系统分配器，但追踪计数
    allocated: AtomicUsize,
    freed: AtomicUsize,
}

impl CountingBumpAllocator {
    const fn new() -> Self {
        CountingBumpAllocator {
            allocated: AtomicUsize::new(0),
            freed: AtomicUsize::new(0),
        }
    }

    fn stats(&self) -> (usize, usize) {
        (
            self.allocated.load(Ordering::Relaxed),
            self.freed.load(Ordering::Relaxed),
        )
    }
}

unsafe impl GlobalAlloc for CountingBumpAllocator {
    unsafe fn alloc(&self, layout: Layout) -> *mut u8 {
        self.allocated.fetch_add(layout.size(), Ordering::Relaxed);
        // 委托给系统分配器
        alloc::alloc(layout)
    }

    unsafe fn dealloc(&self, ptr: *mut u8, layout: Layout) {
        self.freed.fetch_add(layout.size(), Ordering::Relaxed);
        alloc::dealloc(ptr, layout)
    }
}

// ━━━ 4.12 Sanitizers: 测试这个分配器 ━━━
// cargo +nightly miri test 可以检测：
// - use-after-free
// - double-free
// - 内存泄漏
// - 越界访问

/// ━━━ 4.3 transmute 的安全替代方案 ━━━
///
/// 不安全写法：
/// let x: u64 = unsafe { std::mem::transmute(3.14f64) };
///
/// 安全替代：
fn safe_type_punning() {
    // 方案1: to_bits / from_bits（标准库提供）
    let bits: u64 = 3.14f64.to_bits();

    // 方案2: bytemuck crate（编译期检查 Pod/Zeroable）
    // let bits: u64 = bytemuck::cast(3.14f64);

    // 方案3: union-based type punning（4.1 unsafe + 显式审计）
    union FloatOrInt {
        f: f64,
        i: u64,
    }
    let u = FloatOrInt { f: 3.14 };
    let bits = unsafe { u.i };  // unsafe 明确，可审计
    // 优于 transmute——union 的布局是明确定义的
}

/// ━━━ 4.4 asm! ━━━
/// spin_loop 的 x86_64 PAUSE 指令——减少功耗
fn spin_loop_hint() {
    #[cfg(target_arch = "x86_64")]
    unsafe {
        std::arch::asm!("pause", options(nomem, nostack));
        // PAUSE 指令告诉 CPU 这只是自旋——省电 + 让超线程
    }
    // 等价于 std::hint::spin_loop()——标准库已经封装好了
}
```

## 3. Demo 2: C FFI 全栈 — extern "C" → 安全包装 → bindgen

### 3.1 设计哲学

> "C 库有几十年的积累——你不可能全部用 Rust 重写。FFI 是 Rust 作为系统语言的必修课。但 FFI 也是不安全的大本营：null 指针、dangling 引用、跨语言内存管理。本 Demo 带你走完整条链路：从 raw extern 'C' 到手工安全包装，再到 bindgen 自动生成。"

### 3.2 场景：包装一个简易的 C 字符串库

```c
// mylib.h — 要包装的 C 库
typedef struct MyString MyString;  // ━━━ 4.9 Opaque type ━━━

MyString* mystring_new(const char* s);
void mystring_free(MyString* s);
const char* mystring_c_str(const MyString* s);
int mystring_len(const MyString* s);
MyString* mystring_concat(const MyString* a, const MyString* b);
```

### 3.3 Step 1: Raw extern "C" 声明

```rust
// ━━━ 4.7 FFI C: extern "C" 块 ━━━

/// Opaque type: Rust 不知道 MyString 的大小/布局（4.9）
#[repr(C)]
struct MyString {
    _private: [u8; 0],  // 零大小占位符——永不实例化
}

extern "C" {
    fn mystring_new(s: *const libc::c_char) -> *mut MyString;
    fn mystring_free(s: *mut MyString);
    fn mystring_c_str(s: *const MyString) -> *const libc::c_char;
    fn mystring_len(s: *const MyString) -> libc::c_int;
    fn mystring_concat(a: *const MyString, b: *const MyString) -> *mut MyString;
}

// ━━━ 4.14 OS: libc crate 提供跨平台的 C 类型 ━━━
// libc::c_char / c_int —— Linux 和 macOS 可能有不同的定义
```

### 3.4 Step 2: 安全包装

```rust
use std::ffi::{CStr, CString};
use std::fmt;

/// ━━━ 4.1 Unsafe: 安全包装的责任模型 ━━━
///
/// 安全包装必须保证：无论用户怎么使用，都不会违反 Rust 的安全保证。
/// 这包括：
/// 1. 不能构造悬垂指针
/// 2. 不能有 use-after-free
/// 3. 不能有 null 指针解引用
/// 4. 跨线程安全性（Send/Sync）

pub struct SafeString {
    inner: *mut MyString,  // 私有——外部不能直接访问
}

impl SafeString {
    /// 从 Rust &str 构造
    pub fn new(s: &str) -> Self {
        let c_str = CString::new(s).expect("CString::new 失败——字符串中有 null 字节");

        // ━━━ Safety（4.1）━━━
        // mystring_new 期望一个有效的 null-terminated C string
        // CString::new 保证这一点
        let inner = unsafe { mystring_new(c_str.as_ptr()) };

        // ━━━ null 检查（4.2 裸指针）━━━
        if inner.is_null() {
            panic!("mystring_new 返回 null——内存分配失败");
        }

        SafeString { inner }
    }

    /// 获取 Rust 字符串引用
    pub fn as_str(&self) -> &str {
        let c_ptr = unsafe { mystring_c_str(self.inner) };
        // Safety: mystring_c_str 返回的指针在 self.inner 有效期间一直有效
        // CStr::from_ptr 不会拷贝数据——零拷贝
        unsafe { CStr::from_ptr(c_ptr) }
            .to_str()
            .expect("C 字符串不是有效的 UTF-8")
    }

    pub fn len(&self) -> usize {
        unsafe { mystring_len(self.inner) as usize }
    }

    pub fn concat(&self, other: &SafeString) -> SafeString {
        let inner = unsafe { mystring_concat(self.inner, other.inner) };
        if inner.is_null() {
            panic!("mystring_concat 返回 null");
        }
        SafeString { inner }
    }
}

// ━━━ Drop: 谁分配谁释放（4.7 FFI）━━━
impl Drop for SafeString {
    fn drop(&mut self) {
        // Safety: self.inner 由 mystring_new 或 mystring_concat 分配
        // 在 mystring_free 调用后不再使用
        unsafe { mystring_free(self.inner) };
    }
}

// ━━━ 4.7 FFI: catch_unwind 防止 panic 跨越 FFI ━━━
// 如果 Rust 的 extern "C" fn 被 C 代码回调调用，
// 内部 panic 必须被捕获——否则 unwind 跨越 FFI 边界是 UB
extern "C" fn rust_callback(data: *mut libc::c_void) -> libc::c_int {
    let result = std::panic::catch_unwind(|| {
        // 处理回调...
        0
    });
    match result {
        Ok(code) => code,
        Err(_) => -1,  // panic → 返回错误码
    }
}

impl fmt::Display for SafeString {
    fn fmt(&self, f: &mut fmt::Formatter<'_>) -> fmt::Result {
        write!(f, "{}", self.as_str())
    }
}
```

## 4. Demo 3: 向量化实战 — SIMD 加速字符串搜索

### 4.1 设计哲学

> "现代 CPU 一次可以处理 256 位（8 个 i32 或 32 个 u8）。不用 SIMD，你的串行代码只用了 CPU 的 1/8 能力。但手动 SIMD 是 unsafe 的重灾区——对齐、边界、平台兼容。本 Demo 带你从标量到 SSE 到 AVX 到 `std::simd`，用 sanitizers 全程验证。"

### 4.2 场景：在字节串中找目标字节的位置

```rust
/// 标量版本：逐个字节比较
fn find_byte_scalar(haystack: &[u8], needle: u8) -> Option<usize> {
    haystack.iter().position(|&b| b == needle)
}

/// ━━━ 4.11 SIMD: SSE2 一次比较 16 字节 ━━━
#[cfg(target_arch = "x86_64")]
mod simd_impl {
    use std::arch::x86_64::*;

    /// SSE2 版本：用 _mm_loadu_si128 一次加载 16 字节
    pub fn find_byte_sse2(haystack: &[u8], needle: u8) -> Option<usize> {
        // 广播 needle 到 16 字节的 SIMD 寄存器
        let needle_vec = unsafe { _mm_set1_epi8(needle as i8) };

        // 按 16 字节分块处理
        for (chunk_idx, chunk) in haystack.chunks(16).enumerate() {
            // ━━━ 4.2 裸指针 + 4.1 Unsafe ━━━
            // _mm_loadu_si128: loadu = 不要求对齐（unaligned）
            let chunk_vec = if chunk.len() == 16 {
                unsafe { _mm_loadu_si128(chunk.as_ptr() as *const __m128i) }
            } else {
                // 尾部不足 16 字节——回退标量
                for (offset, &b) in chunk.iter().enumerate() {
                    if b == needle {
                        return Some(chunk_idx * 16 + offset);
                    }
                }
                return None;
            };

            // ━━━ 4.3 transmute 等价操作: SIMD 比较 ━━━
            let cmp = unsafe { _mm_cmpeq_epi8(chunk_vec, needle_vec) };
            let mask = unsafe { _mm_movemask_epi8(cmp) } as u32;

            if mask != 0 {
                // 找到第一个匹配位
                let offset = mask.trailing_zeros() as usize;
                return Some(chunk_idx * 16 + offset);
            }
        }
        None
    }
}

/// ━━━ 4.4 asm! + 4.5 intrinsics: CPU 特性检测 ━━━
fn has_avx2() -> bool {
    #[cfg(target_arch = "x86_64")]
    {
        // ━━━ 4.5 Intrinsics: cpuid 指令封装 ━━━
        is_x86_feature_detected!("avx2")
    }
    #[cfg(not(target_arch = "x86_64"))]
    false
}

/// ━━━ 4.6 hint: black_box 用于基准测试 ━━━
#[cfg(test)]
mod bench {
    use super::*;
    use std::hint::black_box;

    #[test]
    fn bench_find_byte() {
        let data = vec![0u8; 100_000];

        // black_box 阻止编译器优化掉"无用"的计算（4.6）
        let result = black_box(find_byte_scalar(black_box(&data), black_box(255)));
        assert_eq!(result, None);
    }
}

/// ━━━ std::simd: 平台无关的 SIMD（nightly）━━━
/// 比 x86_64 intrinsic 更优雅——一份代码，多平台
///
/// ```rust,ignore
/// use std::simd::{Simd, Mask, ToBitMask};
///
/// fn find_byte_portable(haystack: &[u8], needle: u8) -> Option<usize> {
///     // Simd<u8, 16> 在 x86 编译为 SSE，在 ARM 编译为 NEON
///     let needle_vec = Simd::splat(needle);
///     // ...
///     None
/// }

/// ━━━ 4.12 Sanitizers: 验证 SIMD 代码 ━━━
/// SIMD 代码容易出：
/// 1. 越界读取（loadu 在 chunk 不足 16 字节时）
/// 2. 未对齐访问（load 而非 loadu）

/// cargo +nightly miri test  # 检测纯 Rust UB
/// RUSTFLAGS="-C target-feature=+avx2" cargo test  # 启用 AVX2 测试

## 5. Demo 4: 动态插件系统 — dlopen + VTable + ABI 稳定

### 5.1 设计哲学

> "插件系统是系统编程的王冠明珠。Rust 虽然不保证 ABI 稳定，但 C ABI 是稳定的——而任何 Rust 类型都可以通过 `#[repr(C)]` + 手动 VTable 暴露为 C ABI。本 Demo 带你实现一个完整的插件系统：编译插件为 cdylib、运行时 dlopen 加载、VTable 派发、跨 FFI 内存管理。"

### 5.2 插件接口定义（共享 crate）

```rust
// plugin_interface crate — 插件和宿主共享的定义

/// ━━━ 4.13 ABI: C ABI 是跨语言/跨版本的稳定接口 ━━━
/// VTable 模式：把 trait 方法表暴露为 C 兼容的结构体

#[repr(C)]
pub struct PluginVTable {
    /// 插件名称
    pub name: unsafe extern "C" fn() -> *const libc::c_char,
    /// 处理数据
    pub process: unsafe extern "C" fn(*const u8, usize) -> *mut u8,
    /// 获取处理后的数据长度
    pub result_len: unsafe extern "C" fn() -> usize,
    /// 释放插件
    pub destroy: unsafe extern "C" fn(),
}

/// ━━━ 4.9 Opaque: 插件内部状态对宿主不可见 ━━━
#[repr(C)]
pub struct PluginContext {
    _private: [u8; 0],
}

/// 每个插件必须导出的入口函数
/// 返回 PluginVTable 和 PluginContext 指针
pub type PluginEntry = unsafe extern "C" fn() -> (*const PluginVTable, *mut PluginContext);
```

### 5.3 插件实现（cdylib）

```rust
// my_plugin crate — 编译为 cdylib
// Cargo.toml: [lib] crate-type = ["cdylib"]

use plugin_interface::*;
use std::ffi::CString;

struct MyPlugin {
    result: Vec<u8>,
}

// ━━━ 4.10 Allocator: 跨 FFI 的内存 ━━━
// 注意：Rust 不同 crate 之间不共享 allocator
// 所以跨 FFI 传递的指针必须是 C 分配（malloc/free）或由同一方管理

static mut PLUGIN_INSTANCE: Option<MyPlugin> = None;

unsafe extern "C" fn get_name() -> *const libc::c_char {
    let name = CString::new("MyPlugin v1.0").unwrap();
    name.into_raw()  // 调用者负责用 CString::from_raw 释放
}

unsafe extern "C" fn process(data: *const u8, len: usize) -> *mut u8 {
    let input = std::slice::from_raw_parts(data, len);
    let plugin = PLUGIN_INSTANCE.as_mut().unwrap();

    // 处理数据...
    plugin.result = input.to_vec();
    for byte in &mut plugin.result {
        *byte = byte.wrapping_add(1);  // 简单的转换
    }

    plugin.result.as_mut_ptr()  // 宿主负责读取，不需要释放
}

unsafe extern "C" fn result_len() -> usize {
    PLUGIN_INSTANCE.as_ref().unwrap().result.len()
}

unsafe extern "C" fn destroy() {
    // ━━━ 4.14 OS: 注意 DLL 卸载顺序 ━━━
    // 在 dlclose 之前必须调用 destroy
    let _ = PLUGIN_INSTANCE.take();
}

// ━━━ 4.7 FFI: #[no_mangle] 确保符号可被 dlsym 找到 ━━━
#[no_mangle]
pub unsafe extern "C" fn plugin_entry() -> (*const PluginVTable, *mut PluginContext) {
    PLUGIN_INSTANCE = Some(MyPlugin { result: Vec::new() });

    let vtable = Box::new(PluginVTable {
        name: get_name,
        process,
        result_len,
        destroy,
    });

    (Box::into_raw(vtable), PLUGIN_INSTANCE.as_mut().unwrap() as *mut MyPlugin as *mut PluginContext)
}
```

### 5.4 宿主加载插件（dlopen）

```rust
// host crate — 加载并使用插件

use plugin_interface::*;
use std::ffi::CStr;

struct LoadedPlugin {
    vtable: *const PluginVTable,
    context: *mut PluginContext,
    // ━━━ 4.14 OS: dlopen 的句柄 ━━━
    #[cfg(target_os = "linux")]
    _lib: *mut libc::c_void,
    #[cfg(target_os = "macos")]
    _lib: *mut libc::c_void,
    #[cfg(target_os = "windows")]
    _lib: *mut libc::c_void,
}

impl LoadedPlugin {
    /// ━━━ 4.13 Dynamic: dlopen/dlsym ━━━
    fn load(path: &str) -> Result<Self, String> {
        // ━━━ 4.14 OS: dlopen ━━━
        let c_path = CString::new(path).map_err(|e| e.to_string())?;

        #[cfg(target_os = "linux")]
        let lib = unsafe {
            libc::dlopen(c_path.as_ptr(), libc::RTLD_NOW)
        };
        #[cfg(target_os = "macos")]
        let lib = unsafe {
            libc::dlopen(c_path.as_ptr(), libc::RTLD_NOW)
        };
        #[cfg(target_os = "windows")]
        let lib = unsafe {
            // Windows 用 LoadLibraryA
            std::ptr::null_mut()  // 简化
        };

        if lib.is_null() {
            #[cfg(unix)]
            let err = unsafe { CStr::from_ptr(libc::dlerror()) };
            return Err(format!("dlopen 失败"));
        }

        // ━━━ dlsym: 查找 "plugin_entry" 符号 ━━━
        let symbol_name = CString::new("plugin_entry").unwrap();
        #[cfg(unix)]
        let entry: PluginEntry = unsafe {
            let ptr = libc::dlsym(lib, symbol_name.as_ptr());
            if ptr.is_null() {
                return Err("dlsym 失败: plugin_entry 不存在".into());
            }
            std::mem::transmute(ptr)  // ━━━ 4.3 transmute ━━━
        };

        let (vtable, context) = unsafe { entry() };
        Ok(LoadedPlugin { vtable, context, _lib: lib })
    }

    fn name(&self) -> &str {
        let c_str = unsafe { ((*self.vtable).name)() };
        unsafe { CStr::from_ptr(c_str) }.to_str().unwrap_or("???")
    }

    fn process(&self, data: &[u8]) -> Vec<u8> {
        let result_ptr = unsafe { ((*self.vtable).process)(data.as_ptr(), data.len()) };
        let len = unsafe { ((*self.vtable).result_len)() };
        unsafe { std::slice::from_raw_parts(result_ptr, len).to_vec() }
    }
}

impl Drop for LoadedPlugin {
    fn drop(&mut self) {
        // ━━━ 4.7 FFI: 先 destroy 再 dlclose ━━━
        unsafe { ((*self.vtable).destroy)() };

        #[cfg(unix)]
        unsafe { libc::dlclose(self._lib) };
    }
}
```

## 6. 综合练习：内存安全的 mmap 文件读取器

### 6.1 题目

实现 `MmapFile`，通过 `mmap` 系统调用（4.14 OS）将文件映射到内存，提供安全的 Rust 接口：

1. **mmap 封装**（4.14）：`unsafe fn map(path) -> Result<MmapFile>`
2. **裸指针管理**（4.2）：用 `NonNull<u8>` 持有映射地址
3. **Deref 到 `[u8]`**（4.2）：安全地暴露为 `&[u8]`
4. **Drop munmap**（4.1）：正确释放映射
5. **Send/Sync 决策**（4.1）：mmap 数据是否跨线程安全？
6. **Sanitizer 验证**（4.12）：ASan 检测越界
7. **可选 SIMD 搜索**（4.11）：在 mmap 数据上用 SIMD 加速搜索

### 6.2 参考答案

```rust
use std::ptr::NonNull;
use std::ops::Deref;
use std::fs;

struct MmapFile {
    ptr: NonNull<u8>,
    len: usize,
}

impl MmapFile {
    fn open(path: &str) -> Result<Self, String> {
        let file = fs::File::open(path).map_err(|e| e.to_string())?;
        let len = file.metadata().map_err(|e| e.to_string())?.len() as usize;

        if len == 0 {
            return Err("空文件".into());
        }

        // ━━━ 4.14 OS: mmap ━━━
        #[cfg(unix)]
        use std::os::unix::io::AsRawFd;
        #[cfg(unix)]
        let ptr = unsafe {
            libc::mmap(
                std::ptr::null_mut(),
                len,
                libc::PROT_READ,
                libc::MAP_PRIVATE,  // MAP_PRIVATE: 修改不写回文件
                file.as_raw_fd(),
                0,
            )
        };

        #[cfg(not(unix))]
        let ptr = std::ptr::null_mut();  // 非 Unix 简化

        if ptr == libc::MAP_FAILED || ptr.is_null() {
            return Err("mmap 失败".into());
        }

        Ok(MmapFile {
            ptr: unsafe { NonNull::new_unchecked(ptr) },
            len,
        })
    }

    fn as_slice(&self) -> &[u8] {
        unsafe { std::slice::from_raw_parts(self.ptr.as_ptr(), self.len) }
    }
}

impl Deref for MmapFile {
    type Target = [u8];
    fn deref(&self) -> &[u8] {
        self.as_slice()
    }
}

impl Drop for MmapFile {
    fn drop(&mut self) {
        // ━━━ 4.1 Unsafe: munmap ━━━
        #[cfg(unix)]
        unsafe { libc::munmap(self.ptr.as_ptr() as *mut libc::c_void, self.len) };
    }
}

// ━━━ 4.1 Send/Sync 决策 ━━━
// MmapFile 的数据在 mmap 区域——不会被 Rust 移动
// 多个线程读取是安全的（只读映射）
unsafe impl Send for MmapFile {}
unsafe impl Sync for MmapFile {}
```
